    Implementing Interface:
Pros and Cons
&#x2022; Advantages
&#x2013; No extra steps needed to call methods in main app
&#x2022; The code is part of the main app, so it can call any method or
access any instance variable, even private ones.
&#x2013; Simple
&#x2022; Widely used in real life for button handlers, when you know you
will have only one button, or for threading code when you need
only one method to run in the background.
&#x2022; Disadvantage
&#x2013; Inflexible: hard to have multiple different versions since you
cannot pass arguments to handler
&#x2022; For example, what if you want to sort arrays two different ways?
If you supply &#x201C;this&#x201D; as second argument to Arrays.sort, it will
refer to the same &#x201C;compare&#x201D; method both times.

    Generic Classes and Methods
&#x2022; Big idea
&#x2013; Even beginning Java programmers need to know how to use
classes that already support generics:
&#x2022; List&lt;Employee&gt; workers = &#x2026;;
&#x2022; Map&lt;String, Employee&gt; workerDatabase = &#x2026;;
&#x2013; But, it is also important to be able to define classes or methods
that support generics. Used a lot with lambdas.
&#x2022; Lambdas and generics have same top-level goal: to make code
that is more flexible and reusable
&#x2022; Syntax
&#x2013; If you put variables in angle brackets in the class or method
definition, this tells Java that future uses of those variables
refer to types, not instances.
&#x2022; public class ArrayList&lt;E&gt; { &#x2026; }
&#x2022; public static &lt;T&gt; T randomElement(T[] array) { &#x2026; }

public class RandomUtils {
private static Random r = new Random();
public static int randomInt(int range) {
return(r.nextInt(range));
}
public static int randomIndex(Object[] array) {
return(randomInt(array.length));
}
public static &lt;T&gt; T randomElement(T[] array) {
return(array[randomIndex(array)]);
}
}

In rest of method, T refers to a type This says that the method takes in an array of T&#x2019;s and returns a T.
For example, if you pass in an array of Strings, you get out a String;
if you pass in an array of Employees, you get out an Employee.
No typecasts required in any of the cases.

String[] names = { "Joe", "John", "Jane" };
String name = RandomUtils.randomElement(names);
Color[] colors = { Color.RED, Color.GREEN, Color.BLUE };
Color color = RandomUtils.randomElement(colors);
Person[] people =
{ new Person("Larry", "Page"), new Person("Larry", "Ellison"),
new Person("Larry", "Bird"), new Person("Larry", "King") };
Person person = RandomUtils.randomElement(people);
Integer[] nums = { 1, 2, 3, 4 }; // Integer[], not int[]
int num = RandomUtils.randomElement(nums);
&#x2022; Points
&#x2013; No typecast required to convert to String, Color, Person, Integer
&#x2013; Autoboxing lets you assign entry from Integer[] to an int, but array
must be Integer[] not int[], since generics work only with Object
types, not primitive types
