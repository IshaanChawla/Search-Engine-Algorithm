						<div class="post-headline">		<h1>1.1 &#8212; Structure of a program</h1>
		</div>		<div class="post-byline">By Alex, on May 30th, 2007</div>		<div class="post-bodycopy clearfix"><p>A computer program is a sequence of instructions that tell the computer what to do.  </p>
<p><strong>Statements and expressions</strong></p>
<p>The most common type of instruction in a program is the <strong>statement</strong>.  A statement in C++ is the smallest independent unit in the language.  In human language, it is analogous to a sentence.  We write sentences in order to convey an idea.  In C++, we write statements in order to convey to the compiler that we want to perform a task.  <strong>Statements in C++ are terminated by a semicolon</strong>.</p>
<p>There are many different kinds of statements in C++.  The following are some of the most common types of simple statements:</p>

<p></p>
<p><code>int x</code> is a <strong>declaration statement</strong>.  It tells the compiler that x is a variable.  You may remember variables from math in school.  They serve the same purpose here: to represent an arbitrary value.  All variables in a program must be declared before they are used.  We will talk more about variables shortly.</p>
<p><code>x = 5</code> is an <strong>assignment statement</strong>.  It assigns a value (5) to a variable (x).</p>
<p><code>std::cout &lt;&lt; x;</code> is an <strong>output statement</strong>.  It outputs the value of x (which we set to 5 in the previous statement) to the screen.</p>
<p>The compiler is also capable of resolving expressions.  An <strong>expression</strong> is an mathematical entity that evaluates to a value.  For example, in math, the expression 2+3 evaluates to the value 5.  Expressions can involve values (such as 2), variables (such as x), operators (such as +) and functions (which return an output value based on some input value).  They can be singular (such as 2, or x), or compound (such as 2+3, 2+x, x+y, or (2+x)*(y-3)).</p>
<p>For example, the statement <code>x =  2 + 3;</code> is a valid assignment statement.  The expression 2+3 evaluates to the value of 5.  This value of 5 is then assigned to x.</p>
<p><strong>Functions</strong></p>
<p>In C++, statements are typically grouped into units called functions.  A <strong>function</strong> is a collection of statements that executes sequentially.  Every C++ program must contain a special function called <strong>main()</strong>.  When the C++ program is run, execution starts with the first statement inside of main().  Functions are typically written to do a very specific job.  For example, a function named Max() might contain statements that figures out which of two numbers is larger.  A function named CalculateGrade() might calculate a student&#8217;s grade.  We will talk more about functions later.</p>
<p><strong>Libraries and the C++ Standard Library</strong></p>
<p>A <strong>library</strong> is a collection of precompiled code (e.g. functions) that has been &#8220;packaged up&#8221; for reuse in many different programs.  Libraries provide a common way to extend what your programs can do.  For example, if you were writing a game, you&#8217;d probably want to include a sound library and a graphics library.</p>
<p>The C++ core language is actually very small and minimalistic (and you&#8217;ll learn most of it in these tutorials).  However, C++ also comes with a library called the <strong>C++ standard library</strong> that provides additional functionality for your use.  One of the most commonly used parts of the C++ standard library is the iostream library, which contains functionality for writing to the screen and getting input from a console user.</p>
<p><strong>Taking a look at a sample program</strong></p>
<p>Now that you have a brief understanding of what statements, functions, and libraries are, let&#8217;s look at a simple &#8220;hello world&#8221; program:</p>

<p></p>
<p>Line 1 is a special type of statement called a <strong>preprocessor directive</strong>.  Preprocessor directives tell the compiler to perform a special task.  In this case, we are telling the compiler that we would like to add the contents of the iostream header to our program.  The iostream header allows us to access functionality in the iostream library, which will allow us to write to the screen.</p>
<p>Line 2 has nothing on it, and is ignored by the compiler.</p>
<p>Line 3 declares the main() function, which as you learned above, is mandatory.  Every program must have a main() function.</p>
<p>Lines 4 and 7 tell the compiler which lines are part of the main function.  Everything between the opening curly brace on line 4 and the closing curly brace on line 7 is considered part of the main() function.</p>
<p>Line 5 is our first statement (you can tell it&#8217;s a statement because it ends with a semicolon), and it is an output statement.  std::cout is a special object that represents the console/screen.  The << symbol is an operator (much like + is an operator) called the <strong>output operator</strong>.  std::cout understands that anything sent to it via the output operator should be printed on the screen.  In this case, we&#8217;re sending it the text &#8220;Hello world!&#8221;.</p>
<p>Line 6 is a new type of statement, called a <strong>return statement</strong>.  When an executable program finishes running, the main() function sends a value back to the operating system that indicates whether it was run successfully or not.</p>
<p>This particular return statement returns the value of 0 to the operating system, which means &#8220;everything went okay!&#8221;.  Non-zero numbers are typically used to indicate that something went wrong, and the program had to abort.  We will discuss return statements in more detail when we discuss functions.</p>
<p>All of the programs we write will follow this template, or a variation on it.  We will discuss each of the lines above in more detail in the upcoming sections.</p>
<p><strong>Syntax and syntax errors</strong></p>
<p>In English, sentences are constructed according to specific grammatical rules that you probably learned in English class in school.  For example, normal sentences end in a period.  The rules that govern how sentences are constructed in a language is called <strong>syntax</strong>.  If you forget the period and run two sentences together, this is a violation of the English language syntax.</p>
<p>C++ has a syntax too: rules about how your programs must be constructed in order to be considered valid.  When you compile your program, the compiler is responsible for making sure your program follows the basic syntax of the C++ language.  If you violate a rule, the compiler will complain when you try to compile your program, and issue you a <strong>syntax error</strong>.</p>
<p>For example, you learned above that statements must end in a semicolon.</p>

<p>Visual studio produces the following error:</p>
<pre>c:\users\apomeranz\documents\visual studio 2013\projects\test1\test1\test1.cpp(6): error C2143: syntax error : missing ';' before 'return'</pre>
<p>This is telling you that you have an syntax error on line 6: You&#8217;ve forgotten a semicolon before the return.  In this case, the error is actually at the end of line 5.  Often, the compiler will pinpoint the exact line where the syntax error occurs for you.  However, sometimes it doesn&#8217;t notice until the next line.</p>
<p>Syntax errors are common when writing a program.  Fortunately, they&#8217;re often easily fixable.  The program can only be fully compiled (and executed) once all syntax errors are resolved.</p>

		<p>Really good question about the difference between declaration and definition.  The easiest way to think about it is as follows:</p>
<p>* A declaration introduces an object object (function, variable, etc&#8230;) and it&#8217;s type.  A function prototype is an example of a declaration.  A declaration is enough to satisfy the compiler.  You can have multiple declarations for the same object.</p>
<p>* A definition actually defines that object.  A function with a body is an example of a definition. A definition is needed to satisfy the linker.  There can only be one definition for an object.</p>
<p>In many cases, a single line serves as both the declaration and definition.  For example, &#8220;int x&#8221; is a declaration AND a definition.</p>
<p>Memory is allocated at the point of instantiation, which is when the object (variable) is actually created.  This point can vary depending on where the variable is.  For example, if a variable lives inside a function, the variable won&#8217;t be created (and memory assigned) until that function executes.</p>
