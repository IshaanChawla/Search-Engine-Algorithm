                <h1>Variables and Types</h1>
                
                <p>Although Java is object oriented, not all types are objects. It is built on top of basic variable types called primitives.</p>
<p>Here is a list of all primitives in Java:</p>
<ul>
<li><code>byte</code> (number, 1 byte)</li>
<li><code>short</code> (number, 2 bytes)</li>
<li><code>int</code> (number, 4 bytes)</li>
<li><code>long</code> (number, 8 bytes)</li>
<li><code>float</code> (float number, 4 bytes)</li>
<li><code>double</code> (float number, 8 bytes)</li>
<li><code>char</code> (a character, 2 bytes)</li>
<li><code>boolean</code> (true or false, 1 byte)</li>
</ul>
<p>Java is a strong typed language, which means variables need to be defined before we use them.</p>
<h3>Numbers</h3>
<p>To declare and assign a number use the following syntax:</p>
<pre><code>int myNumber;
myNumber = 5;
</code></pre>
<p>Or you can combine them:</p>
<pre><code>int myNumber = 5;
</code></pre>
<p>To define a floating point number, use the following syntax:</p>
<pre><code>double d = 4.5;
d = 3.0;
</code></pre>
<p>If you want to use float, you will have to cast:</p>
<pre><code>float f = (float) 4.5;
</code></pre>
<p>Or, You can use this:</p>
<pre><code>float f = 4.5f (f is a shorter way of casting float)
</code></pre>
<h3>Characters and Strings</h3>
<p>In Java, a character is it's own type and it's not simply a number, so it's not common to put an ascii value in it, there is a special syntax for chars:</p>
<pre><code>char c = 'g';
</code></pre>
<p><code>String</code> is not a primitive. It's a real type, but Java has special treatment for String.</p>
<p>Here are some ways to use a string:</p>
<pre><code>// Create a string with a constructor
String s1 = new String("Who let the dogs out?");
// Just using "" creates a string, so no need to write it the previous way.
String s2 = "Who who who who!";
// Java defined the operator + on strings to concatenate:
String s3 = s1 + s2;
</code></pre>
<p>There is no operator overloading in Java! The operator <code>+</code> is only defined for strings, you will never see it with other objects, only primitives.</p>
<p>You can also concat string to primitives:</p>
<pre><code>int num = 5;
String s = "I have " + num + " cookies"; //Be sure not to use "" with primitives.
</code></pre>
<h3>boolean</h3>
<p>Every comparison operator in java will return the type <code>boolean</code> that not like other languages can only accept two special values: <code>true</code> or <code>false</code>.</p>
<pre><code>boolean b = false;
b = true;

boolean toBe = false;
b = toBe || !toBe;
if (b) {
    System.out.println(toBe);
}

int children = 0;
b = children; // Will not work
if (children) { // Will not work
    // Will not work
}
</code></pre>
