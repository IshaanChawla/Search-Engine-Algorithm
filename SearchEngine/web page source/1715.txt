Start planning for Google I/O!
</a></h2>
<div class='post-header'>
<div class='published'><span class='caption'>Posted:</span>
<span class='publishdate' itemprop='datePublished'>Wednesday, May 6, 2015</span>
</div>
<div class='share'>
<span class='gplus-wrapper'>
<div class='g-plusone' data-href='http://googledevelopers.blogspot.com/2015/05/start-planning-for-google-io.html' data-size='medium'></div>
</span>
<a class='twitter-share-button' data-lang='en' data-url='http://googledevelopers.blogspot.com/2015/05/start-planning-for-google-io.html' href='https://twitter.com/share'>Tweet</a>
<iframe allowTransparency='true' frameborder='0' scrolling='no' src='http://www.facebook.com/plugins/like.php?href=http://googledevelopers.blogspot.com/2015/05/start-planning-for-google-io.html&layout=button_count&show_faces=false&width=77&action=like&colorscheme=light&height=20' style='border:none; overflow:hidden; width:77px; height:20px;'></iframe>
</div>
</div>
<div class='post-content' itemprop='articleBody'>
<p><i>Posted by <a href="https://plus.google.com/+mikepegg21">Mike Pegg</a>, reppin' I/O since 2011</i></p>

<p>Today we launched the official schedule for Google I/O 2015 at <a href="https://events.google.com/io2015/schedule">google.com/io</a>. At this year&#8217;s event, happening May 28-29 in San Francisco, we&#8217;ll host more than 200 talks centered around  some important topics which matter to you: <b><a href="https://events.google.com/io2015/schedule?filters=Develop%20%26%20Design#day1/">Design & Develop</a></b>, to help you build beautiful, powerful apps; <b><a href="https://events.google.com/io2015/schedule?filters=Engage%20%26%20Earn#day1/">Earn & Engage</a></b>, where we&#8217;ll cover tools to grow your user base and create sustainable, successful businesses; and <b><a href="https://events.google.com/io2015/schedule?filters=What%27s%20Next#day2">What&#8217;s Next</a></b>, a peek into Google&#8217;s emerging platforms. With just over three weeks until Google I/O, start planning your schedule today! </p>

<h3>Start building your schedule</h3>
<p>Whether you&#8217;re attending in person or virtually, you can get started building your schedule. Don&#8217;t worry about converting the start and end times to your local time zone, we&#8217;ve taken care of that for you. Simply sign in to the <a href="https://events.google.com/io2015/schedule#myschedule">I/O website</a> to add talks directly to &#8220;My Schedule.&#8221; If you&#8217;re using Chrome (on Android or desktop), you can enable notifications for events added to your schedule so that you can be sure to catch them. That way, you won&#8217;t miss exciting sessions like Astro Teller&#8217;s &#8220;<a href="https://events.google.com/io2015/schedule?filters=Play%2CWhat%27s%20Next#day2/106c458e-b8dc-e411-b87f-00155d5066d7">Helping Moonshots Survive Contact with the Real World</a>&#8221; or an <a href="https://events.google.com/io2015/schedule?filters=Play%2CWhat%27s%20Next#day2/65718f8b-b6d4-e411-b87f-00155d5066d7">update from the ATAP team</a> on some cool new projects they&#8217;re working on. All sessions will be livestreamed, so whether you&#8217;re watching from one of the 400 <a href="https://events.google.com/io2015/offsite#join">I/O Extended Locations around the world</a> or the comfort of your own desk, we&#8217;ve got you covered. </p>

<h3>Attending in person</h3>
<p>In addition to the traditional breakout sessions, which are livestreamed, if you&#8217;re attending in person, you&#8217;ll also get a chance to go to more than 100 <b><a href="https://events.google.com/io2015/schedule?filters=Sandbox%20talks#day1">sandbox talks</a></b>. These intimate, 20-minute talks are often more technical, and the smaller size means that you&#8217;ll get a chance to interact directly with the Googlers teaching them. Together, you can roll up your sleeves and tackle topics ranging from &#8220;<a href="https://events.google.com/io2015/schedule#day1/508516fc-81dc-e411-b87f-00155d5066d7">Memory Performance & Tooling</a>&#8221; to &#8220;<a href="https://events.google.com/io2015/schedule?filters=Play#day1/90d26332-5cdd-e411-b87f-00155d5066d7">What's new in the Google Play Developer Console</a>.&#8221; Most sandbox talks will happen twice throughout the two-day event, so you&#8217;ll have more chances to participate. </p>

<div class="separator" style="clear: both; text-align: center;"><a href="http://3.bp.blogspot.com/-ApZZNDKyJV0/VUpGn8AuQQI/AAAAAAAAAs4/BW3S3c23QMY/s1600/io14-018.jpg" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://3.bp.blogspot.com/-ApZZNDKyJV0/VUpGn8AuQQI/AAAAAAAAAs4/BW3S3c23QMY/s640/io14-018.jpg" /></a></div>
<center><i>This year, there are over 100 sandbox talks: intimate, 20-minute technical talks where you can roll up your sleeves and interact directly with Googlers. </i></center>


<p>Don&#8217;t forget to save time in your schedule for a <b><a href="https://events.google.com/io2015/schedule?filters=Code%20labs#day2">code lab</a></b> or two. Back by popular demand, these self-paced workshops will showcase a variety of technologies from Google on mobile, wearables, and Cloud to name a few. We&#8217;ll provide the workstations and tablets for use on-site - just bring yourself any time during the two days of I/O! If you have your own device, Googlers will be on hand to help you get set up so you can jump into it. </p>

<h3>See you soon</h3>
<p>We&#8217;re getting really excited about Google I/O 2015 and today&#8217;s schedule is just a preview of what&#8217;s to come. We&#8217;ll be adding more sessions, sandbox talks, and events to the schedule as we get closer to I/O. But, we can&#8217;t give everything away beforehand.  Be sure to check the agenda again after the keynote on Day 1,  for those top secret talks.  We look forward to connecting with you in-person, at <a href="https://events.google.com/io2015/offsite">I/O Extended</a> or via <a href="https://events.google.com/io2015/offsite">I/O Live</a> in a few weeks!</p>

Docs, Sheets and Forms add-ons now open to all developers
</a></h2>
<div class='post-header'>
<div class='published'><span class='caption'>Posted:</span>
<span class='publishdate' itemprop='datePublished'>Thursday, April 23, 2015</span>
</div>
<div class='share'>
<span class='gplus-wrapper'>
<div class='g-plusone' data-href='http://googledevelopers.blogspot.com/2015/04/docs-sheets-and-forms-add-ons-now-open.html' data-size='medium'></div>
</span>
<a class='twitter-share-button' data-lang='en' data-url='http://googledevelopers.blogspot.com/2015/04/docs-sheets-and-forms-add-ons-now-open.html' href='https://twitter.com/share'>Tweet</a>
<iframe allowTransparency='true' frameborder='0' scrolling='no' src='http://www.facebook.com/plugins/like.php?href=http://googledevelopers.blogspot.com/2015/04/docs-sheets-and-forms-add-ons-now-open.html&layout=button_count&show_faces=false&width=77&action=like&colorscheme=light&height=20' style='border:none; overflow:hidden; width:77px; height:20px;'></iframe>
</div>
</div>
<div class='post-content' itemprop='articleBody'>
<p><i>Posted by <a href="https://plus.google.com/105831704374179338116/posts">Saurabh Gupta</a>, Product Manager</i></p>

<p>Back in 2014, we introduced <a href="https://developers.google.com/apps-script/add-ons?utm_campaign=docs-add-ons-415&utm_source=gdbc&utm_medium=blog">add-ons for Google Docs, Sheets, and Forms</a> in developer preview. Since then, the developer community  has built a wide variety of features to help millions of Docs, Sheets and Forms users become more productive. Over the last few months, we launched a number of developer-friendly features that made it easier to build, test, deploy and distribute add-ons. Some key capabilities include: </p>

<ul><li>Ability to <a href="http://googledevelopers.blogspot.com/2015/03/publishing-google-docs-add-ons-for.html?utm_campaign=apps-dwao-313&utm_source=gdbc&utm_medium=blog">publish</a> add-ons in Google Apps Marketplace</li>
<li>Ability to <a href="https://developers.google.com/apps-script/add-ons/domain-wide?utm_campaign=docs-add-ons-415&utm_source=gdbc&utm_medium=blog#bundling_with_an_existing_google_apps_marketplace_app">bundle your add-on with existing Apps Marketplace listing</a></li>
<li>Availability of Apps Script <a href="http://googleappsdeveloper.blogspot.com/2014/10/installable-triggers-in-add-ons-for.html">triggers</a> in add-ons</li>
<li><a href="https://developers.google.com/apps-script/add-ons/test?utm_campaign=docs-add-ons-415&utm_source=gdbc&utm_medium=blog">Testing tools for add-ons</a></li>
<li><a href="https://developers.google.com/apps-script/add-ons/publish?utm_campaign=docs-add-ons-415&utm_source=gdbc&utm_medium=blog">Using standalone scripts to publish add-ons</a></li>
<li>Option to use your own Google Developers console project</li>
<li><a href="http://googleappsdeveloper.blogspot.com/2014/12/speeding-up-htmlservice.html">Iframe sandbox</a> mode for faster UI</li></ul>

<p>With these features under our belt, we are ready to graduate add-ons out of developer preview. Starting today, any developer can publish an add-on. To ensure users find the best tools for them, every new add-on will undergo a <a href="https://developers.google.com/apps-script/add-ons/publish?utm_campaign=docs-add-ons-415&utm_source=gdbc&utm_medium=blog">review for adherence to our guidelines</a> before it&#8217;s available in the add-ons store. </p>

<p>We can&#8217;t wait to see what you will build!</p>

<h2 class='title' itemprop='name'>
<a href='http://googledevelopers.blogspot.in/2015/04/a-final-farewell-to-clientlogin-oauth.html' itemprop='url' title='A final farewell to ClientLogin, OAuth 1.0 (3LO), AuthSub, and OpenID 2.0 '>
A final farewell to ClientLogin, OAuth 1.0 (3LO), AuthSub, and OpenID 2.0 
</a></h2>
<div class='post-header'>
<div class='published'><span class='caption'>Posted:</span>
<span class='publishdate' itemprop='datePublished'>Tuesday, April 21, 2015</span>
</div>
<div class='share'>
<span class='gplus-wrapper'>
<div class='g-plusone' data-href='http://googledevelopers.blogspot.com/2015/04/a-final-farewell-to-clientlogin-oauth.html' data-size='medium'></div>
</span>
<a class='twitter-share-button' data-lang='en' data-url='http://googledevelopers.blogspot.com/2015/04/a-final-farewell-to-clientlogin-oauth.html' href='https://twitter.com/share'>Tweet</a>
<iframe allowTransparency='true' frameborder='0' scrolling='no' src='http://www.facebook.com/plugins/like.php?href=http://googledevelopers.blogspot.com/2015/04/a-final-farewell-to-clientlogin-oauth.html&layout=button_count&show_faces=false&width=77&action=like&colorscheme=light&height=20' style='border:none; overflow:hidden; width:77px; height:20px;'></iframe>
</div>
</div>
<div class='post-content' itemprop='articleBody'>
<p><i>Posted by William Denniss, Product Manager, Identity and Authentication</i></p>

<p>Support for ClientLogin, OAuth 1.0 (3LO<sup>1</sup>), AuthSub, and OpenID 2.0 has ended, and the shutdown process has begun. Clients attempting to use these services will begin to fail and must be migrated to <a href="https://developers.google.com/accounts/docs/OAuth2?utm_campaign=oauth-415&utm_source=gdbc&utm_medium=blog">OAuth 2.0</a> or <a href="https://developers.google.com/accounts/docs/OpenIDConnect?utm_campaign=oauth-415&utm_source=gdbc&utm_medium=blog">OpenID Connect</a> immediately.</p>

<p>To migrate a sign-in system, the easiest path is to use the <a href="https://developers.google.com/identity/sign-in/?utm_campaign=oauth-415&utm_source=gdbc&utm_medium=blog">Google Sign-in SDKs</a> (see the <a href="https://developers.google.com/identity/sign-in/auth-migration?utm_campaign=oauth-415&utm_source=gdbc&utm_medium=blog">migration documentation</a>). Google Sign-in is built on top of our standards-based OAuth 2.0 and OpenID Connect infrastructure and provides a single interface for authentication and authorization flows on Web, Android and iOS.  To migrate server API use, we recommend using one of our <a href="https://developers.google.com/api-client-library/?utm_campaign=oauth-415&utm_source=gdbc&utm_medium=blog">OAuth 2.0 client libraries</a>.</p>

<p>We are moving away from legacy authentication protocols, focusing our support on OpenID Connect and OAuth 2.0.  These modern open standards enhance the security of Google accounts, and are generally easier for developers to integrate with.</p>

<p><sup>1</sup>3LO stands for 3-legged OAuth where there's an end-user that provides consent. In contrast, 2-legged (2LO) correspond to Enterprise authorization scenarios such as organizational-wide policies control access. Both OAuth1 3LO and 2LO flows are deprecated, but this announcement is specific to OAuth1 3LO.</p>

<h2 class='title' itemprop='name'>
<a href='http://googledevelopers.blogspot.in/2015/04/works-with-google-cardboard-creativity.html' itemprop='url' title='Works with Google Cardboard: creativity plus compatibility'>
Works with Google Cardboard: creativity plus compatibility
</a></h2>
<div class='post-header'>
<div class='published'><span class='caption'>Posted:</span>
<span class='publishdate' itemprop='datePublished'>Thursday, April 16, 2015</span>
</div>
<div class='share'>
<span class='gplus-wrapper'>
<div class='g-plusone' data-href='http://googledevelopers.blogspot.com/2015/04/works-with-google-cardboard-creativity.html' data-size='medium'></div>
</span>
<a class='twitter-share-button' data-lang='en' data-url='http://googledevelopers.blogspot.com/2015/04/works-with-google-cardboard-creativity.html' href='https://twitter.com/share'>Tweet</a>
<iframe allowTransparency='true' frameborder='0' scrolling='no' src='http://www.facebook.com/plugins/like.php?href=http://googledevelopers.blogspot.com/2015/04/works-with-google-cardboard-creativity.html&layout=button_count&show_faces=false&width=77&action=like&colorscheme=light&height=20' style='border:none; overflow:hidden; width:77px; height:20px;'></iframe>
</div>
</div>
<div class='post-content' itemprop='articleBody'>
<p><i>Posted by Andrew Nartker, Product Manager, Google Cardboard</i></p>

<p>All of us is greater than any single one of us. That&#8217;s why we open sourced the Cardboard viewer design on day one. And why we&#8217;ve been working on virtual reality (VR) tools for manufacturers and developers ever since. We want to make VR better together, and the community continues to inspire us.</p>

<p>For example: what began with cardboard, velcro and some lenses has become a part of <a href="http://news.mattel.com/news/mattel-collaborates-with-google-to-reimagine-the-iconic-view-master-toy">toy fairs</a> and <a href="http://www.dreamsincarnated.com/store">art shows</a> and <a href="http://www.google.com/get/cardboard/sundance.html">film festivals</a> all over the world. There are also hundreds of Cardboard apps on Google Play, including <a href="https://play.google.com/store/apps/details?id=com.volvo.volvoreality&hl=en">test drives</a>, <a href="https://play.google.com/store/apps/details?id=com.Cosmic.rollercoaster">roller coaster rides</a>, and <a href="https://play.google.com/store/apps/details?id=com.jauntvr.preview.tnf">mountain climbs</a>. And people keep finding new ways to bring VR into their daily lives&#8212;from <a href="http://www.latimes.com/business/la-fi-vr-college-tours-20150320-story.html">campus tours</a> to <a href="https://www.youtube.com/watch?v=LV7Lzoovrek">marriage proposals</a> to <a href="http://www.virgin.com/travel/virgin-holidays-to-let-customers-try-before-they-buy">vacation planning</a>.</p>

<p>It&#8217;s what we dreamed about when we folded our first piece of cardboard, and combined it with a smartphone: a VR experience for everyone! And less than a year later, there&#8217;s a tremendous diversity of VR viewers and apps to choose from. To keep this creativity going, however, we also need to invest in compatibility. That&#8217;s why we&#8217;re announcing a new program called Works with Google Cardboard.</p>

<p>At its core, the program enables any Cardboard viewer to work well with any Cardboard app. And the result is more awesome VR for all of us.</p>

<div class="separator" style="clear: both; text-align: center;"><a href="http://3.bp.blogspot.com/-WlcZnNS7I7s/VS_nzPFBoRI/AAAAAAAAArs/sSxn-WQWJU8/s1600/image01.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://3.bp.blogspot.com/-WlcZnNS7I7s/VS_nzPFBoRI/AAAAAAAAArs/sSxn-WQWJU8/s640/image01.png" /></a></div>

<h3>For makers: compatibility tools, and a certification badge</h3>

<p>These days you can find Cardboard viewers made from all sorts of materials&#8212;<a href="http://www.lgnewsroom.com/newsroom/contents/64924">plastic</a>, <a href="https://plus.google.com/113069089238945334645/posts/7sce7LuWwMJ">wood</a>, <a href="http://www.knoxlabs.com/products/knox-aluminum">metal</a>, even <a href="http://www.cnet.com/how-to/how-to-make-a-vr-headset-out-of-a-pizza-box/">pizza boxes</a>. The challenge is that each viewer may have slightly different optics and dimensions, and apps actually need this info to deliver a great experience. That&#8217;s why, as part of today&#8217;s program, we&#8217;re releasing a new tool that configures any viewer for every Cardboard app, automatically.</p>

<p>As a manufacturer, all you need to do is define your viewer&#8217;s key parameters (like focal length, input type, and inter-lens distance), and you&#8217;ll get a QR code to place on your device. Once a user scans this code using the <a href="https://play.google.com/store/apps/details?id=com.google.samples.apps.cardboarddemo&hl=en">Google Cardboard app</a>, all their other Cardboard VR experiences will be optimized for your viewer. And that&#8217;s it.</p>

<p>Starting today, manufacturers can also apply for a program certification badge. This way potential users will know, at a glance, that a VR viewer works great with Cardboard apps and games. Visit the <a href="http://www.google.com/get/cardboard/manufacturers.html">Cardboard website</a> to get started.</p>

<div class="separator" style="clear: both; text-align: center;"><a href="http://2.bp.blogspot.com/-Vu32JB-SD9c/VS_l9Qow87I/AAAAAAAAArY/93DXHeze-xo/s1600/image02.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://2.bp.blogspot.com/-Vu32JB-SD9c/VS_l9Qow87I/AAAAAAAAArY/93DXHeze-xo/s400/image02.png" /></a><div><i>The GoggleTech C1-Glass viewer works with Google Cardboard</i></div></div>

<h3>For developers: design guidelines and SDK updates</h3>

<p>Whether you&#8217;re building your first VR app, or you&#8217;ve done it ten times before, creating an immersive experience comes with a unique set of design questions like, &#8220;How should I orient users at startup?&#8221; Or &#8220;How do menus even work in VR?&#8221;</p>

<p>We&#8217;ve explored these questions (and many more) since launch, and today we&#8217;re sharing our initial learnings with the developer community. Our new <a href="http://google.com/design/spec-vr">design guidelines</a> focus on overall usability, as well as common VR pitfalls, so take a look and let us know your thoughts.</p>

<p>Of course, we want to make it easier to design <i>and</i> build great apps. So today we're also updating the <a href="http://www.google.com/get/cardboard/developers.html">Cardboard SDKs</a> for Android and Unity&#8212;including improved head tracking and drift correction. In addition, both SDKs support the Works with Google Cardboard program, so all your apps will play nice with all certified VR viewers.</p>

<h3>For users: apps + viewers = choices</h3>

<p>The number of Cardboard apps has quickly grown from dozens to hundreds, so we&#8217;re expanding our Google Play collection to help you find high-quality apps even faster. New categories include <a href="https://play.google.com/store/apps/collection/promotion_300156b_cardboard_music_video">Music and Video</a>, <a href="https://play.google.com/store/apps/collection/promotion_300156a_cardboard_games">Games</a>, and <a href="https://play.google.com/store/apps/collection/promotion_3001568_cardboard_experiences">Experiences</a>. Whether you&#8217;re <a href="https://play.google.com/store/apps/details?id=com.bickhofe.waa">blasting asteroids</a>, or reliving the Saturday Night Live <a href="https://play.google.com/store/apps/details?id=com.shakingearthdigital.vrsecardboard">40th Anniversary Special</a>, there&#8217;s plenty to explore on Google Play.</p>

<div class="separator" style="clear: both; text-align: center;"><a href="http://1.bp.blogspot.com/-zXKHsVrePzQ/VS_nBsocKXI/AAAAAAAAArk/LbcncDsGaE0/s1600/image00.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://1.bp.blogspot.com/-zXKHsVrePzQ/VS_nBsocKXI/AAAAAAAAArk/LbcncDsGaE0/s400/image00.png" /></a><div><i>New collections of Cardboard apps on Google Play
</i></div></div>

<p>Today&#8217;s Works with Google Cardboard announcement means you&#8217;ll get the same great VR experience across a wide selection of Cardboard viewers. <a href="http://www.google.com/get/cardboard/get-cardboard.html">Find the viewer</a> that fits you best, and then fire up your favorite apps.</p>

<h3>For the future: Thrive Audio and Tilt Brush are joining the Google family</h3>

<p>Most of today&#8217;s VR experiences focus on what you see, but what you hear is just as important. That&#8217;s why we&#8217;re excited to welcome the <a href="http://www.mee.tcd.ie/thrive/">Thrive Audio team</a> from the School of Engineering in Trinity College Dublin to Google. With their ambisonic surround sound technology, we can start bringing immersive audio to VR.</p>

<p>In addition, we&#8217;re thrilled to have the <a href="http://www.tiltbrush.com/">Tilt Brush team</a> join our family. With its innovative approach to 3D painting, Tilt Brush won last year&#8217;s Proto Award for Best Graphical User Interface. We&#8217;re looking forward to having them at Google, and building great apps together.</p>

<p>Ultimately, today&#8217;s updates are about making VR better together. Join the fold, and let&#8217;s have some fun.</p>

<p><i>Posted by Angana Ghosh, Product Manager, Google Fit</i></p>

Find out more about <a href="https://developers.google.com/fit/overview?utm_campaign=fit-chal-415&utm_source=gdbc&utm_medium=blog">integrating Google Fit into your app</a>.</p>

</a></h2>

<p><i>Posted by Cheryl Simon Retzlaff, Software Engineer on the Realtime API team</i></p>

<p><i>Originally <a href="http://googleappsdeveloper.blogspot.com/2015/04/the-realtime-api-in-memory-mode-debug.html">posted</a> to the Google Apps Developer blog</i></p>

<p>Real-time collaboration is a powerful feature for getting work done inside Google docs. We extended that functionality with the Realtime API to enable you to create Google-docs style collaborative applications with minimal effort.</p>

<p>Integration of the API becomes even easier with a new <a href="https://developers.google.com/drive/realtime/in-memory?utm_campaign=realtime-api-415&utm_source=gdbc&utm_medium=blog">in memory mode</a>, which allows you to manipulate a Realtime document using the standard API without being connected to our servers.  No user login or authorization is required.  This is great for building applications where Google login is optional, writing tests for your app, or experimenting with the API before configuring auth.</p>

<p>The <a href="https://developers.google.com/drive/realtime/debugger?utm_campaign=realtime-api-415&utm_source=gdbc&utm_medium=blog">Realtime debug console</a> lets you view, edit and debug a Realtime model.  To launch the debugger, simply execute <code>gapi.drive.realtime.debug();</code> in the JavaScript console in Chrome.</p>

<div class="separator" text-align: center;"><a href="http://3.bp.blogspot.com/-WjGi4nd2OTE/VSwXSIKEOfI/AAAAAAAABN0/T9unxegfbks/s1600/image00.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://3.bp.blogspot.com/-WjGi4nd2OTE/VSwXSIKEOfI/AAAAAAAABN0/T9unxegfbks/s640/image00.png" /></a></div>

<p>Finally, we have refreshed the developer guides to make it easier for you to learn about the API as a new or advanced user.  Check them out at <a href="https://developers.google.com/drive/realtime?utm_campaign=realtime-api-415&utm_source=gdbc&utm_medium=blog">https://developers.google.com/drive/realtime</a>.</p>

<p>For details on these and other recent features, see the <a href="https://developers.google.com/drive/realtime/release-notes?utm_campaign=realtime-api-415&utm_source=gdbc&utm_medium=blog">release note</a>.</p>

</div>
<div class='share'>

<div class='post-content' itemprop='articleBody'>
<p><i>Posted by Jon Simantov, Fun Propulsion Labs at Google</i></p>

<p><i>Originally <a href="http://google-opensource.blogspot.com/2015/04/we-throw-pie-with-little-help-from-our.html ">posted</a> to the Google Open Source blog</i></p>

<p>Fun Propulsion Labs at Google* is back today with some new releases for game developers. We&#8217;ve updated Pie Noon (our <a href="https://github.com/google/pienoon">open source</a> Android TV game) with networked multi-screen action, and we&#8217;ve also added some delicious new libraries we&#8217;ve been baking since the original release: the Pindrop audio library and the Motive animation system.</p>

<h3>Pie Noon multi-screen action</h3>

<p>Got an Android TV and up to 4 friends with Android phones or tablets? You&#8217;re ready for some strategic multi-player mayhem in this updated game mode. Plan your next move in secret on your Android phone: will you throw at an opponent, block an incoming attack, or take the risky approach and wait for a larger pie? Choose your target and action, then watch the Android TV to see what happens!</p>

<!--[Interactive video]  --><iframe allowfullscreen="" frameborder="0" height="315" src="https://www.youtube.com/embed/aR5WQaq70fw" style="box-shadow: 3px 10px 18px 1px #999; display: block; margin-bottom:1em; margin-left: 70px;" width="560"></iframe><br/>

<p>We used the <a href="https://developers.google.com/games/services/android/nearby?utm_campaign=pienoon-415&utm_source=gdbc&utm_medium=blog">NearbyConnections API</a> from the most recent version of <a href="https://developers.google.com/games/services/?utm_campaign=pienoon-415&utm_source=gdbc&utm_medium=blog">Google Play Games services</a> to easily connect smartphones to your Android TV and turn our original Pie Noon party game into a game of turn-based strategy. You can grab the <a href="https://play.google.com/store/apps/details?id=com.google.fpl.pie_noon&hl=en">latest version of Pie Noon</a> from Google Play to try it out, or crack open the <a href="https://github.com/google/pienoon">source code</a> and take a look at how we used <a href="http://google.github.io/flatbuffers/">FlatBuffers</a> to encode data across the network in a fast, portable, bandwidth-efficient way.</p>

<h3>Pindrop: an open source game audio library</h3>

<p>Pindrop is a cross-platform C++ library for managing your in-game audio. It supports cross compilation to Android, Linux, iOS and OSX. An early version of this code was part of the first Pie Noon release, but it&#8217;s now available as a separate library that you can use in your own games. Pindrop handles loading and unloading sound banks, tracking sound locations and listeners, prioritization of your audio channels, and more.</p>

<p>Pindrop is built on top of several other pieces of open source technology:</p>
<ul><li><a href="https://www.libsdl.org/projects/SDL_mixer/">SDL Mixer</a> is used as a backend for actually playing the audio.</li>
<li>The loading of data and configuration files is handled by our serialization library, <a href="http://google.github.io/flatbuffers/">FlatBuffers</a>.</li>
<li>Our own math library, <a href="http://google.github.io/mathfu/">MathFu</a>, is used for a number of under-the-hood calculations.</li></ul>

<p>You can download the latest open source release from our <a href="https://github.com/google/pindrop">GitHub</a> page. Documentation is available <a href="http://google.github.io/pindrop">here</a> and a sample project is included in the source tree. Please feel free to post any questions in our <a href="https://groups.google.com/forum/#!forum/pindrop">discussion list</a>.</p>

<h3>Motive: an open source animation system</h3>
<p>The Motive animation system can breathe life into your static scenes. It does this by applying motion to simple variables. For example, if you&#8217;d like a flashlight to shine on a constantly-moving target, Motive can animate the flashlight so that it moves smoothly yet responsively.</p>

<p>Motive animates both <a href="http://en.wikipedia.org/wiki/Spline_(mathematics)">spline-based motion</a> and <a href="http://en.wikipedia.org/wiki/Procedural_animation">procedural motion</a>. These types of motion are not technically difficult, but they are artistically subtle. It's easy to get the math wrong. It's easy to end up with something that moves as required but doesn't quite feel right. Motive does the math and lets you focus on the feeling.</p>

<p>Motive is scalable. It's designed to be extremely fast. It also has a tight memory footprint -- smaller than traditional animation compression -- that's based on <a href="https://www.youtube.com/watch?v=6ON6fYO4FbY">Dual Cubic Splines</a>. Our hope is that you might consider using Motive as a high-performance back-end to your existing full-featured animation systems. </p>

<p>This initial release of Motive is feature-light since we focused our early efforts on doing something simple very quickly. We support procedural and spline-based animation, but we don't yet support data export from animation packages like <a href="http://www.blender.org/">Blender</a> or <a href="http://www.autodesk.com/products/maya/overview">Maya</a>. Motive 1.0 is suitable for props -- trees, cameras, extremities -- but not fully rigged character models.  Like all FPL technologies, Motive is <a href="https://github.com/google/motive">open source</a> and cross-platform. Please check out the <a href="https://groups.google.com/forum/#!forum/motive-anim">discussion list</a>, too.</p>

Sun Surveyor brings augmented reality to photographers using Google Maps APIs
</a></h2>
<div class='post-header'>
<div class='published'><span class='caption'>Posted:</span>
<span class='publishdate' itemprop='datePublished'>Tuesday, April 7, 2015</span>
</div>
<div class='share'>

<p><i>Originally <a href="http://googlegeodevelopers.blogspot.com/2015/04/sun-surveyor-brings-augmented-reality.html">posted</a> to the Google Geo Developers blog</i></p>

<p><i><b>Editor&#8217;s note:</b> This post is written by Adam Ratana, developer of Sun Surveyor. Read how Sun Surveyor is using <a href="https://developers.google.com/maps/?utm_campaign=sunsurveyor-415&utm_source=gdbc&utm_medium=blog">Google Maps APIs</a> to help photographers capture the perfect photo. </i></p>

<p><i>
Posted by Adam Ratana, developer of Sun Surveyor</i></p>

<p>I&#8217;m a photography enthusiast, and I&#8217;m always looking for ways to improve my work. That&#8217;s what led me to develop <a href="http://www.sunsurveyor.com/">Sun Surveyor</a>, an <a href="https://itunes.apple.com/us/app/sun-surveyor/id525176875?mt=8&amp;ign-mpt=uo%3D4">iOS</a> and <a href="https://play.google.com/store/apps/details?id=com.ratana.sunsurveyor">Android</a> app that uses <a href="https://developers.google.com/maps/?utm_campaign=sunsurveyor-415&utm_source=gdbc&utm_medium=blog">Google Maps APIs</a> to visualize the location of the sun and the moon anywhere in the world. The app makes it easy to figure out when the natural lighting will be just right &#8212; and get the ideal shot.</p>

<!--[Interactive video]  --><iframe allowfullscreen="" frameborder="0" height="315" src="https://www.youtube.com/embed/88rG6FHEeUg" style="box-shadow: 3px 10px 18px 1px #999; display: block; margin-bottom:1em; margin-left: 70px;" width="560"></iframe>

<p>Sun Surveyor uses augmented reality to overlay the paths of the sun and moon on a camera&#8217;s view, so you can see where in the sky they&#8217;ll be at a specific time and place. Photographers can use it to plan their shots ahead of time, and businesses can use it to gauge things like how best to align solar panels to make the most efficient use of the angle of the sun.</p>

<p>The app uses multiple Google Maps APIs, including the <a href="https://developers.google.com/maps/documentation/elevation/?utm_campaign=sunsurveyor-415&utm_source=gdbc&utm_medium=blog">Elevation API</a>, the <a href="https://developers.google.com/maps/documentation/timezone/?utm_campaign=sunsurveyor-415&utm_source=gdbc&utm_medium=blog">Time Zone API</a>, the <a href="https://developers.google.com/maps/documentation/ios/?utm_source=welovemapsdevelopersutm_campaign=mdr-motw">Google Maps SDK for iOS</a> and the <a href="https://developers.google.com/maps/documentation/android/?utm_campaign=sunsurveyor-415&utm_source=gdbc&utm_medium=blog">Google Maps Android API</a>. The Android API, which includes Street View, was particularly helpful. It allowed me to overlay the path of the sun and moon on any Street View location anywhere in the world. For programming details, see this <a href="http://googlegeodevelopers.blogspot.com/2013/09/sun-surveyor-shines-with-street-view.html">blog post</a>.</p>

<p>The following screen captures give you a sense of how the app works. They show overlays on top of the iconic Half Dome in Yosemite National Park. The first shows the paths of the sun (yellow line) and moon (blue line) over an aerial view of Yosemite Valley. The green line shows the distance between the photographer and the object to be photographed &#8212; in this case, Half Dome.</p>

<div class="separator" style="clear: both; text-align: center;">
<a href="http://4.bp.blogspot.com/-VSi3X9gnPCE/VSLxxE-glnI/AAAAAAAAAQI/vM5sxyEYP7I/s1600/Screen%2BShot%2B2015-04-06%2Bat%2B1.31.42%2BPM.png" imageanchor="1" style="clear: left; float: left; margin-bottom: 1em; margin-right: 1em;"><img border="0" src="http://4.bp.blogspot.com/-VSi3X9gnPCE/VSLxxE-glnI/AAAAAAAAAQI/vM5sxyEYP7I/s1600/Screen%2BShot%2B2015-04-06%2Bat%2B1.31.42%2BPM.png" height="401" width="640" /></a></div>

<p>This next screen capture shows how the app looks when in Street View mode. Again, the yellow line shows the sun&#8217;s path, and the blue line shows the moon&#8217;s path. The green line represents the horizon. You can see how the app lets you plan the right time to get a shot of the sun behind Half Dome: in this particular instance, 8:06 am.</p>

<div class="separator" style="clear: both; text-align: center;">
<a href="http://1.bp.blogspot.com/-_E2cnfuMP50/VSLx9AVTJSI/AAAAAAAAAQQ/2F8jsot-YtA/s1600/Screen%2BShot%2B2015-04-06%2Bat%2B1.32.08%2BPM.png" imageanchor="1" style="clear: left; float: left; margin-bottom: 1em; margin-right: 1em;"><img border="0" src="http://1.bp.blogspot.com/-_E2cnfuMP50/VSLx9AVTJSI/AAAAAAAAAQQ/2F8jsot-YtA/s1600/Screen%2BShot%2B2015-04-06%2Bat%2B1.32.08%2BPM.png" height="400" width="640" /></a></div>

<p>Nearly 500,000 people around the world have downloaded the free version of Sun Surveyor, and many have paid for the full edition. They&#8217;re taking remarkable photos as a result, and what started as a hobby for me has turned into a business &#8212; thanks to Google Maps APIs.</p>

<h2 class='title' itemprop='name'>
<a href='http://googledevelopers.blogspot.in/2015/04/ho-ho-ho-googles-santa-tracker-is-now.html' itemprop='url' title='Ho Ho Ho! Google&#39;s Santa Tracker is now open source'>
Ho Ho Ho! Google's Santa Tracker is now open source
</a></h2>
<div class='post-header'>
<div class='published'><span class='caption'>Posted:</span>
<span class='publishdate' itemprop='datePublished'>Tuesday, April 7, 2015</span>
</div>
<div class='share'>
<span class='gplus-wrapper'>
<div class='g-plusone' data-href='http://googledevelopers.blogspot.com/2015/04/ho-ho-ho-googles-santa-tracker-is-now.html' data-size='medium'></div>
</span>
<a class='twitter-share-button' data-lang='en' data-url='http://googledevelopers.blogspot.com/2015/04/ho-ho-ho-googles-santa-tracker-is-now.html' href='https://twitter.com/share'>Tweet</a>
<iframe allowTransparency='true' frameborder='0' scrolling='no' src='http://www.facebook.com/plugins/like.php?href=http://googledevelopers.blogspot.com/2015/04/ho-ho-ho-googles-santa-tracker-is-now.html&layout=button_count&show_faces=false&width=77&action=like&colorscheme=light&height=20' style='border:none; overflow:hidden; width:77px; height:20px;'></iframe>
</div>
</div>
<div class='post-content' itemprop='articleBody'>
<p><i>Posted by <a href="https://plus.google.com/+AnkurKotwal">Ankur Kotwal</a>, Software Engineer</i></p>

<p>The holiday spirit is about giving and though we&#8217;re early into April, we&#8217;re still in that spirit. Today, we&#8217;re announcing that Google's Santa Tracker is now open source on GitHub at <a href="https://github.com/google/santa-tracker-web">google/santa-tracker-web</a> and <a href="https://github.com/google/santa-tracker-android">google/santa-tracker-android</a>. Now you can see how we&#8217;ve used many of our developer products to build a fun and engaging experience that runs across the web and Android.</p>

<div class="separator" style="clear: both; text-align: center;"><a href="http://1.bp.blogspot.com/-9NBUWxqR_Sw/VSNhdfRk7vI/AAAAAAAAAqs/m5RPiwL0R_Y/s1600/image05.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://1.bp.blogspot.com/-9NBUWxqR_Sw/VSNhdfRk7vI/AAAAAAAAAqs/m5RPiwL0R_Y/s640/image05.png" /></a></div>

<p>Santa Tracker isn&#8217;t just about watching Santa&#8217;s progress as he delivers presents on December 24. Visitors can also have fun with the winter-inspired games and an interactive North Pole village while Santa prepares for his big journey throughout the holidays.</p>

<p>Below is a summary of what we&#8217;ve released as open source.</p>

<h3>Android app</h3>

<ul><li>The Santa Tracker Android app is a single APK, supporting all devices running Ice Cream Sandwich (4.0) and up. The source code for the app can be found <a href="https://github.com/google/santa-tracker-android">here</a>.</li>

<div class="separator" style="clear: both; text-align: center;"><a href="http://1.bp.blogspot.com/-3lCqBgwM9IA/VSNd_HhjH7I/AAAAAAAAApw/VkcOSJLNh8A/s1600/image03.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://1.bp.blogspot.com/-3lCqBgwM9IA/VSNd_HhjH7I/AAAAAAAAApw/VkcOSJLNh8A/s400/image03.png" /></a></div>

<li><a href="https://github.com/google/santa-tracker-android/tree/master/app/village">Santa&#8217;s village</a> is a canvas-based graphical launcher for videos, games and the tracker. In order to span 10,000 pixels in width, the village uses the Android resource hierarchy in a <a href="https://github.com/google/santa-tracker-android/blob/master/app/village/src/main/res/values-xxhdpi/village_images.xml">unique way</a> to scale graphics without needing assets for multiple density buckets, thereby reducing the APK size.</li>
<li>Games on Santa Tracker Android are built using a combination of technologies such as <a href="http://www.jbox2d.org/">JBox2D</a> (<a href="https://github.com/google/santa-tracker-android/tree/master/app/santa-tracker/src/main/java/com/google/android/apps/santatracker/games/gumball">gumball game</a>), Android view hierarchy (<a href="https://github.com/google/santa-tracker-android/tree/master/app/santa-tracker/src/main/java/com/google/android/apps/santatracker/games/matching">memory match game</a>) and OpenGL with a purpose-built rendering engine (<a href="https://github.com/google/santa-tracker-android/tree/master/app/santa-tracker/src/main/java/com/google/android/apps/santatracker/games/jetpack">jetpack game</a>).</li>
<li>To help with user engagement, the app uses the <a href="https://developers.google.com/app-indexing/webmasters/appindexingapi?utm_campaign=santatracker-415&utm_source=gdbc&utm_medium=blog">App Indexing API</a> to enable autocomplete support for Santa Tracker games from Google Search. This is done using <a href="https://github.com/google/santa-tracker-android/blob/2c4a620e90da009537f593e191219c438f1c5d93/app/santa-tracker/src/main/java/com/google/android/apps/santatracker/games/common/GameActivity.java#L142">deep linking</a>.</li></ul>

<!--[Interactive video]  --><iframe allowfullscreen="" frameborder="0" height="315" src="https://www.youtube.com/embed/kYLrK-gD2Yg" style="box-shadow: 3px 10px 18px 1px #999; display: block; margin-bottom:1em; margin-left: 70px;" width="560"></iframe>


<h3>Android Wear</h3>

<ul><li>The custom watch faces on Android Wear provide a personalized touch. Having Santa or one of his friendly elves tell the time brings a smile to all. <a href="https://www.youtube.com/watch?v=VkvHKtmsvYA">Building custom watch faces</a> is a lot of fun but providing a performant, battery friendly watch face requires <a href="http://android-developers.blogspot.com/2014/12/making-performant-watch-face.html">certain considerations</a>. The watch face source code can be found <a href="https://github.com/google/santa-tracker-android/blob/master/app/wearable/src/main/java/com/google/android/apps/santatracker/SantaWatchFaceService.java">here</a>.</li>
<li>Santa Tracker uses notifications to let users know when Santa has started his journey. The notifications are further <a href="https://github.com/google/santa-tracker-android/blob/master/app/wearable/src/main/java/com/google/android/apps/santatracker/WearNotificationService.java">enhanced to provide a great experience on wearables</a> using custom backgrounds and actions that deep link into the app.</li></ul>

<div class="separator" style="clear: both; text-align: center;"><a href="http://3.bp.blogspot.com/-zGvKRA2aHio/VSNeQjTzVvI/AAAAAAAAAqA/JV79iMVQs9U/s1600/image00.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://3.bp.blogspot.com/-zGvKRA2aHio/VSNeQjTzVvI/AAAAAAAAAqA/JV79iMVQs9U/s400/image00.png" /></a></div>

<h3>On the web</h3>

<div class="separator" style="clear: both; text-align: center;"><a href="http://4.bp.blogspot.com/-W3IsVvBXI1s/VSNhus5K6rI/AAAAAAAAAq0/KM7O3SzUTKM/s1600/image02.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://4.bp.blogspot.com/-W3IsVvBXI1s/VSNhus5K6rI/AAAAAAAAAq0/KM7O3SzUTKM/s640/image02.png" /></a></div>

<ul><li>Santa Tracker on the web was built using <a href="https://www.polymer-project.org">Polymer</a>, a powerful new library from the Chrome team based on Web Components. Santa Tracker&#8217;s use of Polymer demonstrates how easy it is to package code into reusable components. Every <a href="https://github.com/google/santa-tracker-web/tree/master/scenes">scene</a> in Santa's village (games, videos, and interactive pages) is a custom element, only loaded when needed, minimizing the startup cost of Santa Tracker.</li>
<li>Santa Tracker&#8217;s interactive and fun experience is built using the Web Animations API, <a href="https://w3c.github.io/web-animations/">a standardized JavaScript API</a> for unifying animated content - this is a huge leap up from using CSS animations. Web Animations can be <a href="http://updates.html5rocks.com/2014/05/Web-Animations---element-animate-is-now-in-Chrome-36">written imperatively</a>, are supported in all modern browsers <a href="https://github.com/web-animations/web-animations-js">via polyfills</a> and Polymer uses them internally to create its amazing material design effects. Examples of these animations can be found using <a href="https://github.com/google/santa-tracker-web/search?l=javascript&q=Animation">this GitHub search</a>.</li>

<div class="separator" style="clear: both; text-align: center;"><a href="http://4.bp.blogspot.com/-vfLshLXpQ0I/VSNeeonO8QI/AAAAAAAAAqQ/EQ6Ad-px3Ks/s1600/image04.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" src="http://4.bp.blogspot.com/-vfLshLXpQ0I/VSNeeonO8QI/AAAAAAAAAqQ/EQ6Ad-px3Ks/s400/image04.png" /></a></div>

<li>Santa believes in mobile first; this year's experience was built to be optimized for the mobile web, including a fully responsive design, touch gesture <a href="https://www.polymer-project.org/0.5/docs/polymer/touch.html">support</a> using Polymer, and new features like <a href="https://www.chromestatus.com/features/5398016231997440">meta-theme color</a> and the <a href="https://www.chromestatus.com/features/6488656873259008">application manifest</a> for <a href="https://developer.chrome.com/multidevice/android/installtohomescreen?utm_campaign=santatracker-415&utm_source=gdbc&utm_medium=blog">add to homescreen</a>.</li>
<li>To provide exceptional localization support, a new <a href="http://ebidel.github.io/i18n-msg/components/i18n-msg/">i18n-msg component</a> was built, a first for Web Components. Modeled after the <a href="https://developer.chrome.com/extensions/i18n?utm_campaign=santatracker-415&utm_source=gdbc&utm_medium=blog">Chrome extension i18n system</a>, it enables live page-refresh for development but has a build step for optimization.</li></ul>

<p>Now that the source code is also available, developers can see many of the parts that come together to make Santa Tracker. We hope that developers are inspired to make their own magical experiences.</p>

New Advanced services in Apps Script
</a></h2>
<div class='post-header'>
<div class='published'><span class='caption'>Posted:</span>
<span class='publishdate' itemprop='datePublished'>Monday, April 6, 2015</span>
</div>
<div class='share'>
<span class='gplus-wrapper'>

</span>

<div class='post-content' itemprop='articleBody'>
<p><i>Posted by <a href="https://plus.google.com/+KalyanReddyGoogle/posts">Kalyan Reddy</a>, Developer Programs Engineer</i></p>

<p><i>Originally <a href="http://googleappsdeveloper.blogspot.com/2015/04/new-advanced-services-in-apps-script.html">posted</a> on the Google Apps Developer blog</i></p>

<p>Apps Script includes many built-in Google services for major products like Gmail and Drive, and lately, we've been working to add other APIs that developers have been clamoring for as <a href="https://developers.google.com/apps-script/guides/services/advanced?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog">advanced Google services</a>. Today, we are launching seven more advanced services, including:</p>
<ul><li><a href="https://developers.google.com/apps-script/advanced/admin-sdk-groups-migration?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog">Admin SDK Groups Migration service</a></li>
<li><a href="https://developers.google.com/apps-script/advanced/admin-sdk-groups-settings?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog">Admin SDK Groups Settings service</a></li>
<li><a href="https://developers.google.com/apps-script/advanced/admin-sdk-license-manager?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog">Admin SDK License Manager service</a></li>
<li><a href="https://developers.google.com/apps-script/advanced/admin-sdk-reseller?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog">Admin SDK Reseller service</a></li>
<li><a href="https://developers.google.com/apps-script/advanced/apps-activity?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog">Apps Activity service</a></li>
<li><a href="https://developers.google.com/apps-script/advanced/doubleclick-campaigns?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog">DoubleClick Campaigns service</a></li>
<li><a href="https://developers.google.com/apps-script/advanced/plus?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog">Google+ service</a></li></ul>

<p>Like all other advanced services in Apps Script, they must first be <a href="https://developers.google.com/apps-script/guides/services/advanced?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog#enabling_advanced_services">enabled</a> before use. Once enabled, they are just as easy to use as built-in Apps Script services -- the editor provides autocomplete, and the authentication flow is handled automatically. </p>

<p>Here is a <a href="https://script.google.com/d/1I_3dFyzoTEW6qGd720EIBYoKKOMOdl0wEzcJlUpfg2aZFs_62qeGx6d-/edit?usp=sharing">sample</a> using the Apps Activity advanced service that shows how to get a list of users that have performed an action on a particular Google Drive file.</p>

<pre class="prettyprint">
function getUsersActivity() {
  var fileId = 'YOUR_FILE_ID_HERE';
  var pageToken;
  var users = {};
  do {
    var result = AppsActivity.Activities.list({
      'drive.fileId': fileId,
      'source': 'drive.google.com',
      'pageToken': pageToken
    });
    var activities = result.activities;
    for (var i = 0; i < activities.length; i++) {
      var events = activities[i].singleEvents;
      for (var j = 0; j < events.length; j++) {
        var event = events[j];
        users[event.user.name] = true;
      }
    }
    pageToken = result.nextPageToken;
  } while (pageToken);
  Logger.log(Object.keys(users));
}</pre>


<p>This function uses the <code>AppsActivity.Activities.list()</code> method, passing in the required parameters <code>drive.fileId</code> and <code>source</code>, and uses page tokens to get the full list of activities. The full list of parameters this method accepts can be found in the <a href="https://developers.google.com/google-apps/activity/v1/reference/?utm_campaign=advanced-services-415&utm_source=gdbc&utm_medium=blog">Apps Activity API's reference documentation</a>.</p>
