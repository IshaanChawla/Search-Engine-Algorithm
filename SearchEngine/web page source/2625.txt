                  <div>
                     <h2 class="title"><a name="freemarker"></a>1.&nbsp;Introduction to FreeMarker
                     </h2>
                  </div>
               </div>
            </div>
            <p>
               <em class="wordasword">FreeMarker</em>
               is a Java-based template engine which can be used in
               stand-alone or
               servlet-based Java programs.
               
            </p>
            <p>
               In
               FreeMarker
               you define templates, which are text files that contain
               the desired output,
               except that they contain placeholders like
               <code class="code">${name}</code>, and even some
               logic like conditionals, loops, etc. In your Java program
               you
               supply the actual
               values for these placeholders
               and the final
               output is
               generated based on
               this input.
               
            </p>
            <p>
               The input of templates is a bunch of named variables that you usually
               provide
               as a
               <code class="code">Map&lt;String, Object&gt;</code>
               (the
               <code class="code">Map</code>
               entries will be
               the variables) or as a JavaBean (the JavaBean
               properties will be the
               variables).
               The variable values can be simple
               strings, numbers and such primitive
               values,
               but also lists, maps, or
               arbitrary Java objects whose methods
               you can call from the template.
               Note that when accessing JavaBean
               properties,
               <code class="code">myObject.myProperty</code>
               syntax should be used instead of
               <code class="code">myObject.getMyProperty()</code>.
               
            </p>
            <p>
               The output of templates is written into a
               <code class="code">Writer</code>
               that you provide,
               so it can go into a HTTP response (for dynamic web
               pages), into a local
               file,
               into a
               <code class="code">String</code>, etc.
               
            </p>
            <p>
               It is configurable from where FreeMarker reads the templates;
               commonly
               used options are
               loading from a file-system directory,
               from the
               class-path, from
               the servlet context (<code class="code">WEB-INF/templates</code>
               or such),
               or even from a database table.
               It's also possible to "load"
               templates directly from
               <code class="code">String</code>
               objects.
               
            </p>
         </div>

                  <div>
                     <h2 class="title"><a name="freemarkertest"></a>4.&nbsp;Basic example
                     </h2>
                  </div>
               </div>
            </div>
            <p>
               Create a new Java project called
               <em class="wordasword">com.vogella.freemarker.first</em>. Create a new folder called
               <code class="filename">lib</code>
               and add the Freemarker library to it. Add this library to the
               classpath for your project.
               
            </p>
            <p>
               If you don't know how to achieve that, please see the
               <a class="ulink" href="http://www.vogella.com/tutorials/Eclipse/article.html" target="_top">Eclipse IDE Tutorial</a>
               for instructions on the required steps.
               
            </p>
            <p>
               Create a new folder called
               <code class="filename">templates</code>
               inside the folder of the
               <em class="wordasword">com.vogella.freemarker.first</em>
               package. Inside that, create the following file with name
               <code class="filename">helloworld.ftl</code>.
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-tag" style="color: #080">&lt;html&gt;</span>
<span class="hl-tag" style="color: #080">&lt;head&gt;</span>
  <span class="hl-tag" style="color: #080">&lt;title&gt;</span>${title}<span class="hl-tag" style="color: #080">&lt;/title&gt;</span>
<span class="hl-tag" style="color: #080">&lt;/head&gt;</span>
<span class="hl-tag" style="color: #080">&lt;body&gt;</span>
  <span class="hl-tag" style="color: #080">&lt;h1&gt;</span>${title}<span class="hl-tag" style="color: #080">&lt;/h1&gt;</span>

  <span class="hl-tag" style="color: #080">&lt;p&gt;</span>${exampleObject.name} by ${exampleObject.developer}<span class="hl-tag" style="color: #080">&lt;/p&gt;</span>

  <span class="hl-tag" style="color: #080">&lt;ul&gt;</span>
    <span class="hl-tag" style="color: #080">&lt;#list</span> <span class="hl-attribute" style="color: #7F0055">systems</span> <span class="hl-attribute" style="color: #7F0055">as</span> <span class="hl-attribute" style="color: #7F0055">system&gt;</span>
      <span class="hl-attribute" style="color: #7F0055">&lt;li&gt;${system_index</span> <span class="hl-attribute" style="color: #7F0055">+</span> <span class="hl-attribute" style="color: #7F0055">1}.</span> <span class="hl-attribute" style="color: #7F0055">${system.name}</span> <span class="hl-attribute" style="color: #7F0055">from</span> <span class="hl-attribute" style="color: #7F0055">${system.developer}&lt;/li&gt;</span>
    <span class="hl-attribute" style="color: #7F0055">&lt;/#list&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;/ul&gt;</span>

<span class="hl-attribute" style="color: #7F0055">&lt;/body&gt;</span>
<span class="hl-attribute" style="color: #7F0055">&lt;/html&gt;</span> </pre><p>
               
            </p>
            <p>
               Create the following class which demonstrates the usage of Java
               objects in
               templates.
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> com.vogella.freemarker.first;

<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> ValueExampleObject {

  <span class="hl-keyword">private</span> String name;
  <span class="hl-keyword">private</span> String developer;

  <span class="hl-keyword">public</span> ValueExampleObject(String name, String developer) {
    <span class="hl-keyword">this</span>.name = name;
    <span class="hl-keyword">this</span>.developer = developer;
  }

  <span class="hl-keyword">public</span> String getName() {
    <span class="hl-keyword">return</span> name;
  }

  <span class="hl-keyword">public</span> String getDeveloper() {
    <span class="hl-keyword">return</span> developer;
  }

} </pre><p>
               
            </p>
            <p>
               Create the following class which creates the input for this
               template and
               creates the output.
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> com.vogella.freemarker.first;

<span class="hl-keyword">import</span> java.io.File;
<span class="hl-keyword">import</span> java.io.FileWriter;
<span class="hl-keyword">import</span> java.io.OutputStreamWriter;
<span class="hl-keyword">import</span> java.io.Writer;
<span class="hl-keyword">import</span> java.util.ArrayList;
<span class="hl-keyword">import</span> java.util.HashMap;
<span class="hl-keyword">import</span> java.util.List;
<span class="hl-keyword">import</span> java.util.Locale;
<span class="hl-keyword">import</span> java.util.Map;

<span class="hl-keyword">import</span> freemarker.template.Configuration;
<span class="hl-keyword">import</span> freemarker.template.Template;
<span class="hl-keyword">import</span> freemarker.template.TemplateExceptionHandler;
<span class="hl-keyword">import</span> freemarker.template.Version;

<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> MainTest {

  <span class="hl-keyword">public</span> <span class="hl-keyword">static</span> <span class="hl-keyword">void</span> main(String[] args) <span class="hl-keyword">throws</span> Exception {

    <em class="hl-comment" style="color: #080">// 1. Configure FreeMarker</em>
    <em class="hl-comment" style="color: #080">//</em>
    <em class="hl-comment" style="color: #080">// You should do this ONLY ONCE, when your application starts,</em>
    <em class="hl-comment" style="color: #080">// then reuse the same Configuration object elsewhere.</em>
    
    Configuration cfg = <span class="hl-keyword">new</span> Configuration();
    
    <em class="hl-comment" style="color: #080">// Where do we load the templates from:</em>
    cfg.setClassForTemplateLoading(MainTest.<span class="hl-keyword">class</span>, <span class="hl-string" style="color:blue">"templates"</span>);
    
    <em class="hl-comment" style="color: #080">// Some other recommended settings:</em>
    cfg.setIncompatibleImprovements(<span class="hl-keyword">new</span> Version(<span class="hl-number">2</span>, <span class="hl-number">3</span>, <span class="hl-number">20</span>));
    cfg.setDefaultEncoding(<span class="hl-string" style="color:blue">"UTF-8"</span>);
    cfg.setLocale(Locale.US);
    cfg.setTemplateExceptionHandler(TemplateExceptionHandler.RETHROW_HANDLER);

    <em class="hl-comment" style="color: #080">// 2. Proccess template(s)</em>
    <em class="hl-comment" style="color: #080">//</em>
    <em class="hl-comment" style="color: #080">// You will do this for several times in typical applications.</em>
    
    <em class="hl-comment" style="color: #080">// 2.1. Prepare the template input:</em>
    
    Map&lt;String, Object&gt; input = <span class="hl-keyword">new</span> HashMap&lt;String, Object&gt;();
    
    input.put(<span class="hl-string" style="color:blue">"title"</span>, <span class="hl-string" style="color:blue">"Vogella example"</span>);

    input.put(<span class="hl-string" style="color:blue">"exampleObject"</span>, <span class="hl-keyword">new</span> ValueExampleObject(<span class="hl-string" style="color:blue">"Java object"</span>, <span class="hl-string" style="color:blue">"me"</span>));
    
    List&lt;ValueExampleObject&gt; systems = <span class="hl-keyword">new</span> ArrayList&lt;ValueExampleObject&gt;();
    systems.add(<span class="hl-keyword">new</span> ValueExampleObject(<span class="hl-string" style="color:blue">"Android"</span>, <span class="hl-string" style="color:blue">"Google"</span>));
    systems.add(<span class="hl-keyword">new</span> ValueExampleObject(<span class="hl-string" style="color:blue">"iOS States"</span>, <span class="hl-string" style="color:blue">"Apple"</span>));
    systems.add(<span class="hl-keyword">new</span> ValueExampleObject(<span class="hl-string" style="color:blue">"Ubuntu"</span>, <span class="hl-string" style="color:blue">"Canonical"</span>));
    systems.add(<span class="hl-keyword">new</span> ValueExampleObject(<span class="hl-string" style="color:blue">"Windows7"</span>, <span class="hl-string" style="color:blue">"Microsoft"</span>));
    input.put(<span class="hl-string" style="color:blue">"systems"</span>, systems);
    
    <em class="hl-comment" style="color: #080">// 2.2. Get the template</em>

    Template template = cfg.getTemplate(<span class="hl-string" style="color:blue">"helloworld.ftl"</span>);
      
    <em class="hl-comment" style="color: #080">// 2.3. Generate the output</em>

    <em class="hl-comment" style="color: #080">// Write output to the console</em>
    Writer consoleWriter = <span class="hl-keyword">new</span> OutputStreamWriter(System.out);
    template.process(input, consoleWriter);

    <em class="hl-comment" style="color: #080">// For the sake of example, also write output into a file:</em>
    Writer fileWriter = <span class="hl-keyword">new</span> FileWriter(<span class="hl-keyword">new</span> File(<span class="hl-string" style="color:blue">"output.html"</span>));
    <span class="hl-keyword">try</span> {
      template.process(input, fileWriter);
    } <span class="hl-keyword">finally</span> {
      fileWriter.close();
    }

  }
} </pre><p>

               <div>
                  <div>
                     <h2 class="title"><a name="freemarkertricks"></a>5.&nbsp;Useful FTL tricks
                     </h2>
                  </div>

            </div>
            <div class="section">
               <div class="titlepage">
                  <div>
                     <div>
                        <h3 class="title"><a name="freemarkertricks_macro"></a>5.1.&nbsp;Reuse common template fragments
                        </h3>
                     </div>
                  </div>
               </div>
               <p>
                  When you find yourself copy-pasting common parts between templates a lot,
                  you should probably use macros.
                  
               </p>
               <p>
                  Continuing our last example, create a new folder called
                  <code class="filename">lib</code>
                  inside the
                  <code class="filename">templates</code>
                  directory, and there create a file called
                  <code class="filename">utils.ftl</code>, with
                  this content:
                  
               </p>
               <p>
                  
               </p><pre class="programlisting"><span class="hl-tag" style="color: #080">&lt;#macro</span> <span class="hl-attribute" style="color: #7F0055">page&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;html&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;head&gt;</span>
    <span class="hl-attribute" style="color: #7F0055">&lt;title&gt;${title}&lt;/title&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;/head&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;body&gt;</span>
    <span class="hl-attribute" style="color: #7F0055">&lt;h1&gt;${title}&lt;/h1&gt;</span>
  
    <span class="hl-attribute" style="color: #7F0055">&lt;#--</span> <span class="hl-attribute" style="color: #7F0055">This</span> <span class="hl-attribute" style="color: #7F0055">processes</span> <span class="hl-attribute" style="color: #7F0055">the</span> <span class="hl-attribute" style="color: #7F0055">enclosed</span> <span class="hl-attribute" style="color: #7F0055">content:</span>  <span class="hl-attribute" style="color: #7F0055">--&gt;</span>
    <span class="hl-attribute" style="color: #7F0055">&lt;#nested&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;/body&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;/html&gt;</span>
<span class="hl-attribute" style="color: #7F0055">&lt;/#macro&gt;</span>

<span class="hl-attribute" style="color: #7F0055">&lt;#macro</span> <span class="hl-attribute" style="color: #7F0055">otherExample</span> <span class="hl-attribute" style="color: #7F0055">p1</span> <span class="hl-attribute" style="color: #7F0055">p2&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;p&gt;The</span> <span class="hl-attribute" style="color: #7F0055">parameters</span> <span class="hl-attribute" style="color: #7F0055">were:</span> <span class="hl-attribute" style="color: #7F0055">${p1},</span> <span class="hl-attribute" style="color: #7F0055">${p2}&lt;/p&gt;</span>
<span class="hl-attribute" style="color: #7F0055">&lt;/#macro&gt;</span> </pre><p>
                  
               </p>
               <p>
                  Now you can simplify
                  <code class="filename">helloworld.ftl</code>
                  like this:
                  
               </p>
               <p>
                  
               </p><pre class="programlisting"><span class="hl-tag" style="color: #080">&lt;#import</span> <span class="hl-attribute" style="color: #7F0055">"lib/utils.ftl"</span> <span class="hl-attribute" style="color: #7F0055">as</span> <span class="hl-attribute" style="color: #7F0055">u&gt;</span> 

<span class="hl-attribute" style="color: #7F0055">&lt;@u.page&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;p&gt;${exampleObject.name}</span> <span class="hl-attribute" style="color: #7F0055">by</span> <span class="hl-attribute" style="color: #7F0055">${exampleObject.developer}&lt;/p&gt;</span>

  <span class="hl-attribute" style="color: #7F0055">&lt;ul&gt;</span>
    <span class="hl-attribute" style="color: #7F0055">&lt;#list</span> <span class="hl-attribute" style="color: #7F0055">systems</span> <span class="hl-attribute" style="color: #7F0055">as</span> <span class="hl-attribute" style="color: #7F0055">system&gt;</span>
      <span class="hl-attribute" style="color: #7F0055">&lt;li&gt;${system_index</span> <span class="hl-attribute" style="color: #7F0055">+</span> <span class="hl-attribute" style="color: #7F0055">1}.</span> <span class="hl-attribute" style="color: #7F0055">${system.name}</span> <span class="hl-attribute" style="color: #7F0055">from</span> <span class="hl-attribute" style="color: #7F0055">${system.developer}&lt;/li&gt;</span>
    <span class="hl-attribute" style="color: #7F0055">&lt;/#list&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;/ul&gt;</span>
  
  <span class="hl-attribute" style="color: #7F0055">&lt;#--</span> <span class="hl-attribute" style="color: #7F0055">Just</span> <span class="hl-attribute" style="color: #7F0055">another</span> <span class="hl-attribute" style="color: #7F0055">example</span> <span class="hl-attribute" style="color: #7F0055">of</span> <span class="hl-attribute" style="color: #7F0055">using</span> <span class="hl-attribute" style="color: #7F0055">a</span> <span class="hl-attribute" style="color: #7F0055">macro:</span> <span class="hl-attribute" style="color: #7F0055">--&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;@u.otherExample</span> <span class="hl-attribute" style="color: #7F0055">p1</span>=<span class="hl-value" style="color: blue">11</span> <span class="hl-attribute" style="color: #7F0055">p2</span>=<span class="hl-value" style="color: blue">22</span><span class="hl-tag" style="color: #080"> /&gt;</span>
<span class="hl-tag" style="color: #080">&lt;/@u.page&gt;</span> </pre><p>
                  
               </p>
               <p>
                  Another way of reusing template fragments is moving the common
                  fragment into
                  its own ftl file. Then just insert it with
                  <code class="code">&lt;#include "lib/myfragment.ftl"&gt;</code>. This is less flexible than macros,
                  but simpler in concept: it mimics copy-pasting.
                  
               </p>
            </div>
            <div class="section">
               <div class="titlepage">
                  <div>
                     <div>
                        <h3 class="title"><a name="freemarkertricks_variables"></a>5.2.&nbsp;Variables
                        </h3>
                     </div>
                  </div>
               </div>
               <p>
                  You can define and assign content to variables inside the FTL files for easy reuse.
                  
               </p>
               <p>
                  
               </p><pre class="programlisting"><span class="hl-tag" style="color: #080">&lt;#assign</span> <span class="hl-attribute" style="color: #7F0055">var_link</span> = <span class="hl-value" style="color: blue">"http://www.vogella.com/people/larsvogel.html"</span><span class="hl-tag" style="color: #080">&gt;</span>

<span class="hl-tag" style="color: #080">&lt;a</span> <span class="hl-attribute" style="color: #7F0055">href</span>=<span class="hl-value" style="color: blue">"${var_link}"</span><span class="hl-tag" style="color: #080">&gt;</span>About Lars Vogel<span class="hl-tag" style="color: #080">&lt;/a&gt;</span> </pre><p>
                  
               </p>
            </div>
            <div class="section">
               <div class="titlepage">
                  <div>
                     <div>
                        <h3 class="title"><a name="freemarkertricks_macro"></a>5.3.&nbsp;Handling null/undefined values
                        </h3>
                     </div>
                  </div>
               </div>
               <p>
                  FreeMarker requires you to provide an explicit default for variables, so avoid
                  values that are
                  <code class="code">null</code>
                  or undefined:
                  
               </p>
               <p>
                  
               </p><pre class="programlisting"><em class="hl-comment" style="color: #080">&lt;!-- Acts like if the color was N/A if there's no color: --&gt;</em>
<span class="hl-tag" style="color: #080">&lt;p&gt;</span>Color: ${color!'N/A'}<span class="hl-tag" style="color: #080">&lt;/p&gt;</span>
 
<em class="hl-comment" style="color: #080">&lt;!-- Avoid the whole color row if there's no color: --&gt;</em>
<span class="hl-tag" style="color: #080">&lt;#if</span> <span class="hl-attribute" style="color: #7F0055">color??&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;p&gt;Color:</span> <span class="hl-attribute" style="color: #7F0055">${color}&lt;/p&gt;</span>
<span class="hl-attribute" style="color: #7F0055">&lt;/#if&gt;</span> </pre><p>
                  
               </p>
            </div>
            <div class="section">
               <div class="titlepage">
                  <div>
                     <div>
                        <h3 class="title"><a name="freemarkertricks_escape"></a>5.4.&nbsp;Escape
                        </h3>
                     </div>
                  </div>
               </div>
               <p>
                  When generating HTML, it's important to escape
                  <code class="code">&lt;</code>,
                  <code class="code">&amp;</code>,
                  etc. in values that were not meant to store HTML and can contain these
                  problematic characters.
                  You can apply such escaping like
                  <code class="code">${message?html}</code>.
                  You can also ask FreeMarker to add
                  <code class="code">?html</code>
                  to all
                  <code class="code">${}</code>
                  -s in a section like this:
                  
               </p>
               <p>
                  
               </p><pre class="programlisting"><span class="hl-tag" style="color: #080">&lt;#escape</span> <span class="hl-attribute" style="color: #7F0055">x</span> <span class="hl-attribute" style="color: #7F0055">as</span> <span class="hl-attribute" style="color: #7F0055">x?html&gt;</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;p&gt;Sender:</span> <span class="hl-attribute" style="color: #7F0055">${from}</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;p&gt;Title:</span> <span class="hl-attribute" style="color: #7F0055">${title}</span>
  <span class="hl-attribute" style="color: #7F0055">&lt;p&gt;Message:</span> <span class="hl-attribute" style="color: #7F0055">${body}</span>
<span class="hl-attribute" style="color: #7F0055">&lt;/#escape&gt;</span> </pre><p>
                  
               </p>
               <p>
                  It's important to understand that
                  <code class="code">#escape</code>
                  only affects the
                  <code class="code">${}</code>
                  bits that are inside the enclosed section in the template file when you
                  look
                  at it in a text editor. That means,
                  <code class="code">${}</code>
                  embracements which are in other templates or macros
                  called from there, won't be affected.
                  
               </p>
