<p></p>
<p>Declaring an enumeration does not allocate any memory.  When a variable of the enumerated type is defined (such as variable paint in the example above), memory is allocated for that variable at that time.</p>
<p>Note that each enumerator is separated by a comma, and the entire enumeration is ended with a semicolon.  </p>
<p><strong>Naming enums</strong></p>
<p>Enum identifiers are often named starting with a capital letter, and the enumerators are often named using all caps.  Because enumerators are placed into the same namespace as the enumeration, an enumerator name can&#8217;t be used in multiple enumerations within the same namespace:</p>

<p><strong>Enum allocation and forward declaration</strong></p>
<p>Enum types are considered part of the integer family of types, and it&#8217;s up to the compiler to determine how much memory to allocate for an enum variable.  The C++ standard says the enum size needs to be large enough to represent all of the enumerator values.  Most often, it will make enum variables the same size as a standard int.</p>
<p>Because the compiler needs to know how much memory to allocate for an enumeration, you cannot forward declare enum types.  However, there is an easy workaround.  Because defining an enumeration does not allocate any memory, if an enumeration is needed in multiple files, it is fine to define the enumeration in a header, and #include that header wherever needed.</p>
<p><strong>What are enumerators useful for?</strong></p>
<p>Enumerated types are incredibly useful for code documentation and readability purposes when you need to represent a specific, predefined set of states.</p>

<p>So in theory, if you have an enumerated value that is larger than an int, the compiler should pick a larger integral type.  Visual Studio 2010 errors on your example because it doesn&#8217;t have any integral types larger than int.  Compilers compliant with C++11 should allow this though, as they could pick a long long integral type.</p>

<p>C++ didn&#8217;t allow the last enumerator to have a comma until C++11, where they started allowing it.<br />
C didn&#8217;t allow the last enumerator to have a comma until C99.</p>
<p>So it really depends on what set of standards your compiler adheres to.  Any C++11 compatible compiler should allow it.  Any pre-C++11 compiler may or may not allow it for C99 compatibility purposes.  It sounds like yours does allow it.</p>
