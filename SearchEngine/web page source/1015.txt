<div style="color:#000080;font-family:Arial;font-size:10pt">
The use of the implicit monitors in Java objects is powerful, but you can achieve a more subtle level of control through
inter-process communication. As you will see, this is especially easy in Java.
<p>Multithreading replaces event loop programming by dividing your tasks into discrete and logical units. Threads also provide a secondary benefit: they do away with polling. Polling is usually implemented by a loop that
is used to check some condition repeatedly. Once the condition is true, appropriate action is taken. This wastes CPU time. For example, consider the classic queuing problem,
where one thread is producing some data and another is consuming it. To make the problem more interesting, suppose that the producer has to wait until the consumer is
finished before it generates more data. In a polling system, the consumer would waste many CPU cycles while it <br>waited for the producer to produce. Once the producer was
finished, it would start polling, wasting more CPU cycles waiting for the consumer to finish, and
so on. Clearly, this situation is undesirable.</p>
<p>To avoid polling, Java includes an elegant interrocess
communication mechanism via the <b>wait( )</b>, <b>notify( )</b>, and <b>notifyAll( ) </b>methods.
These methods are implemented as <b>final
</b>
methods in <b>Object</b>, so all classes have them. All three
methods can be called only from within a <b>synchronized </b>method. Although conceptually
advanced from a computer science perspective, the rules for using these methods are
actually quite simple:</p>
<ul>
  <li><b>wait( ) </b>tells the calling thread to give up the monitor
and go to sleep until some other
<br>thread enters the same monitor and calls <b>notify( )</b>.</li>
  <li>
<b>
notify( ) </b>wakes up the first thread that called <b>wait(
) </b>on the same object.</li>
  <li>
<b>
notifyAll( ) </b>wakes up all the threads that called <b>wait(
) </b>on the same object. The
<br>highest priority thread will run first.</li>
</ul>
<p>These methods are declared within <b>Object</b>, as shown here:</p>
<p>final void wait( ) throws InterruptedException
<br>final void notify( )
<br>final void notifyAll( )</p>
<p>Additional forms of <b>wait( ) </b>exist that allow you to
specify a period of time to wait. The following sample program incorrectly implements a simple
form of the producer/consumer problem. It consists of four classes: <b>Q</b>,
the queue that you're trying to synchronize; <b>Producer</b>, the threaded object that is
producing queue entries; <b>Consumer</b>, the threaded object that is consuming queue entries; and <b>PC</b>,
the tiny class that creates the single <b>Q</b>, <b>Producer</b>, and <b>Consumer</b>.</p>
<code>
  <p>// An incorrect implementation of a producer and consumer.
<br>class Q {
<br>int n;
<br>synchronized int get() {
<br>System.out.println(&quot;Got: &quot; + n);
<br>return n;
<br>}
<br>synchronized void put(int n) {
<br>this.n = n;
<br>System.out.println(&quot;Put: &quot; + n);
<br>}
<br>}</p>
  <p>class Producer implements Runnable {
<br>Q q;
<br>Producer(Q q) {
<br>this.q = q;
<br>new Thread(this, &quot;Producer&quot;).start();
<br>}
<br>public void run() {
<br>int i = 0;
<br>while(true) {
<br>q.put(i++);
<br>}
<br>}
<br>}</p>
  <p>class Consumer implements Runnable {
<br>Q q;
<br>Consumer(Q q) {
<br>this.q = q;
<br>new Thread(this, &quot;Consumer&quot;).start();
<br>}
<br>public void run() {
<br>while(true) {
<br>q.get();
<br>}
<br>}
<br>}</p>
  <p>class PC {
<br>public static void main(String args[]) {
<br>Q q = new Q();
<br>new Producer(q);
<br>new Consumer(q);
<br>System.out.println(&quot;Press Control-C to stop.&quot;);
<br>}
<br>}</p>
</code>
<p>Although the <b>put( ) </b>and <b>get( ) </b>methods on <b>Q </b>are
synchronized, nothing stops the producer from overrunning the consumer, nor will anything stop
the consumer from consuming the same queue value twice. Thus, you get the
erroneous output shown here (the exact output will vary with processor speed and task load):</p>
<p>Put: 1
<br>Got: 1
<br>Got: 1
<br>Got: 1
<br>Got: 1
<br>Got: 1
<br>Put: 2
<br>Put: 3
<br>Put: 4
<br>Put: 5
<br>Put: 6
<br>Put: 7
<br>Got: 7</p>
<p>As you can see, after the producer put 1, the consumer started
and got the same 1 five times in a row. Then, the producer resumed and produced 2
through 7 without letting the consumer have a chance to consume them.</p>
<p>The proper way to write this program in Java is to use <b>wait(
) </b>and <b>notify( ) </b>to signal in both directions, as shown here:</p><code>
<p>// A correct implementation of a producer and consumer.
<br>class Q {
<br>int n;
<br>boolean valueSet = false;
<br>synchronized int get() {
<br>if(!valueSet)
<br>try {
<br>wait();
<br>} catch(InterruptedException e) {
<br>System.out.println(&quot;InterruptedException caught&quot;);
<br>}
<br>System.out.println(&quot;Got: &quot; + n);
<br>valueSet = false;
<br>notify();
<br>return n;
<br>}
<br>synchronized void put(int n) {
<br>if(valueSet)
<br>try {
<br>wait();
<br>} catch(InterruptedException e) {
<br>System.out.println(&quot;InterruptedException caught&quot;);
<br>}
<br>this.n = n;
<br>valueSet = true;
<br>System.out.println(&quot;Put: &quot; + n);
<br>notify();
<br>}
<br>}</p>
  <p>class Producer implements Runnable {
<br>Q q;
<br>Producer(Q q) {
<br>this.q = q;
<br>new Thread(this, &quot;Producer&quot;).start();
<br>}
<br>public void run() {
<br>int i = 0;
<br>while(true) {
<br>q.put(i++);
<br>}
<br>}
<br>}</p>
  <p>class Consumer implements Runnable {
<br>Q q;
<br>Consumer(Q q) {
<br>this.q = q;
<br>new Thread(this, &quot;Consumer&quot;).start();
<br>}
<br>public void run() {
<br>while(true) {
<br>q.get();
<br>}
<br>}
<br>}</p>
  <p>class PCFixed {
<br>public static void main(String args[]) {
<br>Q q = new Q();
<br>new Producer(q);
<br>new Consumer(q);
<br>System.out.println(&quot;Press Control-C to stop.&quot;);
<br>}
<br>}</p>
</code>
<p>Inside <b>get( )</b>, <b>wait( ) </b>is called. This causes its
execution to suspend until the <b>Producer
</b>
notifies you that some data is ready. When this happens,
execution inside <b>get( )
</b>
resumes. After the data has been obtained, <b>get( ) </b>calls <b>notify(
)</b>. This tells <b>Producer </b>that it is okay to put more data in the queue. Inside <b>put( )</b>, <b>wait(
) </b>suspends execution until the
<b>
Consumer </b>has removed the item from the queue. When execution
resumes, the next item of data is put in the queue, and <b>notify( ) </b>is
called. This tells the <b>Consumer </b>that it should now remove it.</p>
<p>Here is some output from this program, which shows the clean
synchronous behavior:</p>
<p>Put: 1
<br>Got: 1
<br>Put: 2
<br>Got: 2
<br>Put: 3
<br>Got: 3
<br>Put: 4
<br>Got: 4
<br>Put: 5
<br>Got: 5</p>
