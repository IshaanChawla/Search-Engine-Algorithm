			<div id="post-7080" class="post-7080 post type-post status-publish format-standard hentry category-eclipse category-simon category-vogella tag-buildship tag-eclipse tag-gradle tag-gradleware tag-tooling tag-tutorial">
			<h2 class="entry-title"><a href="http://blog.vogella.com/2015/04/17/buildship-on-the-mission-to-mars/" rel="bookmark">Buildship on the mission to Mars</a></h2>

			<div class="entry-meta">
				<span class="meta-prep meta-prep-author">Posted on</span> <a href="http://blog.vogella.com/2015/04/17/buildship-on-the-mission-to-mars/" title="5:00 pm" rel="bookmark"><span class="entry-date">April 17, 2015</span></a> <span class="meta-sep">by</span> <span class="author vcard"><a class="url fn n" href="http://blog.vogella.com/author/simon/" title="View all posts by Simon Scholz">Simon Scholz</a></span>			</div><!-- .entry-meta -->

				<div class="entry-content">
				<p><a href="http://blog.vogella.com/wp-content/uploads/2015/04/2286865364_900e1fe74e_z.jpg" onclick="_gaq.push(['_trackEvent', 'outbound-article-int', 'http://blog.vogella.com/wp-content/uploads/2015/04/2286865364_900e1fe74e_z.jpg', '']);" ><img class="alignnone wp-image-7081 size-medium" src="http://blog.vogella.com/wp-content/uploads/2015/04/2286865364_900e1fe74e_z-300x199.jpg" alt="2286865364_900e1fe74e_z" width="300" height="199" /></a></p>
<p>Gradleware decided to implement their own tooling solution for building project within the Eclipse IDE and the Buildship project was born.</p>
<p><a href="http://blog.vogella.com/wp-content/uploads/2015/04/Selection_103.png" onclick="_gaq.push(['_trackEvent', 'outbound-article-int', 'http://blog.vogella.com/wp-content/uploads/2015/04/Selection_103.png', '']);" ><img class="alignnone wp-image-7082 size-full" src="http://blog.vogella.com/wp-content/uploads/2015/04/Selection_103.png" alt="Selection_103" width="907" height="107" /></a></p>
<p>Since march there are currently 123 commits on the master branch of the GitHub project, which can be found here: <a href="https://github.com/eclipse/buildship" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/eclipse/buildship', 'https://github.com/eclipse/buildship']);" title="Buildship on GitHub">https://github.com/eclipse/buildship</a></p>
<p>We from the vogella company will be working together with Gradleware and hopefully many other contributors as well.</p>
<p>Therefore we started a new tutorial concerning the Buildship project, which will be constantly updated, when new features will be available. See <a title="Buildship tutorial" href="http://www.vogella.com/tutorials/EclipseGradle/article.html" target="_blank">http://www.vogella.com/tutorials/EclipseGradle/article.html<br />
</a> Detailed information can also be found on GitHub: <a title="Buildship on GitHub" href="https://github.com/eclipse/buildship">https://github.com/eclipse/buildship</a></p>
<p>Here is a little impression for you:</p>
<p><a href="http://blog.vogella.com/wp-content/uploads/2015/04/gradle_blog_teaser.png" onclick="_gaq.push(['_trackEvent', 'outbound-article-int', 'http://blog.vogella.com/wp-content/uploads/2015/04/gradle_blog_teaser.png', '']);" ><img class="alignnone size-full wp-image-7086" src="http://blog.vogella.com/wp-content/uploads/2015/04/gradle_blog_teaser.png" alt="gradle_blog_teaser" width="898" height="510" /></a></p>
<p>As I already &#8220;println&#8221; in the build script, please help us by <a href="http://www.vogella.com/tutorials/EclipseGradle/article.html#buildship_source" onclick="_gaq.push(['_trackEvent', 'outbound-article-int', 'http://www.vogella.com/tutorials/EclipseGradle/article.html#buildship_source', 'contributing to the Buildship project']);" title="Buildship tutorial for contribution"   target="_blank">contributing to the Buildship project</a>. So make your hands dirty, read the tutorial and give the Buildship project a try.</p>
<p>I&#8217;d also like to quote Wayne Beaton (<a href="https://waynebeaton.wordpress.com/2015/04/05/screenshot-of-the-week-gradle-tools/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://waynebeaton.wordpress.com/2015/04/05/screenshot-of-the-week-gradle-tools/', 'Blog entry concerning Buildship']);" >Blog entry concerning Buildship</a>):</p>
<blockquote><p>Great Fixes for Buildship will qualify for the <a href="https://www.eclipse.org/projects/greatfix/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://www.eclipse.org/projects/greatfix/', 'Great Fixes for Mars']);" >Great Fixes for Mars</a> Skills Competition!</p></blockquote>
<p>&nbsp;</p>

			<h2 class="entry-title"><a href="http://blog.vogella.com/2015/03/12/dealing-with-whitespace-legacy-in-eclipse-projects/" rel="bookmark">Dealing with whitespace legacy in Eclipse projects</a></h2>

			<div class="entry-meta">
				<span class="meta-prep meta-prep-author">Posted on</span> <a href="http://blog.vogella.com/2015/03/12/dealing-with-whitespace-legacy-in-eclipse-projects/" title="12:21 pm" rel="bookmark"><span class="entry-date">March 12, 2015</span></a> <span class="meta-sep">by</span> <span class="author vcard"><a class="url fn n" href="http://blog.vogella.com/author/vogella/" title="View all posts by Lars Vogel">Lars Vogel</a></span>			</div><!-- .entry-meta -->

				<div class="entry-content">
				<p>One of most annoying things during Gerrit reviews are &#8220;unnecessary whitespaces&#8221; change discussion. This applies for the contributors and the reviewers. </p>
<p>As we in the Eclipse platform.ui project currently see a flow incoming Gerrit reviews from various people, I want to minimize these issues to keep everyone involved in this process happy. Therefore I started to cleanup the existing whitespaces and a activate the save actions on the project to avoid the creation of new whitespace.</p>
<p>If you also would like to do this use Source -&gt; Clean-up on your source folder and activate afterwards the &#8220;Remove unnecessary whitespace&#8221; save action via the project setting. </p>
<p>Eclipse can be instructed to ignore these changes in its blame annotations via the &#8220;Ignore whitespace changes&#8221; preference setting in Team -&gt; Git.</p>
<p>With these changes you bite the bullet once but afterwards the Gerrit review process becomes much more enjoyable. </p>
