		<h1 class="firstheading"><a href="http://www.codingunit.com/c-tutorial-first-c-program-hello-world" rel="bookmark" title="First C program, Hello World">
		First C program, Hello World</a></h1>

			<div class="entry">
				<div id="watermark">
					<p>
After talking about the history and compilers it is time to make our first program. We like to make C programs under Linux, so we will use a text editor and the gnu compiler. But all programs we make in these tutorials will also work under Windows.&nbsp;Remember, the examples included in the C and C++ tutorials are all console programs. That means they use text to communicate. All compilers support the compilation of console programs. Check the user&#8217;s manual of your compiler for more info on how to compile them. (It is not doable for us to write this down for every compiler).<span id="more-139"></span>
</p>
<p>
Open a text editor (vi, emacs, notepad) or make a console project in Visual Studio Express and type the following lines of code (Don&#8217;t use cut/paste, type them. This is better for learning purposes):
</p>
<div id="cl">
	<PRE><CODE><br />
	#include&lt;stdio.h&gt;</p>
<p>	int main()<br />
	{<br />
		printf(&#8220;Hello World\n&#8221;);<br />
		return 0;<br />
	}<br />
	</CODE></PRE>
	</div>
<p>
Save the program with the name: hello.c
</p>
<h2>#include&lt;stdio.h&gt;</h2>
<p>With this line of code we include a file called stdio.h. (Standard Input/Output header file). This file lets us use certain commands for input or output which we can use in our program. (Look at it as lines of code commands) that have been written for us by someone else). For instance it has commands for input like reading from the keyboard and output commands like printing things on the screen.
</p>
<h2>int main()</h2>
<p>The int is what is called the return value (in this case of the type integer). Where it used for will be explained further down. Every program must have a main(). It is the starting point of every program. The round brackets are there for a<br />
reason, in a later tutorial it will be explained, but for now it is enough to know that they have to be there.
</p>
<h2>{}</h2>
<p>The two curly brackets (one in the beginning and one at the end) are used to group all commands together. In this case all the commands between the two curly brackets belong to main(). The curly brackets are often used in the C language to group commands together. (To mark the beginning and end of a<br />
group or function.).
</p>
<h2>printf(&#8220;Hello World\n&#8221;);</h2>
<p>The printf is used for printing things on the screen, in this case the words: Hello World. As you can see the data that is to be printed is put inside round brackets. The words Hello World are inside inverted ommas, because they are what is called a string. (A single letter is called a character and a series of characters is called a string). Strings must always be put between inverted commas. The \n is called an escape sequence. In this case it represents a newline character. After printing something to the screen you usually want to print something on the next line. If there is no \n then a next printf command will print the string on the same line. </p>
<p>Commonly used escape sequences are: </p>
<ul>
<li> \n (newline)</li>
<li> \t (tab)</li>
<li> \v (vertical tab)</li>
<li> \f (new page)</li>
<li> \b (backspace)</li>
<li> \r (carriage return)</li>
</ul>
<p>
After the last round bracket there must be a semi-colon. The semi-colon shows that it is the end of the command. (So in the future, don&#8217;t forget to put a semi-colon if a command ended).
</p>
<h2>return 0;</h2>
<p>When we wrote the first line &#8220;int main()&#8221;, we declared that main&nbsp;must return an integer int main(). (int is short for integer which is another word for number). With the command return 0; we can return the value null to the operating system. When you return with a zero you tell the operating system that there were no errors while running the program. (Take a look at <a href="http://www.codingunit.com/c-tutorial-variables-and-constants" title="C tutorial - variables and constants">the C tutorial &#8211; variables and constants</a> for more information on integer variables).
 </p>
<h2>Compile</h2>
<p>If you have typed everything correct there should be no problem to compile your program. After compilation you now should have a hello.exe (Windows) or hello binary (UNIX/Linux). You can now run this program by typing hello.exe (Windows) or ./hello (UNIX/Linux).
</p>
<p>
If everything was done correct, you should now see the words &#8220;Hello World&#8221; printed on the screen.
</p>
<p>
<b>Congratulations!!!!!</b><br /> You have just made your first program in C.
</p>
<h2>Comments in your program</h2>
<p>The Hello World program is a small program that is easy to understand. But a program can contain thousands of&nbsp;lines of code and can be so complex that it is hard for us to understand. To make our lives easier it is possible to write an explanation or comment in a program. This makes it much easier to understand the code. (Even if you did not look at the code for years).These comments will be ignored by the compiler at compilation time.
</p>
<p>
Comments have to be put after // or be placed between /* */ .<br />
Here is an example of how to comment the Hello World source code :
</p>
<div id="cl">
	<PRE><CODE><br />
	/*	Description	: Print Hello World on the screen<br />
		Author		: Your name<br />
		Date		: 01/01/2007				*/</p>
<p>	#include&lt;stdio.h&gt;</p>
<p>	int main()<br />
	{<br />
		//Print something and then newline<br />
		printf(&#8220;Hello World\n&#8221;);<br />
		return 0;<br />
	}<br />
	</CODE></PRE>
	</div>
<p></p>
<h2>Indentation</h2>
<p>As you can see the printf and return statements have been indented or moved to the right side. This is<br />
done to make the code more readable. In a program as Hello World, it seems a stupid thing to do. But as the programs become more complex, you will see that it makes the code more readable.
</p>
<p>
So,&nbsp;always use&nbsp;indentations and comments to make the code more readable. It will make your life much easier if the code becomes more complex.</p>

			<p>Ajay, the reason you can include stdio.h multiple times is due to a condition that is usually included at the top of an INCLUDE file:<br />
#ifndef _INC_STDIO    &lt;&lt; at the beginning &#8230;.<br />
#define _INC_STDIO<br />
&#8230;.<br />
#endif  &lt;&lt; placed on the last line.</p>
<p>This effectively keeps the include instructions in stdio.h from only being executed once.  It defines the _INC_STDIO at the start of the first include and turns all future includes into a No-Op.  So your 2nd include of stdio.h is safe, but is not doing anything.</p>
