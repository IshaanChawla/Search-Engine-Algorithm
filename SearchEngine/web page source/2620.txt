                  <div>
                     <h2 class="title"><a name="dependencyinjection"></a>1.&nbsp;Dependency Injection
                     </h2>
                  </div>
               </div>
            </div>
            <p> Java components / classes should be as independent as possible
               of other Java classes. This increases the possibility to reuse these
               classes and to test them independently of other classes(Unit Testing).
               To decouple Java components from other Java components the dependency
               to a certain other class should get injected into them rather that the
               class itself creates / finds this object.
            </p>
            <p> A class A has a dependency to class B if class uses class B as a
               variable.
            </p>
            <p>
               If dependency injection is used then the class B is given to class A
               via
               
            </p>
            <div class="itemizedlist">
               <ul class="itemizedlist" type="disc">
                  <li class="listitem">
                     <p>the constructor of the class A - this is then called
                        construction injection
                     </p>
                  </li>
                  <li class="listitem">
                     <p>a setter - this is then called setter injection</p>
                  </li>
               </ul>
            </div>
            <p>
               
            </p>
            <p> The general concept between dependency injection is called
               Inversion of Control. A class should not configure itself but should
               be configured from outside.
            </p>
            <p> A design based on independent classes / components increases the
               re-usability and possibility to test the software. For example, if a
               class A expects a Dao (Data Access object) for receiving the data from
               a database you can easily create another test object which mocks the
               database connection and inject this object into A to test A without
               having an actual database connection.
            </p>
            <p> A software design based on dependency injection is possible with
               standard Java.
            </p>
            <p> Spring just simplifies the use of dependency injection
               by providing a standard way of providing the configuration
               and by managing the reference to the created objects.
            </p>
         </div>
         <div class="section">
            <div class="titlepage">
               <div>
                  <div>
                     <h2 class="title"><a name="springoverview"></a>2.&nbsp;Spring Overview
                     </h2>
                  </div>
               </div>
            </div>
            <p> The Spring Framework is a very comprehensive framework.</p>
            <p> The fundamental functionality provided by the Spring Container
               is dependency injection. Spring provides a light-weight container,
               e.g. the Spring core container, for dependency injection (DI).
            </p>
            <p> This container lets you inject required objects into other
               objects. This results in a design in which the Java class are not
               hard-coupled. The injection in Spring is either done via setter
               injection of via construction injection.
            </p>
            <p>These classes which are managed by Spring must conform to
               the JavaBean standard.
            </p>
            <p>In the context of Spring classes are also referred to as beans
               or as spring beans.
            </p>
            <p>
               The Spring core container:
               
            </p>
            <div class="itemizedlist">
               <ul class="itemizedlist" type="disc">
                  <li class="listitem">
                     <p>handles the configuration, generally based on
                        annotations or on an XML file (XMLBeanFactory)
                     </p>
                  </li>
                  <li class="listitem">
                     <p>manages the selected Java classes via the BeanFactory
                        
                     </p>
                  </li>
               </ul>
            </div>
            <p>
               
            </p>
            <p> The core container uses the so-called bean factory to create
               new objects. New objects are generally created as Singletons if not
               specified differently.
            </p>
            <p>
               
            </p>
         </div>

            </div>
            <p> We will later use the following datamodel for the example.

            <p> Create a Java project "de.vogella.spring.di.model" and create
               the following packages and classes.
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> writer;

<span class="hl-keyword">public</span> <span class="hl-keyword">interface</span> IWriter {
  <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> writer(String s);
} </pre><p>
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> writer;

<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> Writer <span class="hl-keyword">implements</span> IWriter {
  <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> writer (String s){
    System.out.println(s);
  }
} </pre><p>
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> writer;

<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> NiceWriter <span class="hl-keyword">implements</span> IWriter {
  <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> writer (String s){
    System.out.println(<span class="hl-string" style="color:blue">"The string is "</span> + s);
  }
} </pre><p>
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> testbean;

<span class="hl-keyword">import</span> writer.IWriter;

<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> MySpringBeanWithDependency {
  <span class="hl-keyword">private</span> IWriter writer;

  <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> setWriter(IWriter writer) {
    <span class="hl-keyword">this</span>.writer = writer;
  }

  <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> run() {
    String s = <span class="hl-string" style="color:blue">"This is my test"</span>;
    writer.writer(s);
  }
} </pre><p>
               
            </p>
            <p> The class "MySpringBeanWithDependency.java" contains a setter
               for the actual writer. We will use the Spring Framework to inject the
               correct writer into this class.
            </p>

               <div>
                  <div>
                     <h2 class="title"><a name="usageannotations"></a>5.&nbsp;Using dependency injection with annotations
                     </h2>
                  </div>
               </div>
            </div>
            <p> As of Spring 2.5 it is possible to configure the dependency
               injection via annotations. I recommend to use this way of configuring
               your Spring beans. The next chapter will also describe the way to
               configure this via XML.
            </p>
            <p> Create a new Java project
               "de.vogella.spring.di.annotations.first" and include the minimal
               required spring jars into your classpath.
            </p>
            <p>Copy your model class from the de.vogella.spring.di.model
               project into this project.
            </p>
            <p> You need now to add annotations to your model to tell Spring
               which beans should be managed by Spring and how they should be
               connected.
            </p>
            <p>Add the @Service annotation the
               MySpringBeanWithDependency.java and NiceWriter.java. Also define with
               @Autowired on the setWriter method that the property "writer" will be
               autowired by Spring.
            </p>
            <p>
               
            </p>
            <div class="tip" style="margin-left: 0.5in; margin-right: 0.5in;">
               <h3 class="title">Tip</h3>@Autowired will tell Spring to search for a Spring bean
               which implements the required interface and place it automatically
               into the setter.
            </div>
            <p>
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> testbean;

<span class="hl-keyword">import</span> org.springframework.beans.factory.annotation.Autowired;
<span class="hl-keyword">import</span> org.springframework.stereotype.Service;

<span class="hl-keyword">import</span> writer.IWriter;

<em><span class="hl-annotation" style="color: gray">@Service</span></em>
<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> MySpringBeanWithDependency {
  <span class="hl-keyword">private</span> IWriter writer;

  <em><span class="hl-annotation" style="color: gray">@Autowired</span></em>
  <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> setWriter(IWriter writer) {
    <span class="hl-keyword">this</span>.writer = writer;
  }

  <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> run() {
    String s = <span class="hl-string" style="color:blue">"This is my test"</span>;
    writer.writer(s);
  }
} </pre><p>
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> writer;

<span class="hl-keyword">import</span> org.springframework.stereotype.Service;

<em><span class="hl-annotation" style="color: gray">@Service</span></em>
<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> NiceWriter <span class="hl-keyword">implements</span> IWriter {
  <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> writer(String s) {
    System.out.println(<span class="hl-string" style="color:blue">"The string is "</span> + s);
  }
} </pre><p>
               
            </p>
            <p> Under the src folder create a folder META-INF and create the
               following file in this folder. This is the Spring configuration file.
               
            </p>
            <p>
               
            </p><pre class="programlisting">&lt;beans xmlns=<span class="hl-string" style="color:blue">"http://www.springframework.org/schema/beans"</span>
  xmlns:xsi=<span class="hl-string" style="color:blue">"http://www.w3.org/2001/XMLSchema-instance"</span>
  xmlns:aop=<span class="hl-string" style="color:blue">"http://www.springframework.org/schema/aop"</span>
  xmlns:context=<span class="hl-string" style="color:blue">"http://www.springframework.org/schema/context"</span>
  xsi:schemaLocation=<span class="hl-string" style="color:blue">"http://www.springframework.org/schema/beans
</span>           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/beans/spring-beans-2.5.xsd</em>
           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/aop</em>
           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/aop/spring-aop-2.5.xsd</em>
           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/context</em>
           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/context/spring-context-2.5.xsd"&gt;</em>

  &lt;context:component-scan base-<span class="hl-keyword">package</span>=<span class="hl-string" style="color:blue">"testbean"</span> /&gt;
  &lt;context:component-scan base-<span class="hl-keyword">package</span>=<span class="hl-string" style="color:blue">"writer"</span> /&gt;

&lt;/beans&gt; </pre><p>
               
            </p>
            <p>
               
            </p>
            <div class="tip" style="margin-left: 0.5in; margin-right: 0.5in;">
               <h3 class="title">Tip</h3>
               You can also configure the log4j logger (this is optional) by copying
               the following file into the source folder.
               
               <p>
                  
               </p><pre class="programlisting">log4j.rootLogger=FATAL, first
log4j.appender.first=org.apache.log4j.ConsoleAppender
log4j.appender.first.layout=org.apache.log4j.PatternLayout
log4j.appender.first.layout.ConversionPattern=%-4r [%t] %-5p %c %x - %m%n </pre><p>
                  
               </p>
            </div>
            <p>
               
            </p>
            <p> Afer this setup you can wire the application together. Create a
               main class which reads the configuration file and starts the
               application.
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> main;

<span class="hl-keyword">import</span> org.springframework.beans.factory.BeanFactory;
<span class="hl-keyword">import</span> org.springframework.context.ApplicationContext;
<span class="hl-keyword">import</span> org.springframework.context.support.ClassPathXmlApplicationContext;

<span class="hl-keyword">import</span> testbean.MySpringBeanWithDependency;

<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> Main {
  <span class="hl-keyword">public</span> <span class="hl-keyword">static</span> <span class="hl-keyword">void</span> main(String[] args) {
    ApplicationContext context = <span class="hl-keyword">new</span> ClassPathXmlApplicationContext(<span class="hl-string" style="color:blue">"META-INF/beans.xml"</span>);
    BeanFactory factory = context;
    MySpringBeanWithDependency test = (MySpringBeanWithDependency) factory
        .getBean(<span class="hl-string" style="color:blue">"mySpringBeanWithDependency"</span>);
    test.run();
  }
} </pre><p>
               
            </p>
            <p> If you run the application then the class for the
               IWriterInterface will be inserted into the Test class. By applying the
               dependency injecting I can later replace this writer with a more
               sophisticated writer.
            </p>
            <p> As a result the class Test does not depend on the concrete
               Writer class, is extensible and can be easily tested.
            </p>
         </div>

               <div>
                  <div>
                     <h2 class="title"><a name="usagexml"></a>6.&nbsp;Using dependency injection with XML
                     </h2>
                  </div>
               </div>
            </div>
            <p> The following example will demonstrate the usage of the
               dependency injection via xml. The example will inject a writer into
               another class.
            </p>
            <p>
               
            </p>
            <div class="tip" style="margin-left: 0.5in; margin-right: 0.5in;">
               <h3 class="title">Tip</h3> I think annotations rock in general, therefore I recommend not
               to use the XML configuration but the annotation one. If you have good
               reason to use the XML configuration please feel free to do so.
            </div>
            <p>
               
            </p>
            <p> Create a new Java project "de.vogella.spring.di.xml.first" and
               include the minimal required spring jars into your classpath.
            </p>
            <p>Copy your model class from the de.vogella.spring.di.model
               project into this project.
            </p>
            <p> Under the src folder create a folder META-INF and create the
               following file in this folder. This is the Spring configuration file.
               
            </p>
            <p>
               
            </p><pre class="programlisting">&lt;beans xmlns=<span class="hl-string" style="color:blue">"http://www.springframework.org/schema/beans"</span>
  xmlns:xsi=<span class="hl-string" style="color:blue">"http://www.w3.org/2001/XMLSchema-instance"</span>
  xmlns:aop=<span class="hl-string" style="color:blue">"http://www.springframework.org/schema/aop"</span>
  xmlns:context=<span class="hl-string" style="color:blue">"http://www.springframework.org/schema/context"</span>
  xsi:schemaLocation=<span class="hl-string" style="color:blue">"http://www.springframework.org/schema/beans
</span>           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/beans/spring-beans-2.5.xsd</em>
           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/aop</em>
           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/aop/spring-aop-2.5.xsd</em>
           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/context</em>
           http:<em class="hl-comment" style="color: #080">//www.springframework.org/schema/context/spring-context-2.5.xsd"&gt;</em>


&lt;bean id=<span class="hl-string" style="color:blue">"writer"</span> <span class="hl-keyword">class</span>=<span class="hl-string" style="color:blue">"writer.NiceWriter"</span> /&gt;

&lt;bean id=<span class="hl-string" style="color:blue">"mySpringBeanWithDependency"</span> <span class="hl-keyword">class</span>=<span class="hl-string" style="color:blue">"testbean.MySpringBeanWithDependency"</span>&gt;
&lt;property name=<span class="hl-string" style="color:blue">"writer"</span> ref=<span class="hl-string" style="color:blue">"writer"</span> /&gt;
&lt;/bean&gt;

&lt;/beans&gt; </pre><p>
               
            </p>
            <p> Again, you can now wire the application together. Create a main
               class which reads the configuration file and starts the application.
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> main;

<span class="hl-keyword">import</span> org.springframework.beans.factory.BeanFactory;
<span class="hl-keyword">import</span> org.springframework.context.ApplicationContext;
<span class="hl-keyword">import</span> org.springframework.context.support.ClassPathXmlApplicationContext;

<span class="hl-keyword">import</span> testbean.MySpringBeanWithDependency;

<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> Main {
  <span class="hl-keyword">public</span> <span class="hl-keyword">static</span> <span class="hl-keyword">void</span> main(String[] args) {
    ApplicationContext context = <span class="hl-keyword">new</span> ClassPathXmlApplicationContext(<span class="hl-string" style="color:blue">"META-INF/beans.xml"</span>);
    BeanFactory factory = context;
    MySpringBeanWithDependency test = (MySpringBeanWithDependency) factory
        .getBean(<span class="hl-string" style="color:blue">"mySpringBeanWithDependency"</span>);
    test.run();
  }
} </pre><p>
               
            </p>
