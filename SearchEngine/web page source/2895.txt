	<div class="post" id="post-207">
		<h1 class="firstheading"><a href="http://www.codingunit.com/printf-format-specifiers-format-conversions-and-formatted-output" rel="bookmark" title="C Tutorial &#8211; printf, Format Specifiers, Format Conversions and Formatted Output">
		C Tutorial &#8211; printf, Format Specifiers, Format Conversions and Formatted Output</a></h1>

			<div class="entry">
				<div id="watermark">
					<p>In this C programming language tutorial we take another look at the printf function. We will look at how to use format specifiers to print formatted output onto the screen. The topics covered are; a little printf background, format specifiers and conversions, formatting of different types and format conversions of strings.<span id="more-207"></span></p>
<h2>printf Background</h2>
<p>The printf function is not part of the C language, because there is no input or output defined in C language itself. The printf function is just a useful function from the standard library of functions that are accessible by C programs. The behavior of printf is defined in the ANSI standard. If the compiler that you&#8217;re using conforms to this standard then all the features and properties should be available to you.</p>
<h2>Format Specifiers</h2>
<p>There are many format specifiers defined in C. Take a look at the following list:</p>
<table border="1" cellspacing="0" cellpadding="2" width="188">
<colgroup>
<col width="82"></col>
<col width="98"></col>
</colgroup>
<tbody>
<tr>
<td width="82">%i or %d</td>
<td width="98">int</td>
</tr>
<tr>
<td width="82">%c</td>
<td width="98">char</td>
</tr>
<tr>
<td width="82">%f</td>
<td width="98">float (see also the note below)</td>
</tr>
<tr>
<td width="82">%s</td>
<td width="98">string</td>
</tr>
</tbody>
</table>
<p><strong>Note:</strong> %f stands for float, but C language has also a thing called &#8220;default argument promotions&#8221;.<br />
Default argument promotions happen in variadic functions. Variadic functions are functions (e.g. printf) which take a variable number of arguments. When a variadic function is called, after lvalue-to-rvalue, array-to-pointer, and function-to-pointer conversions, each argument that is a part of the variable argument list undergoes additional conversions known as <strong>default argument promotions</strong>: </p>
<ul>
<li> float arguments are converted to double as in floating-point promotion</li>
<li> bool, char, short, and unscoped enumerations are converted to int or wider integer types as in integer promotion</li>
</ul>
<p>So for example, float parameters are converted to doubles, and char&#8217;s are converted to int&#8217;s. If you actually needed to pass, for example, a char instead of an int, the function would have to convert it back.</p>
<p>That&#8217;s enough on that side step of variadic function and &#8220;default argument promotions&#8221;. </p>
<p>Let us take a look at an example of printf formatted output (that why you here, isn&#8217;t it?):</p>
<div id="cl">
<pre><code>
	#include&lt;stdio.h&gt;

	main()
	{
		int a,b;
		float c,d;

		a = 15;
		b = a / 2;
		printf("%d\n",b);
		printf("%3d\n",b);
		printf("%03d\n",b);

		c = 15.3;
		d = c / 3;
		printf("%3.2f\n",d);
	}

</code></pre>
</div>
<p>Output of the source above:</p>
<div id="cl">
<pre><code>
7
   7
007
5.10

</code></pre>
</div>
<p>As you can see in the first printf statement we print a decimal. In the second printf statement we print the same decimal, but we use a width (%3d) to say that we want three digits (positions) reserved for the output.<br />
The result is that two &#8220;space characters&#8221; are placed before printing the character. In the third printf statement we say almost the same as the previous one. Print the output with a width of three digits, but fill the space with 0.</p>
<p>In the fourth printf statement we want to print a float. In this printf statement we want to print three position before the decimal point (called width) and two positions behind the decimal point (called precision).</p>
<p>The \n used in the printf statements is called an escape sequence. In this case it represents a newline character. After printing something to the screen you usually want to print something on the next line. If there is no \n then a next printf command will print the string on the same line. Commonly used escape sequences are:</p>
<ul>
<li>\n (newline)</li>
<li>\t (tab)</li>
<li>\v (vertical tab)</li>
<li>\f (new page)</li>
<li>\b (backspace)</li>
<li>\r (carriage return)</li>
<li>\n (newline)</li>
</ul>
<p>Let&#8217;s take another look at a printf formatted output in a more application like example:</p>
<div id="cl">
<pre><code>
#include&lt;stdio.h&gt;

main()
{
	int Fahrenheit;

	for (Fahrenheit = 0; Fahrenheit &lt;= 300; Fahrenheit = Fahrenheit + 20)
		printf("%3d %06.3f\n", Fahrenheit, (5.0/9.0)*(Fahrenheit-32));
}

</code></pre>
</div>
<p>Output of the source above:</p>
<div id="cl">
<pre><code>
  0 -17.778
 20 -6.667
 40 04.444
 60 15.556
 80 26.667
100 37.778
120 48.889
140 60.000
160 71.111
180 82.222
200 93.333
220 104.444
240 115.556
260 126.667
280 137.778
300 148.889

</code></pre>
</div>
<p>As you can see we print the Fahrenheit temperature with a width of 3 positions. The Celsius temperature is printed with a width of 6 positions and a precision of 3 positions after the decimal point. Let&#8217;s recap:</p>
<ul>
<li>%d		(print as a decimal integer)</li>
<li>%6d     (print as a decimal integer with a width of at least 6 wide)</li>
<li>%f		(print as a floating point)</li>
<li>%4f		(print as a floating point with a width of at least 4 wide)</li>
<li>%.4f	(print as a floating point with a precision of four characters after the decimal point)</li>
<li>%3.2f	(print as a floating point at least 3 wide and a precision of 2)</li>
</ul>
<h2>Formatting other Types</h2>
<p>Until now we only used integers and floats, but there are more types you can use. Take a look at the following example:</p>
<div id="cl">
<pre><code>
#include&lt;stdio.h&gt;

main()
{
	printf("The color: %s\n", "blue");
	printf("First number: %d\n", 12345);
	printf("Second number: %04d\n", 25);
	printf("Third number: %i\n", 1234);
	printf("Float number: %3.2f\n", 3.14159);
	printf("Hexadecimal: %x\n", 255);
	printf("Octal: %o\n", 255);
	printf("Unsigned value: %u\n", 150);
	printf("Just print the percentage sign %%\n", 10);
}

</code></pre>
</div>
<p>Output of the source example:</p>
<div id="cl">
<pre><code>
The color: blue
First number: 12345
Second number: 0025
Third number: 1234
Float number: 3.14
Hexadecimal: ff
Octal: 377
Unsigned value: 150
Just print the percentage sign %

</code></pre>
</div>
<p><strong>Note:</strong> In the last printf statement only the percentage sign is printed. </p>
<p>The number 10 in this statement doesn&#8217;t matter; it&#8217;s not used in the output.  So if you want to print a percentage number you would use something like this: printf(&#8220;%2d%%\n&#8221;, 10); (The output will be 10%)</p>
<h2>Formatting Strings</h2>
<p>By now you have seen most of the format conversion possible, but there is one type that is a little different<br />
and that are string format conversions. Take a look at the following example:</p>
<div id="cl">
<pre><code>
#include&lt;stdio.h&gt;

main()
{
	printf(":%s:\n", "Hello, world!");
	printf(":%15s:\n", "Hello, world!");
	printf(":%.10s:\n", "Hello, world!");
	printf(":%-10s:\n", "Hello, world!");
	printf(":%-15s:\n", "Hello, world!");
	printf(":%.15s:\n", "Hello, world!");
	printf(":%15.10s:\n", "Hello, world!");
	printf(":%-15.10s:\n", "Hello, world!");
}

</code></pre>
</div>
<p>The output of the example above:</p>
<div id="cl">
<pre><code>
:Hello, world!:
:  Hello, world!:
:Hello, wor:
:Hello, world!:
:Hello, world!  :
:Hello, world!:
:     Hello, wor:
:Hello, wor     :

</code></pre>
</div>
<p>As you can see, the string format conversion reacts very different from number format conversions.</p>
<ul>
<li>The printf(&#8220;:%s:\n&#8221;, &#8220;Hello, world!&#8221;); statement prints the string (nothing special happens.)</li>
<li>The printf(&#8220;:%15s:\n&#8221;, &#8220;Hello, world!&#8221;); statement prints the string, but print 15 characters. If the string is smaller the &#8220;empty&#8221; positions will be filled with &#8220;whitespace.&#8221;</li>
<li>The printf(&#8220;:%.10s:\n&#8221;, &#8220;Hello, world!&#8221;); statement prints the string, but print only 10 characters of the string.</li>
<li>The printf(&#8220;:%-10s:\n&#8221;, &#8220;Hello, world!&#8221;); statement prints the string, but prints at least 10 characters. If the string is smaller &#8220;whitespace&#8221; is added at the end. (See next example.)</li>
<li>The printf(&#8220;:%-15s:\n&#8221;, &#8220;Hello, world!&#8221;); statement prints the string, but prints at least 15 characters. The string in this case is shorter than the defined 15 character, thus &#8220;whitespace&#8221; is added at the end (defined by the minus sign.)</li>
<li>The printf(&#8220;:%.15s:\n&#8221;, &#8220;Hello, world!&#8221;); statement prints the string, but print only 15 characters of the string. In this case the string is shorter than 15, thus the whole string is printed.</li>
<li>The printf(&#8220;:%15.10s:\n&#8221;, &#8220;Hello, world!&#8221;); statement prints the string, but print 15 characters.<br />
If the string is smaller the &#8220;empty&#8221; positions will be filled with &#8220;whitespace.&#8221; But it will only print a maximum of 10 characters, thus only part of new string (old string plus the whitespace positions) is printed.</li>
<li>The printf(&#8220;:%-15.10s:\n&#8221;, &#8220;Hello, world!&#8221;); statement prints the string, but it does the exact same thing as the previous statement, accept the &#8220;whitespace&#8221; is added at the end.</li>
</ul>
<p><strong>A little warning!</strong><br />
The printf function uses its first argument to determine how many arguments will follow and of what types they are. If you don&#8217;t use enough arguments or if they are of the wrong type than printf will get confuses, with as a result wrong answers.</p>
<p>That&#8217;s all for this C tutorial. Just make some examples of your own, they are easy to make. This is the only way to learn and see how the format conversions reacts.</p>

The precision for &#8220;G&#8221; specifies the maximum number of significant digits printed.</p>
<p>Try the three specifiers by printing the value, for example:</p>
<p>#include&lt;stdio.h&gt;</p>
<p>int main {<br />
  float B=9876.54321;<br />
  printf(&#8220;%12.5G\n&#8221;, B);<br />
  printf(&#8220;%12.5f\n&#8221;, B);<br />
  printf(&#8220;%12.5E\n&#8221;, B);<br />
}</p>

			<p>@Steve,</p>
<p>I hope you realize that unless your GPS is accurate to 3 meters, you are not going to actually have 3 meters of accuracy, regardless of how many decimal points it reports.</p>
<p>Regarding your storage issue. Single precision floats have at most 6 to 9 decimal digits of precision. When converting to a string, you are essentially generating a decimal representation of that float. This means that you are guaranteed that a number with 6 decimal digits can be converted back and forth between a float and it&#8217;s string representation (which is decimal). However, any more than that really depends on the number (ie some 7, 8, and 9 digit decimals can be converted exactly while others won&#8217;t).</p>
<p>So, in your case, you have an 8 digit decimal number that you first convert to float, then back to decimal (in string form). You should expect that at most, the first 6 digits will match. </p>
<p>In you second example, you used atoi which converts an ascii string to an integer. Integers are whole numbers. They don&#8217;t have decimal points. So, the output is exactly what you should have gotten. Try atof.</p>
