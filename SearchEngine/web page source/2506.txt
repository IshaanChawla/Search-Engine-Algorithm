<h1 class="h1">Hibernate Interview Questions</h1>
<p>Hibernate interview questions are asked to the students because it is a widely used ORM tool. The important list of top 20 hibernate interview questions and answers for freshers and professionals are given below.</p>
<hr/>
<h3 class="h3">1) What is hibernate?</h3>
<p>Hibernate is an open-source and lightweight ORM tool that is used to store, manipulate and retrieve data from the database.</p>
<a href="hibernate-tutorial">more details...</a>

<hr/>
<h3 class="h3">2) What is ORM?</h3>
<p>ORM is an acronym for Object/Relational mapping. It is a programming strategy to map object with the data stored in the database. It simplifies data creation, data manipulation and data access.</p>

<hr/>
<h3 class="h3">3) Explain hibernate architecture?</h3>
<p>Hibernate architecture comprises of many interfaces such as Configuration, SessionFactory, Session, Transaction etc.</p> <a href="hibernate-architecture">more details...</a>
<img src="images/hibernate/architecture.jpg" alt="hibernate framework architecture"/>

<hr/>
<h3 class="h3">4) What are the core interfaces of Hibernate?</h3>
<p>The core interfaces of Hibernate framework are:</p>
<ul>
<li>Configuration</li>
<li>SessionFactory</li>
<li>Session</li>
<li>Query</li>
<li>Criteria</li>
<li>Transaction</li>
</ul>

<hr/>
<h3 class="h3">5) What is SessionFactory?</h3>
<p>SessionFactory provides the instance of Session. It is a factory of Session. It holds the data of second level cache that is not enabled by default.</p> <a href="hibernate-architecture">more details...</a>

<hr/>
<h3 class="h3">6) Is SessionFactory a thread-safe object?</h3>
<p>Yes, SessionFactory is a thread-safe object, many threads can access it simultaneously.</p>

<hr/>
<h3 class="h3">7) What is Session?</h3>
<p>It maintains a connection between hibernate application and database.</p>
<p>It provides methods to store, update, delete or fetch data from the database such as persist(), update(), delete(), load(), get() etc.</p>
<p>It is a factory of Query, Criteria and Transaction i.e. it provides factory methods to return these instances.</p>
<a href="hibernate-architecture">more details...</a>

<hr/>
<h3 class="h3">8) Is Session a thread-safe object?</h3>
<p>No, Session is not a thread-safe object, many threads can't access it simultaneously. In other words, you cannot share it between threads.</p>

<hr/>
<h3 class="h3">9) What is the difference between session.save() and session.persist() method?</h3>
<table class="alt">
<tr><th>No.</th><th>save()</th><th>persist()</th></tr>
<tr><td>1)</td><td>returns the identifier (Serializable) of the instance.</td><td>return nothing because its return type is void.</td></tr>
<tr><td>2)</td><td>Syn: public Serializable save(Object o)</td><td>Syn: public void persist(Object o) </td></tr>
</table>


<hr/>
<h3 class="h3">10) What is the difference between get and load method?</h3>
<p>The differences between get() and load() methods are given below.</p>
<table class="alt">
<tr><th>No.</th><th>get()</th><th>load()</th></tr>
<tr><td>1)</td><td>Returns <b>null</b> if object is not found.</td><td>Throws <b>ObjectNotFoundException</b> if object is not found.</td></tr>
<tr><td>2)</td><td>get() method always <b>hit the database</b>. </td><td>load() method <b>doesn't hit</b> the database. </td></tr>
<tr><td>3)</td><td>It returns real object <b>not proxy</b>.</td><td>It returns <b>proxy<b> object.</td></tr>
<tr><td>4)</td><td>It should be used if <b>you are not sure</b> about the existence of instance.</td><td>It should be used if <b>you are sure</b> that instance exists.</td></tr>
</table>

<hr/>
<h3 class="h3">11) What is the difference between update and merge method?</h3>
<p>The differences between update() and merge() methods are given below.</p>
<table class="alt">
<tr><th>No.</th><th>update() method</th><th>merge() method</th></tr>
<tr><td>1)</td><td>Update means to edit something.</td><td>Merge means to combine something.</td></tr>
<tr><td>2)</td><td>update() should be used if session doesn't contain an already persistent state with same id. It means update should be used inside the session only. After closing the session it will throw error.</td><td>merge() should be used if you don't know the state of the session, means you want to make modification at any time.</td></tr>
</table>

<p>Let's try to understand the difference by the example given below:</p>

<div class="codeblock"><textarea name="code" class="java">
...
SessionFactory factory = cfg.buildSessionFactory();
Session session1 = factory.openSession();
 
Employee e1 = (Employee) session1.get(Employee.class, Integer.valueOf(101));//passing id of employee
session1.close();
 
e1.setSalary(70000);
 
Session session2 = factory.openSession();
Employee e2 = (Employee) session1.get(Employee.class, Integer.valueOf(101));//passing same id

Transaction tx=session2.beginTransaction();
session2.merge(e1);

tx.commit();
session2.close();
</textarea></div>
<p>After closing session1, e1 is in detached state. It will not be in session1 cache. So if you call update() method, it will throw an error.</p>
<p>Then, we opened another session and loaded the same Employee instance. If we call merge in session2, changes of e1 will be merged in e2.</p>

<hr/>
<h3 class="h3">12) What are the states of object in hibernate?</h3>
<p>There are 3 states of object (instance) in hibernate.</p>
<ol>
<li><b>Transient</b>: The object is in transient state if it is just created but has no primary key (identifier) and not associated with session.</li>
<li><b>Persistent</b>: The object is in persistent state if session is open, and you just saved the instance in the database or retrieved the instance from the database.</li>
<li><b>Detached</b>: The object is in detached state if session is closed. After detached state, object comes to persistent state if you call lock() or update() method.</li>
</ol>


<hr/>
<h3 class="h3">13) What are the inheritance mapping strategies?</h3>
<p>There are 3 ways of inheritance mapping in hibernate.</p>
<ol>
<li>Table per hierarchy</li>
<li>Table per concrete class</li>
<li>Table per subclass</li>
</ol>
<a href="hibernate-inheritance-mapping-tutorial">more details...</a>

<hr/>
<h3 class="h3">14) How to make a immutable class in hibernate?</h3>
<p>If you mark a class as mutable="false", class will be treated as an immutable class. By default, it is mutable="true".</p>

<hr/>
<h3 class="h3">15) What is automatic dirty checking in hibernate?</h3>
<p>The automatic dirty checking feature of hibernate, calls update statement automatically on the objects that are modified in a transaction.</p>
<p>Let's understand it by the example given below:</p>
<div class="codeblock"><textarea name="code" class="java">
...
SessionFactory factory = cfg.buildSessionFactory();
Session session1 = factory.openSession();
Transaction tx=session2.beginTransaction();
 
Employee e1 = (Employee) session1.get(Employee.class, Integer.valueOf(101));
 
e1.setSalary(70000);
 
tx.commit();
session1.close();
</textarea></div>
<p>Here, after getting employee instance e1 and we are changing the state of e1.</p>
<p>After changing the state, we are committing the transaction. In such case, state will be updated automatically. This is known as dirty checking in hibernate.</p>

<hr/>
<h3 class="h3">16) How many types of association mapping are possible in hibernate?</h3>
<p>There can be 4 types of association mapping in hibernate.</p>
<ol>
<li>One to One</li>
<li>One to Many</li>
<li>Many to One</li>
<li>Many to Many</li>
</ol>
<hr/>
<h3 class="h3">17) Is it possible to perform collection mapping with One-to-One and Many-to-One?</h3>
<p>No, collection mapping can only be performed with One-to-Many and Many-to-Many</p>

<hr/>
<h3 class="h3">18) What is lazy loading in hibernate?</h3>
<p>Lazy loading in hibernate improves the performance. It loads the child objects on demand.</p>
<p>Since Hibernate 3, lazy loading is enabled by default, you don't need to do lazy="true". It means not to load the child objects when parent is loaded.</p>

<hr/>
<h3 class="h3">19) What is HQL (Hibernate Query Language)?</h3>
<p>Hibernate Query Language is known as an object oriented query language. It is like structured query language (SQL).</p>
<p>The main advantage of HQL over SQL is:</p>
<ol>
<li>You don't need to learn SQL</li>
<li>Database independent</li>
<li>Simple to write query</li>
</ol>

<hr/>
<h3 class="h3">20) What is the difference between first level cache and second level cache?</h3>
<table class="alt">
<tr><th>No.</th><th>First Level Cache</th><th>Second Level Cache</th></tr>
<tr><td>1)</td><td>First Level Cache is <b>associated with Session</b>.</td><td>Second Level Cache is associated with <b>SessionFactory</b>.</td></tr>
<tr><td>2)</td><td>It is <b>enabled</b> by default.</td><td>It is <b>not enabled</b> by default.</td></tr>
</table>
