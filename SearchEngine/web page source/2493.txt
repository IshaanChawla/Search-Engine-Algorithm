<h1 class="h1">Internationalization and Localization in Java</h1>
<div id="upr">
<ol>
<li><a href="#">Internationalization and Localization</a></li>
<li><a href="#interculture">Understanding the culturally dependent data</a></li>
<li><a href="#interlocale">Locale class</a></li>
<li><a href="#interlocaleex1">Example of Local class that prints the informations of the default locale</a></li>
<li><a href="#interlocaleex2">Example of Local class that prints english in different languages</a></li>
<li><a href="#interlocaleex3">Example of Local class that print display language of many locales</a></li>
 </ol>
</div>

<p><strong>Internationalization</strong> is also abbreviated as I18N because there are total 18 characters between the first letter 'I' and the last letter 'N'.</p>
<p>Internationalization is a mechanism to create such an application that can be adapted to different languages and regions.</p>
<p>Internationalization is one of the powerful concept of java if you are developing an application and want to display messages, currencies, date, time etc. according to the specific region or language.
</p>
<p><strong>Localization</strong> is also abbreviated as I10N because there are total 10 characters between the first letter 'L' and last letter 'N'. Localization is the mechanism to create such an application that can be adapted to a specific language and region by adding locale-specific text and component.
</p>

<br/><br/>
<fieldset>

<div id="youknow">

<ul>
<li>What is the use of Locale class ?</li>
<li>How can we globalize the messages (or) What is the use of ResourceBundle class?</li>
<li>How can we internationalize the date, time, number, currency and measurements?</li>
</ul>

</div>
</fieldset>

<a id="interculture"></a>
<h2 class="h2">Understanding the culturally dependent data before starting internationalization</h2>
<p>Before starting the internationalization, Let's first understand what are the informations that differ from one region to another. There is the list of culturally dependent data:</p>
<ul>
<li>Messages</li>
<li>Dates</li>
<li>Times</li>
<li>Numbers</li>
<li>Currencies</li>
<li>Measurements</li>
<li>Phone Numbers</li>
<li>Postal Addresses</li>
<li>Labels on GUI components etc.</li>
</ul>

<hr/>
<a id="interlocale"></a>
<h2 class="h2">Importance of Locale class in Internationalization</h2>
<p>An object of Locale class represents a geographical or cultural region. This object can be used to get the locale specific information such as country name, language, variant etc.
</p>
<h2 class="h2">Fields of Locale class</h2>
<p>There are fields of Locale class:</p>
<ol>
<li>public static final Locale ENGLISH</li>
<li>public static final Locale FRENCH</li>
<li>public static final Locale GERMAN</li>
<li>public static final Locale ITALIAN</li>
<li>public static final Locale JAPANESE</li>
<li>public static final Locale KOREAN</li>
<li>public static final Locale CHINESE</li>
<li>public static final Locale SIMPLIFIED_CHINESE</li>
<li>public static final Locale TRADITIONAL_CHINESE</li>
<li>public static final Locale FRANCE</li>
<li>public static final Locale GERMANY</li>
<li>public static final Locale ITALY</li>
<li>public static final Locale JAPAN</li>
<li>public static final Locale KOREA</li>
<li>public static final Locale CHINA</li>
<li>public static final Locale PRC</li>
<li>public static final Locale TAIWAN</li>
<li>public static final Locale UK</li>
<li>public static final Locale US</li>
<li>public static final Locale CANADA</li>
<li>public static final Locale CANADA_FRENCH</li>
<li>public static final Locale ROOT</li>
</ol>



<h2 class="h2">Constructors of Locale class</h2>
<p>There are three constructors of Locale class.They are as follows:</p>
<ol>
<li>Locale(String language)</li>
<li>Locale(String language, String country)</li>
<li>Locale(String language, String country, String variant)</li>
</ol>


<h2 class="h2">Commonly used methods of Locale class</h2>
<p>There are given commonly used methods of Locale class.</p>
<ol>
<li><strong>public static Locale getDefault()</strong> it returns the instance of current Locale</li>
<li><strong>public static Locale[] getAvailableLocales()</strong> it returns an array of available locales.</li>
<li><strong>public String getDisplayCountry()</strong> it returns the country name of this locale object.</li>
<li><strong>public String getDisplayLanguage()</strong> it returns the language name of this locale object.</li>
<li><strong>public String getDisplayVariant()</strong> it returns the variant code for this locale object.</li>
<li><strong>public String getISO3Country()</strong> it returns the three letter abbreviation for the current locale's country.</li>
<li><strong>public String getISO3Language()</strong> it returns the three letter abbreviation for the current locale's language.</li>

</ol>


<hr/>
<a id="interlocaleex1"></a>
<h3 class="h3ex">Example of Local class that prints the informations of the default locale</h3>
<p>In this example, we are displaying the informations of the default locale. If you want to get the informations about any specific locale, comment the first line statement and uncomment the second line statement in the main method.
</p>


<div class="codeblock"><textarea name="code" class="java">

import java.util.*;
public class LocaleExample {
public static void main(String[] args) {
Locale locale=Locale.getDefault();
//Locale locale=new Locale("fr","fr");//for the specific locale

System.out.println(locale.getDisplayCountry());
System.out.println(locale.getDisplayLanguage());
System.out.println(locale.getDisplayName());
System.out.println(locale.getISO3Country());
System.out.println(locale.getISO3Language());
System.out.println(locale.getLanguage());
System.out.println(locale.getCountry());
	
}
}


</textarea></div>

<a id="interlocaleex2"></a>
<h3 class="h3ex">Example of Local class that prints english in different languages</h3>
<p>In this example, we are displaying english language in different language. Let's see how english is written in french and spanish languages.
</p>

<div class="codeblock"><textarea  name="code" class="java">

import java.util.*;
public class LocaleExample2 {
	public static void main(String[] args) {
		Locale enLocale = new Locale("en", "US");
		Locale frLocale = new Locale("fr", "FR");
		Locale esLocale = new Locale("es", "ES");
		System.out.println("English language name (default): " + 
		                    enLocale.getDisplayLanguage());

		System.out.println("English language name in French: " + 
		                    enLocale.getDisplayLanguage(frLocale));
		System.out.println("English language name in spanish: " + 
                enLocale.getDisplayLanguage(esLocale));
	}

}

</textarea></div>
<div class="codeblock3"><textarea>
<strong>Output:</strong>English language name (default): English
       English language name in French: anglais
       English language name in spanish: ingl?s

</textarea></div>

<a id="interlocaleex3"></a>
<h3 class="h3ex">Example of Local class that print display language of many locales</h3>
<table >
<tr><td>In this example, we are displaying the display lanuage of many locales.
</td></tr>
</table>

<div class="codeblock"><textarea  name="code" class="java">

import java.util.*;
public class LocaleEx {
public static void main(String[] args) {
Locale[] locales = { new Locale("en", "US"),
 new Locale("es", "ES"), new Locale("it", "IT") }; 

for (int i=0; i< locales.length; i++) { 
 String displayLanguage = locales[i].getDisplayLanguage(locales[i]); 
 System.out.println(locales[i].toString() + ": " + displayLanguage); 
} 
}

}

</textarea></div>
<div class="codeblock3"><pre>
Output:en_US: English
       es_ES: espa?ol
       it_IT: italiano
