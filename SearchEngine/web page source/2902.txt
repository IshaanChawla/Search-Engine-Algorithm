		<h1 class="firstheading"><a href="http://www.codingunit.com/c-tutorial-command-line-parameter-parsing" rel="bookmark" title="C Tutorial &#8211; Command Line Parameter Parsing">
		C Tutorial &#8211; Command Line Parameter Parsing</a></h1>

			<div class="entry">
				<div id="watermark">
					<p>In this tutorial we take another look at command line parameter parsing with C programs. In a <a title="command line arguments and prototype functions" href="http://www.codingunit.com/c-tutorial-more-on-c-functions"><strong>previous command line parameter tutorial</strong></a> we already looked at some simple command line argument parsing example. Please read that tutorial before advancing below.<span id="more-225"></span></p>
<h2>More Complex Command Line Parameter Parsing Example</h2>
<p>Command-line parameters can be used for many things. Let&#8217;s take a look at another example that will print different things depending on the flags given. If the –f flag is used then the given argument is printed once.<br />
If the –d flag is used the argument is printed twice.</p>
<p>Let&#8217;s look at the example:</p>
<div id="cl">
<pre><code>
#include &lt;stdio.h&gt;
#include &lt;stdlib.h&gt;

void usage(void)
{
	printf("Usage:\n");
	printf(" -f&lt;name&gt;\n");
	printf(" -d&lt;name&gt;\n");
	exit (8);
}

int main(int argc, char *argv[])
{
	printf("Program name: %s\n", argv[0]);

	while ((argc &gt; 1) &amp;&amp; (argv[1][0] == '-'))
	{
		switch (argv[1][1])
		{
			case 'f':
				printf("%s\n",&amp;argv[1][2]);
				break;

			case 'd':
				printf("%s\n",&amp;argv[1][2]);
				printf("%s\n",&amp;argv[1][2]);
				break;

			default:
				printf("Wrong Argument: %s\n", argv[1]);
				usage();
		}

		++argv;
		--argc;
	}
	return (0);
}

</code></pre>
</div>
<p>After compiling you can run the program with the following parameters:</p>
<ul>
<li># program</li>
<li># program –fhello</li>
<li># program –dbye</li>
<li># program –fhello –dbye</li>
<li># program –w</li>
</ul>
<p><strong>Note:</strong> that there is no space between the flag and the flag argument.</p>
<h2>Closer Look at the Program</h2>
<p>First the program name is printed (it will always be printed.)</p>
<p>The &#8220;while loop&#8221; looks for the dash sign. In case of the –f the flag argument is printed once. If the flag is –d then the flag argument is printed twice.</p>
<p>The argv[][] array will be used as follows:</p>
<p>For example: –f&lt;name&gt;</p>
<ul>
<li>argv[1][0] contains the dash sign</li>
<li>argv[1][1] contains the &#8220;f&#8221;</li>
<li>argv[1][2] contains the flag argument name</li>
</ul>
<p>If a wrong flag sign is given then the usage is printed onto the screen.</p>
<h2>Experiment Yourself</h2>
<p>If you want to experiment further with command line parsing then you should expand the program above with the ability to add a space between the flag and the flag argument. Another thing you could add is parsing the program name to the usage function and prints it from this function.</p>
<p>That&#8217;s all for this tutorial.</p>
