<div class="accordion-div">
  <h1 class="book_part"><a name="overview"><b>Overview and Course Options</b></a></h1>
<div> <!-- Accordion Content -->
  <p>Most new apps should be developed with the Java-EE-standard JSF 2 (usually with PrimeFaces), not with the older and
    lower-level servlet and JSP libraries. However, there are two situations where organizations still use
    servlets and JSP:</p>
  <ul>
    <li>When maintaining and extending successful legacy apps that were built with servlets and JSP</li>
    <li>When using direct JavaScript libraries on the client side (e.g., jQuery, Ext/JS or Dojo)</li>
  </ul>
  <p>If either of these situations describe your project, please <a href="mailto:hall@coreservlets.com">contact Marty Hall</a>
    to arrange a course at your location that is customized to your needs. Marty is the author of the world's alltime
    bestselling servlet and JSP text (Core Servlets and JSP) and has taught Java EE in 10 countries and dozens of US venues.
    Here are some of the possible course options, but of course things can be customized further to fit your needs. </p>
  <ul>
    <li><b>Intermediate Servlets and JSP.</b> For developers that know Java already and are working on full servlet/JSP
      applications.</li>
    <li><b>Intermediate Servlets and JSP with Java.</b> For developers that do not yet know Java, 
      but need to work working on full servlet/JSP applications. THis is a single course that includes 
      an abbreviated version of the <a href="java-training.html">Java programming crash course</a>, followed by an   
      abbreviated version of the first course above.</li>
    <li><b>Ajax.</b> This is a course that looks at Ajax in JavaScript (usually with jQuery), and uses Java on
      the server to answer to Ajax requests. More details on this course are at 
      <a href="http://courses.coreservlets.com/ajax-training.html">the Ajax training page</a>.</li>
    <li><b>Servlets for JSF Developers.</b> This is a course for those that are experienced with JSF development
      and want to learn the underlying servlet API to give them even more control over the app.</li>
  </ul>
  Email <a href="mailto:hall@coreservlets.com">hall@coreservlets.com</a> to discuss
  which options would work best for your developers.
  
</div> <!-- Accordion Content -->

<h1 class="book_part"><a name="audience"><b>Intended Audience</b></a></h1>
<div> <!-- Accordion Content -->
  All of the servlet and JSP courses presume that the students have moderate or extensive previous Java
  experience. The courses move much too fast for newcomers to Java. If your developers do not already know Java,
  then you should start with one of the 
  <a href="http://courses.coreservlets.com/java-training.html">crash courses in Java programming</a>,
  then move on to servlets and JSP after that.
</div> <!-- Accordion Content -->

<h1 class="book_part"><a name="syllabus"><b>Syllabus Choices</b></a></h1>
<div> <!-- Accordion Content -->
  Here is a potpourri of possible topics. As discussed above, the topics covered in any course are customizable, 
  but the most common options are: core servlets and JSP, core servlets and JSP plus Java, advanced servlets and JSP,
  Ajax with Java on the server, and servlets for JSF developers. 
  When you book a course, we will first decide on the exact topics based on your needs 
  and level of experience.  Email <a href="mailto:hall@coreservlets.com">hall@coreservlets.com</a> to inquire
  about a custom course at your location.
  <h2 class="topics">Overview and Setup</h2>
  <ul>
    <li>Understanding the role of servlets</li>
    <li>Evaluating servlets vs. other technologies</li>
    <li>Understanding the role of JSP</li>
    <li>Configuring the server</li>
    <li>Configuring your development environment</li>
    <li>Testing the server setup</li>
    </ul>
  <h2 class="topics">Servlet Basics</h2>
  <ul>
    <li>The basic structure of servlets</li>
    <li>A simple servlet that generates plain text</li>
    <li>A servlet that generates HTML</li>
    <li>Servlets and packages</li>
    <li>Some utilities that help build HTML</li>
    <li>The servlet life cycle</li>
    <li>Servlet debugging strategies</li>
    </ul>
  <h2 class="topics">Handling the Client Request: Form Data</h2>
  <ul>
    <li>The role of form data</li>
    <li>Creating and submitting HTML forms</li>
    <li>Reading individual request parameters</li>
    <li>Reading the entire set of request parameters</li>
    <li>Handling missing and malformed data</li>
    <li>Dealing with incomplete form submissions</li>
    <li>Filtering special characters out of the request parameters</li>
    </ul>
  <h2 class="topics">Handling the Client Request: HTTP Request Headers</h2>
  <ul>
    <li>Reading HTTP request headers</li>
    <li>Building a table of all the request headers</li>
    <li>Understanding the various request headers</li>
    <li>Reducing download times by compressing pages</li>
    <li>Differentiating among types of browsers</li>
    </ul>
  <h2 class="topics">Generating the Server Response: HTTP Status Codes</h2>
  <ul>
    <li>Format of the HTTP response</li>
    <li>How to set status codes</li>
    <li>What the status codes are good for</li>
    <li>Shortcut methods for redirection and error pages</li>
    <li>A servlet that redirects users to browser-specific pages</li>
    <li>A front end to various search engines</li>
    </ul>
  <h2 class="topics">Generating the Server Response: HTTP Response Headers</h2>
  <ul>
    <li>Format of the HTTP response</li>
    <li>Setting response headers</li>
    <li>Understanding what response headers are good for</li>
    <li>Building Excel spread sheets</li>
    <li>Generating JPEG images dynamically</li>
    <li>Sending incremental updates to the browser</li>
    </ul>
  <h2 class="topics">Handling Cookies</h2>
  <ul>
    <li>Understanding the benefits and drawbacks of cookies</li>
    <li>Sending outgoing cookies</li>
    <li>Receiving incoming cookies</li>
    <li>Tracking repeat visitors</li>
    <li>Specifying cookie attributes</li>
    <li>Differentiating between session cookies and persistent cookies</li>
    <li>Simplifying cookie usage with utility classes</li>
    <li>Modifying cookie values</li>
    <li>Remembering user preferences</li>
    </ul>
  <h2 class="topics">Session Tracking</h2>
  <ul>
    <li>Implementing session tracking from scratch</li>
    <li>Using basic session tracking</li>
    <li>Understanding the session-tracking API</li>
    <li>Differentiating between server and browser sessions</li>
    <li>Encoding URLs</li>
    <li>Storing immutable objects vs. storing mutable objects</li>
    <li>Tracking user access counts</li>
    <li>Accumulating user purchases</li>
    <li>Implementing a shopping cart</li>
    <li>Building an online store</li>
    </ul>
  <h2 class="topics">JSP Intro and Overview</h2>
  <ul>
    <li>Understanding the need for JSP</li>
    <li>Evaluating the benefits of JSP</li>
    <li>Comparing JSP to other technologies</li>
    <li>Avoiding JSP misconceptions</li>
    <li>Understanding the JSP lifecycle</li>
    <li>Installing JSP pages</li>
    <li>Looking at JSP in the real world</li>
    </ul>
  <h2 class="topics">Invoking Java Code with JSP Scripting Elements</h2>
  <ul>
    <li>Static vs. dynamic text</li>
    <li>Dynamic code and good JSP design</li>
    <li>JSP expressions</li>
    <li>Servlets vs. JSP pages for similar tasks</li>
    <li>JSP scriptlets</li>
    <li>JSP declarations</li>
    <li>Predefined variables</li>
    <li>Comparison of expressions, scriptlets, and declarations</li>
    </ul>
  <h2 class="topics">Controlling the Structure of Generated Servlets:
    The JSP page Directive</h2>
  <ul>
    <li>Understanding the purpose of the page directive</li>
    <li>Designating which classes are imported</li>
    <li>Specifying the MIME type of the page</li>
    <li>Generating Excel spreadsheets</li>
    <li>Participating in sessions</li>
    <li>Setting the size and behavior of the output buffer</li>
    <li>Designating pages to handle JSP errors</li>
    <li>Controlling threading behavior</li>
    </ul>
  <h2 class="topics">Including Files and Applets in JSP Pages</h2>
  <ul>
    <li>Using jsp:include to include pages at request time</li>
    <li>Using &lt;%@ include ... %&gt; (the include directive)
      to include files at page translation time</li>
    <li>Understanding why jsp:include is usually
      better than the include directive</li>
    <li>Using jsp:plugin to include applets for the Java Plug-in</li>
    </ul>
  <h2 class="topics">Using JavaBeans Components in JSP Documents</h2>
  <ul>
    <li>Understanding the benefits of beans</li>
    <li>Creating beans</li>
    <li>Installing bean classes on your server</li>
    <li>Accessing bean properties</li>
    <li>Explicitly setting bean properties</li>
    <li>Automatically setting bean properties from request parameters</li>
    <li>Sharing beans among multiple servlets and JSP pages</li>
    </ul>
  <h2 class="topics">Integrating Servlets and JSP:
    The Model View Controller (MVC) Architecture</h2>
  <ul>
    <li>Understanding the benefits of MVC</li>
    <li>Using RequestDispatcher to implement MVC</li>
    <li>Forwarding requests from servlets to JSP pages</li>
    <li>Handling relative URLs</li>
    <li>Choosing among different display options</li>
    <li>Comparing data-sharing strategies</li>
    </ul>
  <h2 class="topics">Simplifying Access to Java Code:
    The JSP 2.0 Expression Language</h2>
  <ul>
    <li>Motivating use of the expression language</li>
    <li>Understanding the basic syntax</li>
    <li>Understanding the relationship of the
      expression language to the MVC architecture</li>
    <li>Referencing scoped variables</li>
    <li>Accessing bean properties, array elements,
      List elements, and Map entries</li>
    <li>Using expression language operators</li>
    <li>Evaluating expressions conditionally</li>
    </ul>
  <h2 class="topics">Using and Deploying Web Applications</h2>
  <ul>
    <li>Purpose of Web applications</li>
    <li>Structure of Web applications</li>
    <li>Setting up Web applications with Tomcat</li>
    <li>Giving custom addresses to servlets</li>
    <li>Sharing data among Web applications</li>
    </ul>
  <h2 class="topics">Controlling Web Application Behavior with web.xml</h2>
  <ul>
    <li>Location and purpose of web.xml</li>
    <li>Custom URLs</li>
    <li>Initialization parameters</li>
    <li>Preloading pages</li>
    <li>Welcome pages</li>
    <li>Error pages</li>
    </ul>
  <h3 class="topics">Declarative Web Application Security</h3>
  <ul>
    <li>Major security concerns</li>
    <li>Declarative vs. programmatic security</li>
    <li>Using form-based authentication</li>
    <li>Using BASIC authentication</li>
    </ul>
  <h3 class="topics">Programmatic Security</h3>
  <ul>
    <li>Problems with declarative security</li>
    <li>Combination security: mixing
      server-managed and servlet-managed
      (programmatic) security</li>
    <li>Pure programmatic security</li>
    </ul>
  <h2 class="topics">Servlet and JSP Filters</h2>
  <ul>
    <li>Filter basics</li>
    <li>Accessing the servlet context</li>
    <li>Using initialization parameters</li>
    <li>Blocking responses</li>
    <li>Modifying responses</li>
    </ul>
  <h2 class="topics">Web Application Lifecyle Listeners</h2>

    <li>Detecting changes in servlet context attributes</li>
    <li>Recognizing session creation and destruction</li>
    <li>Watching for changes in session attributes</li>
    <li>Combining activities</li>
    </ul>
  <h2 class="topics">Creating Custom JSP Tag Libraries: The Basics</h2>
  <ul>
