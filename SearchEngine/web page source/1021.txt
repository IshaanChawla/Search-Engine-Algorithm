<div style="color:#000080;font-family:Arial;font-size:10pt">
<p>The <b>HashMap </b>class uses a hash table to implement the <b>Map
</b>interface. This allows the execution time of basic operations, such as <b>get() </b>and <b>put()</b>, to remain constant even for
large sets.
<br>The following constructors are defined:
<code>HashMap( )
<br>HashMap(Map <i>m</i>)
<br>HashMap(int <i>capacity</i>)
<br>HashMap(int <i>capacity</i>, float <i>fillRatio</i>)</code>
<p>The first form constructs a default hash map. The second form
initializes the hash map by using the elements of <i>m</i>. The third form initializes the
capacity of the hash map to
<i>
capacity</i>. The fourth form initializes both the capacity and
fill ratio of the hash map by using its arguments. The meaning of capacity and fill ratio is
the same as for <b>HashSet</b>, described earlier.</p>
<p>
<b>
HashMap </b>implements <b>Map </b>and extends <b>AbstractMap</b>.
It does not add any methods of its own. You should note that a hash map does <i>not </i>guarantee the
order of its elements. Therefore, the order in which elements are added to a hash map
is not necessarily the order in which they are read by an iterator.</p>
<p>The following program illustrates <b>HashMap</b>. It maps names
to account balances. Notice how a set-view is obtained and used.</p>
<code>
import java.util.*;
<br>class HashMapDemo {
<br>public static void main(String args[]) {
<br>// Create a hash map
<br>HashMap hm = new HashMap();
<br>// Put elements to the map
<br>hm.put(&quot;John Doe&quot;, new Double(3434.34));
<br>hm.put(&quot;Tom Smith&quot;, new Double(123.22));
<br>hm.put(&quot;Jane Baker&quot;, new Double(1378.00));
<br>hm.put(&quot;Todd Hall&quot;, new Double(99.22));
<br>hm.put(&quot;Ralph Smith&quot;, new Double(-19.08));
<br>// Get a set of the entries
<br>Set set = hm.entrySet();
<br>// Get an iterator
<br>Iterator i = set.iterator();
<br>// Display elements
<br>while(i.hasNext()) {
<br>Map.Entry me = (Map.Entry)i.next();
<br>System.out.print(me.getKey() + &quot;: &quot;);
<br>System.out.println(me.getValue());
<br>}
<br>System.out.println();
<br>// Deposit 1000 into John Doe's account
<br>double balance = ((Double)hm.get(&quot;John Doe&quot;)).doubleValue();
<br>hm.put(&quot;John Doe&quot;, new Double(balance + 1000));
<br>System.out.println(&quot;John Doe's new balance: &quot; +
<br>hm.get(&quot;John Doe&quot;));
<br>}
<br>}</p>
</code>
<p>Output from this program is shown here:</p>
<code>Ralph Smith: -19.08
<br>Tom Smith: 123.22
<br>John Doe: 3434.34
<br>Todd Hall: 99.22
<br>Jane Baker: 1378.0
<br>John Doe's current balance: 4434.34</code>
<p>The program begins by creating a hash map and then adds the
mapping of names to balances. Next, the contents of the map are displayed by using a
set-view, obtained by calling <b>entrySet()</b>. The keys and values are displayed by
calling the <b>getKey() </b>and
<b>
getValue() </b>methods that are defined by <b>Map.Entry</b>.
Pay close attention to how the deposit is made into John Doe's account. The <b>put() </b>method
automatically replaces any preexisting value that is associated with the specified key with
the new value. Thus, after John Doe's account is updated, the hash map will still contain
just one &quot;John Doe&quot; account.</p>
