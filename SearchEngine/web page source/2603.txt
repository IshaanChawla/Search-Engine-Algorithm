               <div>
                  <div>
                     <h2 class="title"><a name="javaannotations"></a>1.&nbsp;Annotations in Java
                     </h2>
                  </div>
               </div>
            </div><a class="indexterm" name="d412743e45"></a><div class="section">
               <div class="titlepage">
                  <div>
                     <div>
                        <h3 class="title"><a name="annotations_definition"></a>1.1.&nbsp;Annotations in Java
                        </h3>
                     </div>
                  </div>
               </div>
               <p>
                  Annotations provide data about a class that is not part of the
                  programming logic itself. They have no direct effect on the
                  code they
                  annotate. Other components can use this information.
                  
               </p>
               <p>
                  Annotations can
                  be preserved at runtime (<code class="code">RetentionPolicy.RUNTIME</code>)
                  or are only available at development time (<code class="code">RetentionPolicy.SOURCE</code>).
                  
               </p>
            </div>
            <div class="section">
               <div class="titlepage">
                  <div>
                     <div>
                        <h3 class="title"><a name="annotation_default"></a>1.2.&nbsp;Standard annotations in Java
                        </h3>
                     </div>
                  </div>
               </div>
               <div class="section">
                  <div class="titlepage">
                     <div>
                        <div>
                           <h4 class="title"><a name="annotation_override"></a>1.2.1.&nbsp;@Override
                           </h4>
                        </div>
                     </div>
                  </div>
                  The
                  <code class="code">@Override</code>
                  annotation can be added to a method. It is used by the Java compiler
                  to check if the annotated
                  method really overrides a method of an
                  interface or the extended
                  class.
                  
               </div>
               <div class="section">
                  <div class="titlepage">
                     <div>
                        <div>
                           <h4 class="title"><a name="annotation_deprecated"></a>1.2.2.&nbsp;@Deprecated
                           </h4>
                        </div>
                     </div>
                  </div>
                  The
                  <code class="code">@Deprecated</code>
                  annotation
                  can be used on a field, method, constructor or class and
                  indicates
                  that this element is outdated and should not be used
                  anymore. Adding
                  <code class="code">@Deprecated</code>
                  to the class does not deprecate automatically all its fields and
                  methods.
                  
               </div>
            </div>
            <div class="section">
               <div class="titlepage">
                  <div>
                     <div>
                        <h3 class="title"><a name="annotation_typeannotations"></a>1.3.&nbsp;Type annotations
                        </h3>
                     </div>
                  </div>
               </div>
               <p>Java 8 supports that annotations can be place on any type. The
                  following gives several examples assuming the annotations exists.
                  
               </p>
               <p>
                  
               </p><pre class="programlisting"><em><span class="hl-annotation" style="color: gray">@NonNull</span></em> String name;
List&lt;<em><span class="hl-annotation" style="color: gray">@NonNull</span></em> String&gt; names;
<span class="hl-keyword">class</span> UnmodifiableList&lt;T&gt; <span class="hl-keyword">implements</span> <em><span class="hl-annotation" style="color: gray">@Readonly</span></em> List&lt;<em><span class="hl-annotation" style="color: gray">@Readonly</span></em> T&gt; {...}
email = (<em><span class="hl-annotation" style="color: gray">@Email</span></em> String) input;

<span class="hl-keyword">void</span> doSomething() <span class="hl-keyword">throws</span> <em><span class="hl-annotation" style="color: gray">@ImportantForMe</span></em> MyException { ... } </pre><p>

                  <div>
                     <div>
                        <h3 class="title"><a name="customannotations_definition"></a>2.1.&nbsp;Define your custom annotation
                        </h3>
                     </div>
                  </div>
               </div>
               <p>
                  The Java programming language allows you to
                  define your
                  custom
                  annotations.
                  
               </p>
               <p>
                  Annotations are defined via the
                  <code class="code">@interface</code>
                  annotation before the class name. Via
                  <code class="code">@Retention</code>
                  you define if the annotation should be retained at runtime or not.
                  The
                  <code class="code">@Target</code>
                  annotation
                  lets you define where this annotation can be used, e.g. the
                  class,
                  fields,
                  methods, etc.
                  
               </p>
               <p>A typical annotation definition would look like the following.
                  
               </p>
               <p>
                  
               </p><pre class="programlisting"><em><span class="hl-annotation" style="color: gray">@Retention(RetentionPolicy.RUNTIME)</span></em>
<em><span class="hl-annotation" style="color: gray">@Target(ElementType.METHOD)</span></em>
<span class="hl-keyword">public</span> <em><span class="hl-annotation" style="color: gray">@interface</span></em> InstallerMethod {
} </pre><p>
                  
               </p>
               <p>
                  You can also define that your annotation is a qualifier for the
                  <code class="code">@Inject</code>
                  annotation.
                  
               </p>
               <p>
                  
               </p><pre class="programlisting"><em><span class="hl-annotation" style="color: gray">@javax.inject.Qualifier</span></em>
<em><span class="hl-annotation" style="color: gray">@Documented</span></em>
<em><span class="hl-annotation" style="color: gray">@Target({ElementType.FIELD, ElementType.PARAMETER})</span></em>
<em><span class="hl-annotation" style="color: gray">@Retention(RetentionPolicy.RUNTIME)</span></em>
<span class="hl-keyword">public</span> <em><span class="hl-annotation" style="color: gray">@interface</span></em> Checker {

} </pre><p>
                  
               </p>
            </div>

                        <h3 class="title"><a name="customannotations_usage"></a>2.2.&nbsp;Accessing your annotation via Java reflection
                        </h3>
                     </div>
                  </div>

               <p>
                  To process your annotation you could write your own annotation
                  processor. Typically you use Java reflection for this. Java
                  reflection
                  allows you to analyze a Java class and use the information
                  contained
                  in this class at runtime.
                  
               </p>
               <p>
                  See
                  <a class="xref" href="#annotations_tutorial" title="4.&nbsp;Exercise: Define and analyze your own annotation">Section&nbsp;4, &#8220;Exercise: Define and analyze your own annotation&#8221;</a> for an example. 
                  
               </p>

                  <div>
                     <h2 class="title"><a name="annotation_repeatable"></a>3.&nbsp;Repeatable annotations in Java 8
                     </h2>
                  </div>
               </div>
            </div>
            <p>As of Java 8 you can repeat the same annotation several times at
               the same place.
               
            </p>
            <p>For this you need to define your annotation and define that they
               can be repeated. You need two definitions for that.
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> com.vogella.java.annotations.repeatable;

<span class="hl-keyword">import</span> java.lang.annotation.Repeatable;

<em><span class="hl-annotation" style="color: gray">@Repeatable(TimeEstimate.class)</span></em>
<span class="hl-keyword">public</span> <em><span class="hl-annotation" style="color: gray">@interface</span></em> Duration {
  <span class="hl-keyword">int</span> hour() <span class="hl-keyword">default</span> <span class="hl-number">1</span>;
} </pre><p>
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> com.vogella.java.annotations.repeatable;



<span class="hl-keyword">public</span> <em><span class="hl-annotation" style="color: gray">@interface</span></em> TimeEstimate {
  Duration[] value();
} </pre><p>
               
            </p>
            <p>Afterwards you can use this annotation in your code several
               times.
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> com.vogella.java.annotations.repeatable;

<em><span class="hl-annotation" style="color: gray">@Duration(hour= 2)</span></em>
<em><span class="hl-annotation" style="color: gray">@Duration</span></em>
<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> TestClass {

} </pre><p>
               
            </p>

               <div>
                  <div>
                     <h2 class="title"><a name="annotations_tutorial"></a>4.&nbsp;Exercise: Define and analyze your own annotation
                     </h2>
                  </div>
               </div>
            </div>
            <p>
               Create a new Java project called
               <em class="wordasword">com.vogella.annotations</em>.
               
            </p>
            <p>Create the following two classes. The first class defines an
               annotation
               and the second class uses this to mark certain methods.
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> com.vogella.annotations;

<span class="hl-keyword">import</span> java.lang.annotation.ElementType;
<span class="hl-keyword">import</span> java.lang.annotation.Retention;
<span class="hl-keyword">import</span> java.lang.annotation.RetentionPolicy;
<span class="hl-keyword">import</span> java.lang.annotation.Target;

<em><span class="hl-annotation" style="color: gray">@Target(value = ElementType.METHOD)</span></em>
<em><span class="hl-annotation" style="color: gray">@Retention(value = RetentionPolicy.RUNTIME)</span></em>
<span class="hl-keyword">public</span> <em><span class="hl-annotation" style="color: gray">@interface</span></em> CanRun {

} </pre><p>
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> com.vogella.annotations;

<span class="hl-keyword">import</span> java.lang.reflect.Method;

<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> AnnotationRunner {

    <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> method1() {
        System.out.println(<span class="hl-string" style="color:blue">"method1"</span>);
    }

    <em><span class="hl-annotation" style="color: gray">@CanRun</span></em>
    <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> method2() {
        System.out.println(<span class="hl-string" style="color:blue">"method2"</span>);
    }

    <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> method3() {
        System.out.println(<span class="hl-string" style="color:blue">"method3"</span>);
    }
    
    <em><span class="hl-annotation" style="color: gray">@CanRun</span></em>
    <span class="hl-keyword">public</span> <span class="hl-keyword">void</span> method5() {
        System.out.println(<span class="hl-string" style="color:blue">"method4"</span>);
    }

} </pre><p>
               
            </p>
            <p>Afterwards create the following test class.
               The
               main method of this
               class analyzes the annotations and calls the
               corresponding methods.
               
            </p>
            <p>
               
            </p><pre class="programlisting"><span class="hl-keyword">package</span> com.vogella.annotations;

<span class="hl-keyword">import</span> java.lang.reflect.Method;

<span class="hl-keyword">public</span> <span class="hl-keyword">class</span> MyTest {

    <span class="hl-keyword">public</span> <span class="hl-keyword">static</span> <span class="hl-keyword">void</span> main(String[] args) {

        AnnotationRunner runner = <span class="hl-keyword">new</span> AnnotationRunner();
        Method[] methods = runner.getClass().getMethods();

        <span class="hl-keyword">for</span> (Method method : methods) {
            CanRun annos = method.getAnnotation(CanRun.<span class="hl-keyword">class</span>);
            <span class="hl-keyword">if</span> (annos != null) {
                <span class="hl-keyword">try</span> {
                    method.invoke(runner);
                } <span class="hl-keyword">catch</span> (Exception e) {
                    e.printStackTrace();
                }
            }
        }
    }
} </pre><p>
               
            </p>
