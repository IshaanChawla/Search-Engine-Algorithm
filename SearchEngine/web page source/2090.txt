<br />This is the first of three sections on network programming in Java. In this section we cover the basics of creating client sockets to connect to existing servers, and we also take an aside to summarize the basic ways of formatting data to send over the socket (i.e., printf) and reading the results (i.e., using StringTokenizer and String.split on the result of readLine).     Client vs. Server
&#x2022; Traditional definition
&#x2013; Client: User of network services
&#x2013; Server: Supplier of network services
&#x2022; Problem with traditional definition
&#x2013; If there are 2 programs exchanging data, it seems unclear
&#x2013; Some situations (e.g., X Windows) seem reversed
&#x2022; Easier way to remember distinction
&#x2013; Server starts first. Server doesn&#x2019;t specify host (just port).
&#x2013; Client starts second. Client specifies host (and port).
&#x2022; Analogy: Company phone line
&#x2013; Installing phone is like starting server
&#x2013; Extension is like port
&#x2013; Person who calls is the client: he specifies both host
(general company number) and port (extension)7

    Parsing Strings using the split
method of String
&#x2022; Basic usage
&#x2013; String[] tokens = mainString.split(delimiterString);
&#x2022; Differences from StringTokenizer
&#x2013; Entire string is the delimiter (not one-char delimiters)
&#x2022; "foobar".split("ob") returns {"fo", "ar"}
&#x2022; "foobar".split("bo") returns {"foobar"}
&#x2013; You can use regular expressions in the delimiter
&#x2022; ^, $, *, +, ., etc for beginning of String, end of String, 0 or
more, 1 or more, any one character, etc.
&#x2022; See http://docs.oracle.com/javase/8/docs/api/java/util/regex/Pattern.html#sum
&#x2013; Unless you use "+", an empty string is returned between
consecutive delimiters
&#x2022; "foobar".split("o") returns {"f", "", "bar"}
&#x2022; "foobar".split("o+") returns {"f", "bar"}24

&#x2013; You can connect to hosts that are using virtual hosting
&#x2022; I.e., sites that host multiple domain names on the same
machine, which account for most small and medium sites

HTTP 1.0
Connect to somehost on port 80
GET /somepath HTTP/1.0
Blank line
HTTP 1.1
Connect to somehost on port 80
GET /somepath HTTP/1.1
Host: somehost
Connection: close
Blank line
