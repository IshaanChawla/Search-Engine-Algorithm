<p>There are two almost identical ways to access characters in a string.  The easier to use and faster version is the overloaded operator[]:</p>
<table border=1 cellspacing=0 cellpadding=3 width=100%>
<tr>
<td>
<b>char&#038; string::operator[] (size_type nIndex)</b><br />
<b>const char&#038; string::operator[] (size_type nIndex) const</b></p>
<ul>
<li>Both of these functions return the character with index nIndex
<li>Passing an invalid index results in undefined behavior
<li>Using length() as the index is valid for const strings only, and returns the value generated by the string&#8217;s default constructor.  It is not recommended that you do this.
<li>Because char&#038; is the return type, you can use this to edit characters in the array
</ul>

<p><strong>Conversion to C-style arrays</strong></p>
<p>Many functions (including all C functions) expect strings to be formatted as C-style strings rather than std::string.  For this reason, std::string provides 3 different ways to convert std::string to C-style strings.</p>
<table border=1 cellspacing=0 cellpadding=3 width=100%>
<tr>
<td>
<b>const char* string::c_str () const</b></p>
<ul>
<li>Returns the contents of the string as a const C-style string
<li>A null terminator is appended
<li>The C-style string is owned by the std::string and should not be deleted
</ul>

		<p>Hi, Yang.</p>
<p>       Let us see whether the aforementioned paragraph is correct or not:</p>
<p>       &#8220;Using length() as the index is valid for const strings only, and returns the value generated by the strings default constructor. It is not recommended that you do this.&#8221;</p>
<p>       Let us write the following sample program:</p>
<p><code><br />
#include &lt;iostream&gt;<br />
#include &lt;string&gt;</p>
<p>using namespace std;</p>
<p>int main()<br />
{<br />
    const string scSource(&quot;abcdefg&quot;);<br />
    string::size_type nLength = scSource.length();<br />
    cout &lt;&lt; &quot;Const string is:             [&quot;<br />
         &lt;&lt; scSource                  &lt;&lt; &quot;]&quot; &lt;&lt; endl;<br />
    cout &lt;&lt; &quot;Const string length is:      [&quot;<br />
         &lt;&lt; nLength                   &lt;&lt; &quot;]&quot; &lt;&lt; endl;<br />
    cout &lt;&lt; &quot;Last character is:           [&quot;<br />
         &lt;&lt; scSource[nLength-1]       &lt;&lt; &quot;]&quot; &lt;&lt; endl;<br />
    cout &lt;&lt; &quot;Using length() returns:      [&quot;<br />
         &lt;&lt; scSource[nLength]         &lt;&lt; &quot;]&quot; &lt;&lt; endl &lt;&lt; endl &lt;&lt; endl;</p>
<p>    string sSource(&quot;hijklmn&quot;);<br />
    nLength = sSource.length();<br />
    cout &lt;&lt; &quot;Non-const string is:         [&quot;<br />
         &lt;&lt; sSource                   &lt;&lt; &quot;]&quot; &lt;&lt; endl;<br />
    cout &lt;&lt; &quot;Non-const string length is : [&quot;<br />
         &lt;&lt; nLength                   &lt;&lt; &quot;]&quot; &lt;&lt; endl;<br />
    cout &lt;&lt; &quot;Last character is:           [&quot;<br />
         &lt;&lt; sSource[nLength-1]        &lt;&lt; &quot;]&quot; &lt;&lt; endl;<br />
    cout &lt;&lt; &quot;Using length() returns:      [&quot;<br />
         &lt;&lt; sSource[nLength]          &lt;&lt; &quot;]&quot; &lt;&lt; endl;</p>
<p>    return 0;<br />
}<br />
</code></p>
<p>       The output of this sample program is:</p>
<p><code><br />
Const string is:             [abcdefg]<br />
Const string length is:      [7]<br />
Last character is:           [g]<br />
Using length() returns:      [ ]</p>
<p>Non-const string is:         [hijklmn]<br />
Non-const string length is : [7]<br />
Last character is:           [n]<br />
Using length() returns:      [ ]<br />
</code></p>
<p>       As you can see, the string::operator[] is applicable using the length() function for both const strings and non-const strings - unlike specified in the aforementioned paragraph.</p>
<p>       However, the results are unpredictable. In the output produced above using the Code::Blocks Integrated Development Environment and the gcc compiler, nothing was returned in both cases. This is probably why length()&#8217;s usage is not recommended.</p>
