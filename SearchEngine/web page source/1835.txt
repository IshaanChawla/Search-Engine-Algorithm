<h1 class="style2">Introduction to Threads</h1>
<p>Multithreading refers to two or more tasks executing concurrently within a single program. A thread is an independent path of execution within a program. Many threads can run concurrently within a program. Every thread in Java is created and controlled by the <strong>java.lang.Thread class</strong>. A Java program can have many threads, and these threads can run concurrently, either asynchronously or synchronously.</p>
<p class="style5">Multithreading has several advantages over Multiprocessing such as;</p>
<ul>
<li>Threads are lightweight compared to processes</li>
<li>Threads share the same address space and therefore can share both data and code</li>
<li>Context switching between threads is usually less expensive than between processes</li>
<li>Cost of thread intercommunication is relatively low that that of process intercommunication</li>
<li>Threads allow different tasks to be performed concurrently.</li>
</ul>
<p><strong>The following figure shows the methods that are members of the Object and Thread Class.</strong></p>
<p><img alt="" src="http://www.javabeginner.com/images/table.JPG" width="296" height="116" /></p>
<h2>Thread Creation</h2>
<p class="style11">There are two ways to create thread in java;</p>
<ul>
<li>Implement the Runnable interface (java.lang.Runnable)</li>
<li>By Extending the Thread class (java.lang.Thread)</li>
</ul>
<h2><span class="style1">Implementing the Runnable Interface</span></h2>
<p>&nbsp;</p>
<p><a onclick="javascript:pageTracker._trackPageview('/outgoing/www.wideskills.com');"  href="http://www.wideskills.com"><img class="aligncenter size-full wp-image-572" alt="wideskills" src="http://www.javabeginner.com/wp-content/uploads/2015/02/wideskills-3.png" width="499" height="154" /></a></p>
<p class="text5"><strong>The Runnable Interface Signature</strong></p>
<p>public interface Runnable {</p>
<p>void run();</p>
<p>}</p>
<p>One way to create a thread in java is to implement the Runnable Interface and then instantiate an object of the class. We need to override the run() method into our class which is the only method that needs to be implemented. The run() method contains the logic of the thread.</p>
<p class="style11"><strong>The procedure for creating threads based on the Runnable interface is as follows:</strong></p>
<p>1. A class implements the Runnable interface, providing the run() method that will be executed by the thread. An object of this class is a Runnable object.</p>
<p>2. An object of Thread class is created by passing a Runnable object as argument to the Thread constructor. The Thread object now has a Runnable object that implements the run() method.</p>
<p>3. The start() method is invoked on the Thread object created in the previous step. The start() method returns immediately after a thread has been spawned.</p>
<p>4. The thread ends when the run() method ends, either by normal completion or by throwing an uncaught exception.</p>
<p>Below is a program that illustrates instantiation and running of threads using the runnable interface instead of extending the Thread class. To start the thread you need to invoke the <strong>start()</strong> method on your object.</p>
<pre class="wp-code-highlight prettyprint">
class RunnableThread implements Runnable {

	Thread runner;
	public RunnableThread() {
	}
	public RunnableThread(String threadName) {
		runner = new Thread(this, threadName); // (1) Create a new thread.
		System.out.println(runner.getName());
		runner.start(); // (2) Start the thread.
	}
	public void run() {
		//Display info about this particular thread
		System.out.println(Thread.currentThread());
	}
}

public class RunnableExample {

	public static void main(String[] args) {
		Thread thread1 = new Thread(new RunnableThread(), &quot;thread1&quot;);
		Thread thread2 = new Thread(new RunnableThread(), &quot;thread2&quot;);
		RunnableThread thread3 = new RunnableThread(&quot;thread3&quot;);
		//Start the threads
		thread1.start();
		thread2.start();
		try {
			//delay for one second
			Thread.currentThread().sleep(1000);
		} catch (InterruptedException e) {
		}
		//Display info about the main thread
		System.out.println(Thread.currentThread());
	}
}
</pre>
<p class="text5"><strong>Output</strong></p>
<p>thread3<br />
Thread[thread1,5,main]
Thread[thread2,5,main]
Thread[thread3,5,main]
Thread[main,5,main]private</p>

<p>This approach of creating a thread by implementing the Runnable Interface must be used whenever the class being used to instantiate the thread object is required to extend some other class.</p>
<h2>Extending Thread Class</h2>
<p><a href="http://americanportal.com/" target="_top"><br />
</a></p>
<p class="style11">The procedure for creating threads based on extending the Thread is as follows:</p>
<p>1. A class extending the Thread class overrides the run() method from the Thread class to define the code executed by the thread.</p>
<p>2. This subclass may call a Thread constructor explicitly in its constructors to initialize the thread, using the super() call.</p>
<p>3. The start() method inherited from the Thread class is invoked on the object of the class to make the thread eligible for running.</p>
<p><span class="style11">Below is a program that illustrates instantiation and running of threads by extending the Thread class instead of implementing the Runnable interface. To start the thread you need to invoke the <strong>start()</strong> method on your object.</span></p>
<pre class="wp-code-highlight prettyprint">
class XThread extends Thread {

	XThread() {
	}
	XThread(String threadName) {
		super(threadName); // Initialize thread.
		System.out.println(this);
		start();
	}
	public void run() {
		//Display info about this particular thread
		System.out.println(Thread.currentThread().getName());
	}
}

public class ThreadExample {

	public static void main(String[] args) {
		Thread thread1 = new Thread(new XThread(), &quot;thread1&quot;);
		Thread thread2 = new Thread(new XThread(), &quot;thread2&quot;);
		//	    The below 2 threads are assigned default names
		Thread thread3 = new XThread();
		Thread thread4 = new XThread();
		Thread thread5 = new XThread(&quot;thread5&quot;);
		//Start the threads
		thread1.start();
		thread2.start();
		thread3.start();
		thread4.start();
		try {
	//The sleep() method is invoked on the main thread to cause a one second delay.
			Thread.currentThread().sleep(1000);
		} catch (InterruptedException e) {
		}
		//Display info about the main thread
		System.out.println(Thread.currentThread());
	}
}
</pre>
<p class="style11">Output</p>
<p>Thread[thread5,5,main]
thread1<br />
thread5<br />
thread2<br />
Thread-3<br />
Thread-2<br />
Thread[main,5,main]
<p><span class="style5"><a onclick="javascript:pageTracker._trackPageview('/downloads/java-thread-example.zip');"  href="http://www.javabeginner.com/java-thread-example.zip">Download</a></span> <strong> Java Thread Program Example</strong></p>
<p><a href="http://americanportal.com/jobs.php" target="_top"><br />

<p>When creating threads, there are two reasons why implementing the Runnable interface may be preferable to extending the Thread class:</p>
<ul>
<li>Extending the Thread class means that the subclass cannot extend any other class, whereas a class implementing the Runnable interface<br />
has this option.</li>
<li>A class might only be interested in being runnable, and therefore, inheriting the full overhead of the Thread class would be excessive.</li>
</ul>

<p>for(;;) System.out.println(&#8220;Stop the world!&#8221;);</p>
<p>}</p>
<p>}</p>
