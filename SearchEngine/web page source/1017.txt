<div style="color:#000080;font-family:Arial;font-size:10pt">
<p>The processing of text often consists of parsing a formatted
input string. <i>Parsing
</i>is the division of text into a set of discrete parts, or <i>tokens,
</i>which in a certain
sequence can convey a semantic meaning. The <b>StringTokenizer
</b>class provides the first
step in this parsing process, often called the <i>lexer
</i>(lexical analyzer) or <i>scanner</i>.
<b>StringTokenizer
</b>
implements the <b>Enumeration
</b>interface. Therefore,
given an input string, you can enumerate the individual tokens contained in it using <b>StringTokenizer</b>.
<p>To use <b>StringTokenizer</b>,
you specify an input string and a string that contains delimiters.
<i>
Delimiters </i>are
characters that separate tokens. Each character in the delimiters string is considered a valid delimiter—for example, <b>&quot;,;:&quot;
</b>sets the delimiters to a
comma, semicolon, and colon. The default set of delimiters consists of
the whitespace characters: space, tab, newline, and carriage return.</p>
<p>The <b>StringTokenizer
</b>constructors are shown
here:</p>
<code>StringTokenizer(String <i>str</i>)
<br>StringTokenizer(String <i>str</i>,
String <i>delimiters</i>)
<br>StringTokenizer(String <i>str</i>,
String <i>delimiters</i>,
boolean <i>delimAsToken</i>)</code>
<p>In all versions, <i>str
</i>is the string that will
be tokenized. In the first version, the default delimiters are used. In the second and third versions, <i>delimiters
</i>is a string that
specifies the delimiters. In the third version, if <i>delimAsToken
</i>is <b>true</b>,
then the delimiters are also returned as tokens when the string is parsed. Otherwise, the
delimiters are not returned.&nbsp;</p>
<p>Delimiters are not returned as tokens by the first two forms. Once you have created a <b>StringTokenizer
</b>object, the <b>nextToken(
) </b>method is used to extract consecutive tokens. The <b>hasMoreTokens(
) </b>method returns <b>true
</b>while there are more tokens to be extracted. Since <b>StringTokenizer
</b>implements <b>Enumeration</b>,
the
<b>
hasMoreElements( ) </b>and
<b>nextElement( ) </b>methods
are also implemented, and they act the same as <b>hasMoreTokens(
) </b>and <b>nextToken(
)</b>, respectively.</p>
<p>Here is an example that creates a <b>StringTokenizer
</b>to parse
&quot;key=value&quot; pairs. Consecutive sets of &quot;key=value&quot; pairs are separated by
a semicolon.</p>
  <code>// Demonstrate StringTokenizer.
<br>import java.util.StringTokenizer;
<br>class STDemo {
<br>static String in = &quot;title=Java-Samples;&quot; +
<br>&quot;author=Emiley J;&quot; +
<br>&quot;publisher=java-samples.com;&quot; +
<br>&quot;copyright=2007;&quot;;
<br>public static void main(String args[]) {
<br>StringTokenizer st = new StringTokenizer(in, &quot;=;&quot;);
<br>while(st.hasMoreTokens()) {
<br>String key = st.nextToken();
<br>String val = st.nextToken();
<br>System.out.println(key + &quot;\t&quot; + val);
<br>}
<br>}
<br>}</code>
<p>The output from this program is shown here:</p>
<code>title Java-samples
<br>author Emiley J
<br>publisher java-samples.com
<br>copyright 2007</code>
