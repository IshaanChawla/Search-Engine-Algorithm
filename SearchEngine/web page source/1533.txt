		</div>		<div class="post-byline">By Alex, on June 5th, 2007</div>		<div class="post-bodycopy clearfix"><p><strong>Keywords</strong></p>

<p>You have already run across some of these keywords, including <em>int</em>, <em>void</em>, <em>return</em>, <em>using</em>, and <em>namespace</em>.  Along with a set of operators, these keywords define the entire language of C++ (preprocessor commands excluded).  Because these keywords have special meaning, your IDEs will change the text color of these words (usually to blue) to make them more visible.</p>
<p>By the time you are done with this tutorial, you will understand what almost all of these words do!</p>
<p><strong>Identifiers, and naming them</strong></p>
<p>The name of a variable, function, class, or other object in C++ is called an <strong>identifier</strong>.  C++ gives you a lot of flexibility to name identifiers as you wish.  However, there are a few rules that must be followed when naming identifiers:</p>
<ul>
<li>The identifier can not be a keyword.  Keywords are reserved.
<li>The identifier can only be composed of letters (lower or upper case), numbers, and the underscore character.  That means the name can not contain symbols (except the underscore) nor whitespace.
<li>The identifier must begin with a letter or an underscore.  It can not start with a number.
<li>C++ distinguishes between lower and upper case letters.  <code>nvalue</code> is different than <code>nValue</code> is different than <code>NVALUE</code>.
</ul>
<p>Now that you know how you <em>can</em> name a variable, let&#8217;s talk about how you <em>should</em> name a variable or function.</p>
<p>First, it is a convention in C++ that variable names begin with a lowercase letter.  If the variable name is one word, the whole thing should be written in lowercase letters.</p><!-- Crayon Syntax Highlighter v2.7.1 -->

<p></p>
<p>Generally, functions are also started with a lowercase letter (though there&#8217;s some disagreement on this point).  We&#8217;ll start ours with lower case letters because main() (which all programs must have) starts with a lower case letter, as do all of the functions in the standard template library.</p>
<p>Identifier names that start with a capital letter are typically used for structs, classes, and enumerations (all of which we will cover later).</p>
<p>If the variable or function name is multi-word, there are two common conventions: separated by underscores, or intercapped (sometimes called CamelCase).</p><!-- Crayon Syntax Highlighter v2.7.1 -->

<p></p>
<p>In this tutorial, we will typically use the intercapped approach because it&#8217;s easier to read (it&#8217;s easy to mistake an underscore for a space in dense blocks of code).  But it&#8217;s common to see either -- the C++ standard library uses the underscore method for both variables and functions.  Sometimes you&#8217;ll see a mix of the two: underscores used for variables and intercaps used for functions.</p>
<p>Second, you should avoid naming your identifiers starting with an underscore, as these names are typically reserved for OS, library, and/or compiler use.</p>
<p>Third, and this is perhaps the most important rule of all, give your identifiers names that actually describe what they are.  It is typical for inexperienced programmers to make variable names as short as possible, either to save on typing or because they figure the meaning is obvious.  This is almost always a mistake.  Ideally, variables should be named in a way that would help someone who has no idea what your code does be able to figure it out as quickly as possible.  In 3 months, when you look at your program again, you&#8217;ll have forgotten how it works, and you&#8217;ll thank yourself for picking variable names that make sense.  The more complex the code the variable is being used in, the better name it should have.</p>
