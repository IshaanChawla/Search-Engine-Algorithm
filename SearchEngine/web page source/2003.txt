    Lists
&#x2022; Problem
&#x2013; You want to make an ordered list of objects. But, even
after you get the first few elements, you don&#x2019;t know how
many more you will have.
&#x2022; Thus, you can&#x2019;t use an array, since the size of arrays must
be known at the time that you allocate it. (Although Java
arrays are better than C++ arrays since the size does not
need to be a compile-time constant)
&#x2022; Solution
&#x2013; Use ArrayList or LinkedList: they stretch as you add
elements to them
&#x2022; Notes
&#x2013; The two options give the same results for the same
operations, but differ in performance7

&#x2013; When you modify code, you want to verify that it still gives right answer.
So, make test cases and re-run them every time you change the code.
&#x2022; Super-simple usage
&#x2013; Put @Test above any zero-arg method
&#x2022; Eclipse will prompt you to include the JUnit library
&#x2013; Use import static org.junit.Assert.*;
&#x2022; Static imports let you use static methods without using class name. E.g., if
you import static java.lang.Math.*, you can use &#x201C;cos&#x201D; instead of &#x201C;Math.cos&#x201D;.
&#x2013; Use assertTrue, assertFalse, assertEquals for testing
&#x2013; R-click in code, Run As &#xF0E0; JUnit Test
&#x2022; Check results printed by Eclipse
&#x2022; Zillions more options,
&#x2013; But above is enough for most cases
&#x2022; For more options and features, see http://junit.org/
