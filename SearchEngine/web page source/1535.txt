<p><strong>Basic formatting</strong></p>
<p>Unlike some other languages, C++ does not enforce any kind of formatting restrictions on the programmer (remember, trust the programmer!).  Many different methods of formatting C++ programs have been developed throughout the years, and you will find disagreement on which ones are best.  Our basic rule of thumb is that the best styles are the ones that produce the most readable code, and provide the most consistency.</p>
<p>Here are our recommendations for basic formatting:</p>
<p>1) Your tabs should be set to 4 spaces (most IDEs have a setting where you can configure this).  Some IDEs default to 3 spaces, which is fine too.</p>
<p>The reason to use spaces instead of tab characters is so that if you open your code in another viewer, it&#8217;ll retain the correct indentation.</p>
<p>2) The braces that tell where a function begins and ends should be aligned with the function name, and be on their own lines:</p><!-- Crayon Syntax Highlighter v2.7.1 -->

<p>Keeping the opening braces on the same line is the recommended formatting in java&#8217;s case, for example, where formatting is just as unrestricted as in c++.</p>
<p>I also prefer 2 spaces wide tabs, but it&#8217;s personal preference and such things as the font or screen resolution may matter. For this reason I use tabs only for indentation, but for alignment on lines strictly spaces. That way a different tab size produces consistent layout.</p>

		<p><code>std::endl</code> does more than just a carriage return to the beginning of the following line. It also forces an immediate flush of the underlying <code>iostream</code> object&#8217;s stream buffer. This defeats the purpose of the buffer in the first place. To me it seemed better (if you just <b>have</b> to type endl) to simply:</p>
<p><code>#define endl &#039;\n&#039;</code></p>
<p>However this creates a problem if you are using the qualified <code>std::endl</code>, the preprocessor changes it to <code>std::&#039;\n&#039;</code> before compiling and that breaks the compilation.</p>
<p>Do it this way:</p>
<p><code>std::string endl(&quot;\n&quot;);</code></p>
<p>This declares a std::string object <code>endl</code> and initializes it to &#8220;\n&#8221;</p>
<p>Now in code following it whenever you do <code>cout &lt;&lt; endl</code> it inserts the <code>\n</code> into the stream instead of <code>std::endl</code> (even if you have used <code>using namespace std;</code> before it).</p>

<p>In your <code>main()</code> function body you have</p>
<p><code>doubleVal(x);</code></p>
<p>and the <code>int</code> value returned by that function is not being assigned to any variable that is local to <code>main()</code>. So essentially you have done nothing constructive by simply calling the function. Comment that line out or remove it and your code will still work.</p>
<p>The reason <code>cout &lt;&lt; doubleVal(x) &lt;&lt; endl;</code> works is because <code>doubleVal(x)</code> is evaluated to the int value returned by the function, before it is inserted into the output stream by the <code>operator&lt;&lt;()</code> member of class <code>ostream</code>.</p>
<p>This mechanics of this will all be revealed later, and since it has been four years since you posted this you probably already know all about it. <img src="http://www.learncpp.com/blog/wp-includes/images/smilies/simple-smile.png" alt=":)" class="wp-smiley" style="height: 1em; max-height: 1em;" /></p>
