						<div class="post-headline">		<h1>0.5 &#8212; Installing an Integrated Development Environment (IDE)</h1>
		</div>		<div class="post-byline">By Alex, on May 28th, 2007</div>		<div class="post-bodycopy clearfix"><p>As mentioned in the previous section, an <strong>Integrated Development Environment (IDE)</strong> contains all of the things you need to develop, compile, link, and debug your programs.  So let&#8217;s install one.</p>
<p>The obvious question is, &#8220;which one?&#8221;.  Keep in mind that you can install multiple IDEs, so there is no &#8220;wrong decision&#8221; here.  During the course of these tutorials, we will be showing you some of the nice features of your IDE, such as how to do integrated debugging.  All of our examples will be done using both <strong>Microsoft&#8217;s Visual C++</strong> (for Windows), and <strong>Code::Blocks</strong> (for Linux or Windows).  Thus we highly recommend you pick one of these.</p>
<p>However, if you would like to try a different IDE, you are free to do so.  The concepts we show you will work for any IDE -- however, different IDE&#8217;s use different keymappings and different setups, and you may have to do a bit of searching to find the equivalent of what we show you.</p>
<p><strong>Windows</strong></p>
<p>If you are developing on a Windows machine (as most of you are), then we highly recommend Microsoft&#8217;s free <a href="https://www.visualstudio.com/en-us/products/visual-studio-express-vs.aspx" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://www.visualstudio.com/en-us/products/visual-studio-express-vs.aspx', 'Visual Studio Express 2013 for Windows Desktop']);" >Visual Studio Express 2013 for Windows Desktop</a>.  Make sure you get the version for <strong>Windows Desktop</strong>, not the similarly named Express 2013 for Windows.  If your operating system is too old to run the 2013 version, you can find the Visual C++ 2010 Express edition toward the bottom of the page.  The installer that you download off of Microsoft&#8217;s web page is actually a downloader.  When you run it, it will download the actual IDE from Microsoft.</p>
<p><i>Note: This tutorial was originally written when Microsoft was distributing the 2005 version of Visual C++.  Consequently, all references and screenshots are targeted to that version.  Running any later versions (such as 2008, 2010, 2013, etc&#8230;) are fine, however, your screens may look slightly different.</i></p>
<p><img src="http://learncpp.com/images/CppTutorial/Section0/VC2005-MSDN.png" alt="Selecting MSDN 2005 Express" /></p>
<p>MSDN express will give you documentation about C/C++ functions that you will refer to often.  If you do not install MSDN Express on your hard drive, then Visual C++ will use the online version at <a href="http://msdn.microsoft.com/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://msdn.microsoft.com/', 'msdn.microsoft.com/']);" >msdn.microsoft.com/</a>.</p>
<p>The installer will likely ask you to reboot your computer after it has finished downloading the components.  If it does, your computer may appear to &#8220;hang&#8221; temporarily.  Don&#8217;t panic, it&#8217;s just installing stuff.  Once you are back into Windows, you may need to run the installer again.  Choose the same options, and it will seem like it&#8217;s going to download everything again, but it won&#8217;t.  It will install the program.</p>
<p><strong>Linux or Windows</strong></p>
<p>If you are developing on Linux (or you are developing on Windows but want to write programs that you can easily port to Linux), we recommend <a href="http://www.codeblocks.org/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://www.codeblocks.org/', 'Code::Blocks']);" >Code::Blocks</a>.  Code::Blocks is a free, open source, cross-platform IDE that will run on both Linux and Windows.</p>
<p>Windows users, make sure to get the version with MinGW bundled.</p>
<p>Alternately, some people prefer to use <a href="http://www.bloodshed.net/devcpp.html" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://www.bloodshed.net/devcpp.html', 'Bloodshed&#8217;s Dev-C++']);" >Bloodshed&#8217;s Dev-C++</a>, which also runs on both Windows and Linux.</p>
<p><strong>Mac OSX</strong></p>
<p>Mac users can use <a href="http://en.wikipedia.org/wiki/Xcode" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://en.wikipedia.org/wiki/Xcode', 'Xcode']);" >Xcode</a> if it is available to you, or <a href="http://www.eclipse.org/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://www.eclipse.org/', 'Eclipse']);" >Eclipse</a>.  Eclipse is not set up to use C++ by default, and you will need to install the optional C++ components.</p>
<p><strong>Moving on</strong></p>
<p>Once your IDE is installed (which is one of the hardest things this tutorial will ask you to do), you are ready to write your first program!</p>
