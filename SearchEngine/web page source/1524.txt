						<div class="post-headline">		<h1>0.6a &#8212; Build configurations</h1>
		</div>		<div class="post-byline">By Alex, on February 17th, 2015</div>		<div class="post-bodycopy clearfix"><p>A <strong>build configuration</strong> (also called a <strong>build target</strong>) is a collection of project settings that determines how your IDE will build your project.  The build configuration typically includes things like what the executable will be named, what directory the executable will be output in, what directories the IDE will look in for other code and header files, whether to keep or exclude debugging information, and how highly to have the compiler optimize your program.  Generally, you will want to leave the settings at their default values unless you have a specific reason to change something.</p>
<p>When you create a new project in your IDE, the IDE will generally set up two different build configurations for you: a release configuration, and a debug configuration.</p>
<p>The <strong>debug configuration</strong> is designed to help you debug your program, and is generally the one you will use when developing your programs.  This configuration turns off all optimizations, and includes debugging information, which makes your programs larger and slower, but much easier to debug.  The debug configuration is usually selected as the active configuration by default.</p>
<p>The <strong>release configuration</strong> is designed to be used when releasing your program to the public.  This version is typically optimized for size and performance, and doesn&#8217;t contain the extra debugging information.</p>
<p>For example, when the &#8220;Hello World&#8221; program from the previous lesson was built using Visual Studio 2013, the executable produced in the debug configuration was 65kb, whereas the executable built in the release version was 12kb.</p>
<p><strong>Switching between debug and release in Visual Studio</strong></p>
<p>There are multiple ways to switch between debug and release in Visual Studio.  The easiest way is to find the &#8220;Solution Configurations&#8221; dropdown in the Standard Toolbar Options menu:</p>
<p><img src="http://learncpp.com/images/CppTutorial/Section0/VS-BuildTarget.jpg" alt="VS Solution Configurations Dropdown" /></p>
<p>It should be set to Debug or Release.  You can change which one you&#8217;re using by selecting it in the dropdown.</p>
<p>You can also access the configuration manager dialog, by selecting Build->Configuration Manager, and change the &#8220;Active solution configuration&#8221;.</p>
<p><strong>Switching between debug and release in Code::Blocks</strong></p>
<p>In Code::Blocks, you should see a toolbar item called &#8220;Build Target&#8221;:</p>
<p><img src="http://learncpp.com/images/CppTutorial/Section0/CB-BuildTarget.jpg" alt="VS Solution Configurations Dropdown" /></p>
<p>It can be set to debug or release in the dropdown.</p>
<p><strong>Summary</strong></p>
<p>The build configuration holds the settings for different versions of your project.  You generally won&#8217;t need to touch these settings, but you may want to switch between debug and release.</p>
<p><em>Rule: Use the debug configuration when developing your programs.  When you&#8217;re ready to release your executable to others, build it using the release configuration.</em></p>
