						<div class="post-headline">		<h1>3.x &#8212; Chapter 3 comprehensive quiz</h1>
		</div>		<div class="post-byline">By Alex, on September 11th, 2011</div>		<div class="post-bodycopy clearfix"><p><strong>Quick review</strong></p>
<p>Always use parentheses to disambiguate the precedence of operators if there is any question or opportunity for confusion.</p>
<p>The arithmetic operators all work like they do in normal mathematics.  The Modulus (%) operator returns the remainder from an integer division.  Beware about rounding or sign errors when the operands of integer division and modulus are negative.</p>
<p>The increment and decrement operators can be used to easily increment or decrement numbers.  Beware of side effects, particular when it comes to the order that function parameters are evaluated.  Do not use a variable that has a side effect applied more than once in a given statement.</p>
<p>Relational operators can be used to compare floating point numbers.  Beware using equality and inequality on floating point numbers.</p>
<p>Logical operators allow us to form compound conditional statements.  Bitwise operators allow us to modify or query individual bits.</p>
<p><strong>Comprehensive quiz</strong></p>
<p>1) Evaluate the following:</p>
<p>a) (5 > 3 &#038;&#038; 4 < 8)<br />
b) (4 > 6 &#038;&#038; true)<br />
c) (3 >= 3 || false)<br />
d) (true || false) ? 4 : 5</p>
<p>2) Answer the following:</p>
<p>a) 7 / 4<br />
b) 14 % 5</p>
<p>3) Convert the following from binary to decimal:<br />
a) 1101<br />
b) 101110</p>
<p>4) Convert the following from decimal to binary:<br />
a) 15<br />
b) 53</p>
<p>5) Why should you never do the following:<br />
a) int y = foo(++x, x);<br />
b) int x = 7 / -2;<br />
c) int x = -5 % 2;<br />
d) float x = 0.1 + 0.1; if (x == 0.2) return true; else return false;<br />
e) int x = 3 / 0;</p>
<p><strong>Solutions</strong></p>
<p>1) <a class="solution_link_show" href="javascript:void(0)" onclick="wpSolutionToggle(document.getElementById('id1796327768'), this, 'Show Solution', 'Hide Solution')">Show Solution</a></p>
<div class="solution_div" id="id1796327768" style="display:none">
<p>a) (5 > 3 &#038;&#038; 4 < 8) becomes (true &#038;&#038; true), which is true.<br />
b) (4 > 6 &#038;&#038; true) becomes (false &#038;&#038; true), which is false.<br />
c) (3 >= 3 || false) becomes (true || false), which is true.<br />
d) (true || false) ? 4 | 5 becomes (true ? 4 : 5), which is 4.
</div>
<p>2) <a class="solution_link_show" href="javascript:void(0)" onclick="wpSolutionToggle(document.getElementById('id483257222'), this, 'Show Solution', 'Hide Solution')">Show Solution</a></p>
<div class="solution_div" id="id483257222" style="display:none">
a) 7 / 4 = 1 remainder 3, so this equals 1.<br />
b) 14 % 5 = 2 remainder 4, so this equals 4.
</div>
<p>3) <a class="solution_link_show" href="javascript:void(0)" onclick="wpSolutionToggle(document.getElementById('id1808381718'), this, 'Show Solution', 'Hide Solution')">Show Solution</a></p>
<div class="solution_div" id="id1808381718" style="display:none">
a) 1101 is ((1 * 8) + (1 * 4) + (0 * 2) + (1 * 1)) = 8 + 4 + 1 = 13<br />
b) 101110 is ((1 * 32) + (0 * 16) + (1 * 8) + (1 * 4) + (1 * 2) + (0 * 1)) = 32 + 8 + 4 + 2 = 46
</div>
<p>4) <a class="solution_link_show" href="javascript:void(0)" onclick="wpSolutionToggle(document.getElementById('id186705995'), this, 'Show Solution', 'Hide Solution')">Show Solution</a></p>
<div class="solution_div" id="id186705995" style="display:none">
a) Using method 1:<br />
15 / 2 = 7 r1<br />
7 / 2 = 3 r1<br />
3 / 2 = 1 r1<br />
1 / 2 = 0 r1</p>
<p>Reading the remainders from bottom to top: 1111</p>
<p>Using method 2:</p>
<p>Is 15 >= 8?  Yes, with 7 left over.<br />
Is 7 >= 4?  Yes, with 3 left over.<br />
Is 3 >= 2?  Yes, with 1 left over.<br />
Is 1 >= 1?  Yes.<br />
So this number is 1111 in binary.</p>
<p>b) Using method 1:<br />
53 / 2 = 26 r1<br />
26 / 2 = 13 r0<br />
13 / 2 = 6 r1<br />
6 / 2 = 3 r0<br />
3 / 2 = 1 r1<br />
1 / 2 = 0 r1</p>
<p>Reading the remainders from bottom to top: 110101</p>
<p>Using method 2:</p>
<p>Is 53 >= 32?  Yes, with 21 left over.<br />
Is 21 >= 16?  Yes, with 5 left over.<br />
Is 5 >= 8?  No.<br />
Is 5 >= 4?  Yes, with 1 left over.<br />
Is 1 > 2?  No.<br />
Is 1 >=1?  Yes.</p>
<p>So this number is 110101 in binary.
</p></div>
<p>5) <a class="solution_link_show" href="javascript:void(0)" onclick="wpSolutionToggle(document.getElementById('id116201815'), this, 'Show Solution', 'Hide Solution')">Show Solution</a></p>
<div class="solution_div" id="id116201815" style="display:none">
a) Because operator++ applies a side effect to x, we should not use x again in the same expression.  In this case, the parameters to function foo() can be evaluated in any order, so it&#8217;s indeterminate whether x or ++x gets evaluated first.  Because ++x changes the value of x, it&#8217;s unclear what values will be passed into the function.<br />
b) It&#8217;s unclear whether the compiler will round this up to -3 or down to -4.<br />
c) It&#8217;s unclear whether this will result in 1 or -1.<br />
d) Floating point errors will cause this to evaluate as false even though it looks like it should be true.<br />
e) Divide by 0 will crash the program.
</div>
