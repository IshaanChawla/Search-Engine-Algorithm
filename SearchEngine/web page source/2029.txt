&#x2022; Naming convention
&#x2013; All Swing component names begin with a capital J and
follow the format JXxx. E.g., JFrame, JPanel, JApplet,
JDialog, JButton. Many are just AWT names with a J.
&#x2022; Lightweight components
&#x2013; Most Swing components are lightweight: formed by
drawing in the underlying window.
&#x2022; Use of paintComponent for drawing

    JLabel
&#x2022; Main new feature: HTML content
&#x2013; If text is "&lt;html&gt;...&lt;/html&gt;", it gets rendered as HTML
&#x2013; HTML labels only work in JDK 1.2.2 or later, or in
Swing 1.1.1 or later.
&#x2013; In JDK 1.2 the label string must begin with &lt;html&gt;, not
&lt;HTML&gt;. It is case-insensitive in JDK 1.3 and 1.4.
&#x2013; JLabel fonts are ignored if HTML is used. If you use
HTML, all font control must be performed by HTML.
&#x2013; You must use &lt;P&gt;, not &lt;BR&gt;, to force a line break.
&#x2013; Other HTML support is spotty.
&#x2022; Be sure to test each HTML construct you use. Permitting
the user to enter HTML text at runtime is asking for
trouble.
