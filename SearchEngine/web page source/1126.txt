target="_blank">JavaServer Faces (JSF) 2.0</a>, is an MVC web framework which focus on simplifies building user interfaces (comes with 100+ ready UI tags) for Java web application and make reusable UI component easy to implement. Unlike JSF 1.x, almost everything is declared in <code>faces-config.xml</code>, with JSF 2.0, you are allowed to use annotation to declare navigating, managed bean or CDI bean, which make your development easier and faster.</p><p>In this tutorial, it provides many step by step examples and explanations on using JavaServer Faces (JSF) 2.0 framework.</p><p>Happy learning JSF 2.0 :)</p><h2 class="tut-title">Quick Start</h2><p>Some quick start examples for JSF 2.0</p><ul><li><a href="http://www.mkyong.com/jsf2/jsf-2-0-hello-world-example/" target="_blank">JSF 2.0 hello world example</a><br /> A JavaServer Faces (JSF) 2.0 hello world example, shows the JSF 2.0 dependencies, basic annotations and configurations. Let you have a quick idea about how JSF 2.0 look like, and also how it differs from JSF 1.x.</li><li><a href="http://www.mkyong.com/jsf2/jsf-2-0-ajax-hello-world-example/" target="_blank">JSF 2.0 + Ajax hello world example</a><br /> In JSF 2.0, coding Ajax has been just like coding a normal HTML tag, it’s extremely easy. In this tutorial, you will restructure the last JSF 2.0 hello world example, so that, when the button is clicked, it will make an Ajax request instead of submitting the whole form.</li><li><a href="http://www.mkyong.com/jsf2/how-to-make-eclipse-ide-supports-jsf-2-0/" target="_blank">How to make Eclipse IDE supports JSF 2.0</a><br /> Here&#8217;s a quick guide to show how to enable the JSF 2.0 features in Eclipse projects.</li><li><a href="http://www.mkyong.com/jsf2/resources-library-in-jsf-2-0/" target="_blank">Resources (library) in JSF 2.0</a><br /> In JSF 2.0, all your resources files like css, images or JavaScript, should put into a “resources” folder in the root of your web application. In JSF 2.0 terminology, all the sub-folder name of the “resources” folder is considered as a “library” in JSF 2.0 web application. Later, you can reference this “library” with JSF tag’s library attribute.</li></ul><div><div style="margin:20px 0;" class="hide_if_width_less_800"><script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script> <ins class="adsbygoogle hide_if_width_less_800" style="display:inline-block;width:728px;height:90px" data-ad-client="ca-pub-2836379775501347" data-ad-slot="7391621200" data-ad-region="mkyongregion"></ins><script>(adsbygoogle=window.adsbygoogle||[]).push({});</script></div></div><h2 class="tut-title">Managed Bean</h2><p>About managed bean configuration and injection in JSF 2.0</p><ul><li><a href="http://www.mkyong.com/jsf2/configure-managed-beans-in-jsf-2-0/" target="_blank">Configure Managed Beans in JSF 2.0</a><br /> In JSF 2.0, Java bean that can be accessed from JSF page is called Managed Bean.
