		<h1 class="firstheading"><a href="http://www.codingunit.com/c-tutorial-how-to-use-time-and-date-in-c" rel="bookmark" title="How to use Time and Date in C">
		How to use Time and Date in C</a></h1>

			<div class="entry">
				<div id="watermark">
					<p>In today&#8217;s C programming language tutorial we take a look at how to use time and date from C programs.<br />
To make use of time and date function you need to include the time.h header file from the standard C library.</p>
<p>This header file contains functions and macros that provide standardized access to time and date. It also contains functions to manipulate and format the time and date output. The <a title="time.h reference page contains" href="http://www.codingunit.com/c-reference-time-h-function-time"><strong>time.h reference page contains</strong></a> more information about the different time and date functions.</p>
<p><span id="more-228"></span></p>
<h2>UNIX Time or POSIX Time</h2>
<p>The function time() returns the type time_t. The time that is returned represents the number of seconds elapsed since 00:00 hours, Jan 1, 1970 UTC. It&#8217;s also called UNIX EPOCH time.</p>
<p>It is widely used not only on Unix-like operating systems but also in many other computing systems.</p>
<p><strong>Fun note:</strong> On February 13, 2009 at exactly 23:31:30 (UTC), UNIX time was equal to &#8216;1234567890&#8217;.</p>
<p>Let&#8217;s take a look at a source code example:</p>
<div id="cl">
<pre><code>
#include &lt;stdio.h&gt;
#include &lt;time.h&gt;
int main ()
{
    time_t sec;
    sec = time (NULL);

    printf ("Number of hours since January 1, 1970 is %ld \n", sec/3600);
    return 0;
}</code></pre>
</div>
<p></p>
<div id="cl">
<pre><code>
Output example:
     Number of hours since January 1, 1970 is 343330</code></pre>
</div>
<h2>MS Windows and Time</h2>
<p>Microsoft Windows is also different time for different things. Normally we only place programs that will compile on UNIX, Linux and Windows, but this time we make an exception and show you also a Windows example.</p>
<p>(<strong>Note:</strong> that the other programs in this tutorial should also work and compile on Windows.)</p>
<p>Windows has it&#8217;s own SYSTEMTIME structure which stores information on the date and time. For instance: Year, Month, Day, Hour, Minutes, etc. The format of this structure looks as follows:</p>
<div id="cl">
<pre><code>
	typedef struct _SYSTEMTIME {
		WORD wYear;
		WORD wMonth;
		WORD wDayOfWeek;
		WORD wDay;
		WORD wHour;
		WORD wMinute;
		WORD wSecond;
		WORD wMilliseconds;
	} SYSTEMTIME;</code></pre>
</div>
<p>Take a look at the following example that uses GetSystemTime function to get the UTC time:</p>
<div id="cl">
<pre><code>
#include &lt;Windows.h&gt;
#include &lt;stdio.h&gt;</code>

void main()
{
	SYSTEMTIME str_t;
	GetSystemTime(&amp;str_t);

	printf("Year:%d\nMonth:%d\nDate:%d\nHour:%d\nMin:%d\nSecond:% d\n"
	,str_t.wYear,str_t.wMonth,str_t.wDay
	,str_t.wHour,str_t.wMinute,str_t.wSecond);

}</pre>
</div>
<p><strong>Note:</strong> the windows.h include, this program will not work on other platforms. Another thing we need to warn you about is that the program doesn&#8217;t respond to daylight saving time issues.</p>
<div id="cl">
<pre><code>
	Output of the Windows example:
		Year: 2009
		Month: 3
		Day: 3
		Hour: 14
		Minute: 59
		Seconds: 23</code></pre>
</div>
<h2>Readable Current Local Time using ctime</h2>
<p>The Unix EPOCH time is not readable for humans. To get a human-readable version of the current local time you can use the ctime() function. The function returns a C string containing the date and time information.</p>
<p>This string is followed by a new-line character (&#8216;\n&#8217;) and it will convert the time_t object pointed by the timer to a C string containing a human-readable version of the corresponding local time and date.</p>
<p><strong>Please note:</strong> that the functions ctime() and asctime() share the array which holds the time string.<br />
If either one of these functions is called, the content of the array is overwritten.</p>
<p>An example of ctime() use:</p>
<div id="cl">
<pre><code>
	#include &lt;time.h&gt;
	#include &lt;stdio.h&gt;</code>

	int main(void)
	{
		time_t mytime;
		mytime = time(NULL);
		printf(ctime(&amp;mytime));

		return 0;
	}</pre>
</div>
<p>The string that is returned will have the following format:</p>
<div id="cl">
<pre><code>
	Www Mmm dd hh:mm:ss yyyy</code>

	Www = which day of the week.
	Mmm = month in letters.
	dd = the day of the month.
	hh:mm:ss = the time in hour, minutes, seconds.
	yyyy = the year.

	Output example:
		Tue Feb 26 09:01:47 2009</pre>
</div>
<h2>Manipulating the time structure with mktime()</h2>
<p>It is also possible to manipulate the time structure and to create your own time using mktime().</p>
<p>Let&#8217;s look at an example (note that the example will not change your system, it only prints a new time.):</p>
<div id="cl">
<pre><code>
	#include &lt;time.h&gt;
	#include &lt;stdio.h&gt;</code>

	int main(void)
	{
		struct tm str_time;
		time_t time_of_day;

		str_time.tm_year = 2012-1900;
		str_time.tm_mon = 6;
		str_time.tm_mday = 5;
		str_time.tm_hour = 10;
		str_time.tm_min = 3;
		str_time.tm_sec = 5;
		str_time.tm_isdst = 0;

		time_of_day = mktime(&amp;str_time);
		printf(ctime(&amp;time_of_day));

		return 0;
	}</pre>
</div>
<p><strong>Note:</strong> that you can set anything you want for the hour, min and sec as long as they don&#8217;t cause a new day to occur.</p>
<div id="cl">
<pre><code>
	Output of the program:
		Thu Jul 05 11:03:05 2012</code></pre>
</div>
<p>As you can see we can also manipulate the time into the future. If you look at str_time.tm_hour and the output you will see that they don&#8217;t correspond (10 and 11). This is because of the daylight saving time set on our computer. If you also have this then you should try the following: set the str_time.tm_mon to 0 to see the effect of daylight saving time. The result will be that the output now will be 10.</p>
<h2>Using the function difftime</h2>
<p>The function difftime() is a very useful function, because it can be used to measure the performance time of a certain part of code. For instance in our example we measure the time of a loop (that is doing nothing at all.)<br />
Take a look at the example:</p>
<div id="cl">
<pre><code>
	#include &lt;stdio.h&gt;
	#include &lt;time.h&gt;</code>

	int main(void)
	{
		time_t start,end;
		volatile long unsigned counter;

		start = time(NULL);

		for(counter = 0; counter &lt; 500000000; counter++)
			; /* Do nothing, just loop */

		end = time(NULL);
		printf("The loop used %f seconds.\n", difftime(end, start));
		return 0;
	}</pre>
</div>
<p></p>
<div id="cl">
<pre><code>
	Output of the example:
		The loop used 2.000000 seconds.</code></pre>
</div>
<h2>Timezones</h2>
<p>It is also possible to work with different timeszones by using gmtime() to convert calendar time to UTC.<br />
If you know the UTC time you can add CET, for example Amsterdam +1 hour. Take a look at the following example:</p>
<div id="cl">
<pre><code>
	#include &lt;stdio.h&gt;
	#include &lt;time.h&gt;</code>

        #define PST (-8)
        #define CET (1)

	int main ()
       {
                time_t raw_time;
                struct tm *ptr_ts;

                time ( &amp;raw_time );
                ptr_ts = gmtime ( &amp;raw_time );

                printf ("Time Los Angeles: %2d:%02d\n",
                     ptr_ts-&gt;tm_hour+PST, ptr_ts-&gt;tm_min);
                printf ("Time Amsterdam: %2d:%02d\n",
                     ptr_ts-&gt;tm_hour+CET, ptr_ts-&gt;tm_min);
		return 0;
         }</pre>
</div>
<p></p>
<div id="cl">
<pre><code>
	Output of the example:
		Time Los Angeles: 06:01
		Time Amsterdam: 15:01</code></pre>
</div>
<h2>Number of clock ticks</h2>
<p>It is also possible to use clock ticks elapsed since the start of the program in your own programs by using the function clock(). For instance you can build a wait function or use it in your frame per second (FPS) function.</p>
<p>Take a look at the following wait example:</p>
<div id="cl">
<pre><code>
	#include &lt;stdio.h&gt;
	#include &lt;time.h&gt;</code>

	void wait ( int sec )
	{
		clock_t end_wait;
		end_wait = clock () + sec * CLK_TCK ;

		while (clock() &lt; end_wait) {}
	}

	int main ()
	{
		printf ("Start the Wait!\n");

		wait (5);	/* Wait for 5 seconds */

		printf ("End Wait!\n");
		return 0;
	}</pre>
</div>
<h2>In Conclusion</h2>
<p>As you can see there are many ways to use time and dates in your programs. You never know when it time to use time functions in your programs, so learn them or at least play with them by making some example programs of your own. Also take a look at our <strong><a href="http://www.codingunit.com/how-to-make-a-calendar-in-c" title="How to make a calendar in C">C language calendar tutorial</a></strong> for a more advance use of the things explained in this tutorial.</p>

			<p>@Sanket<br />
It all depends what you are trying to do. For example do you need precision or is an approximation alright.<br />
For more precision you can find tools like <a href="http://www.code-wizards.com/projects/libfaketime/index.html" rel="nofollow">libfaketime</a> online. (On the site you also find the source code of the faketime library.)</p>
<p>For an approximation result, you can use something like the example below:<br />
(note: just a quick and dirty example, no error checking, etc)</p>
<p>#include &lt;stdio.h&gt;<br />
#include &lt;time.h&gt;</p>
<p>void wait ( int sec ) {<br />
	int a =0;<br />
	clock_t end_wait;<br />
	end_wait = clock () + sec * CLK_TCK;<br />
	while (clock() &lt; end_wait) {</p>
<p>		// Your useless calculations.<br />
		a++;<br />
	}<br />
	printf (&#8220;%i\n&#8221;, a);<br />
}</p>
<p>void waitspeedup ( int sec, int speedupfactor ) {<br />
	int num,a = 0;<br />
	num = sec / speedupfactor;<br />
	printf (&#8220;The new wait time is: %i\n&#8221;, num);<br />
	clock_t end_wait;<br />
	end_wait = clock () + num * CLK_TCK;<br />
	while (clock() &lt; end_wait) {</p>
<p>		// Your useless calculations<br />
		a++;<br />
	}<br />
	// The fake result (because of speedup factor)<br />
	printf (&#8220;%i\n&#8221;, a);<br />
	// Multiply result with speedupfactor to get an approximation<br />
	// of the actual results.<br />
	printf (&#8220;%i\n&#8221;, a*speedupfactor);<br />
}</p>
<p>int main () {<br />
	printf (&#8220;Waiting.1&#8230;.\n&#8221;);<br />
	// Wait approx. 12 seconds<br />
	wait(12);<br />
	printf (&#8220;Waiting.2&#8230;.\n&#8221;);<br />
	// Wait approx. 12 / 2 = 6 seconds<br />
	waitspeedup(12,2);<br />
	// There are 86400 seconds in a day<br />
	// do something like this 86400/1200=72 sec.<br />
	return 0;<br />
}</p>
<p>Hope that this is useful for you. Good luck!</p>

			<p>Don&#8217;t use the methods used on this page for high resolution timing en never use wait() function (because it blocks), but instead take a look at the functions sleep() and usleep(). For delays of multiple seconds, your best bet is probably to use sleep(). For delays of at least tens of milliseconds (about 10 ms seems to be the minimum delay), usleep() should work. These functions give the CPU to other processes (&#8220;sleep&#8221;), so CPU time isn&#8217;t wasted. See the manual pages sleep(3) and usleep(3) for details. But be careful with such small time delays, because they can&#8217;t behave different on different operating systems.</p>
<p>Take a look at this <a href="http://tldp.org/HOWTO/IO-Port-Programming-4.html" rel="nofollow">page</a> that has more about high resolution timing on Linux.</p>
<p>Hope it helps!</p>
