		<h1 class="firstheading"><a href="http://www.codingunit.com/c-tutorial-binary-file-io" rel="bookmark" title="C Tutorial &#8211; Binary File I/O">
		C Tutorial &#8211; Binary File I/O</a></h1>

			<div class="entry">
				<div id="watermark">
					<p>In an earlier tutorial we talked about file I/O functions and the use of text files. In this C programming tutorial we are going to talk about the use of binary files.</p>
<h2>Binary files</h2>
<p>Binary files are very similar to arrays of structures, except the structures are in a disk-file rather than an array in memory. Binary files have two features that distinguish them from text files:<span id="more-218"></span></p>
<ul>
<li>You can instantly use any structure in the file.</li>
<li>You can change the contents of a structure anywhere in the file.</li>
</ul>
<p>After you have opened the binary file, you can read and write a structure or seek a specific position in the file. A file position indicator points to record 0 when the file is opened.</p>
<p>A read operation reads the structure where the file position indicator is pointing to. After reading the structure the pointer is moved to point at the next structure.</p>
<p>A write operation will write to the currently pointed-to structure. After the write operation the file position indicator is moved to point at the next structure.</p>
<p>The fseek function will move the file position indicator to the record that is requested.</p>
<p>Remember that you keep track of things, because the file position indicator can not only point at the beginning of a structure, but can also point to any byte in the file.</p>
<p>The fread and fwrite function takes four parameters:</p>
<ul>
<li>A memory address</li>
<li>Number of bytes to read per block</li>
<li>Number of blocks to read</li>
<li>A file variable</li>
</ul>
<p>For example:</p>
<div id="cl">
<pre><code>	
     fread(&amp;my_record,sizeof(struct rec),1,ptr_myfile);

</code></pre>
</div>
<p>This fread statement says to read x bytes (size of rec) from the file ptr_myfile into memory address &amp;my_record. Only one block is requested. Changing the one into ten will read in ten blocks of x bytes at once.</p>
<p>Let&#8217;s look at a write example:</p>
<div id="cl">
<pre><code>
	#include&lt;stdio.h&gt;

	/* Our structure */
	struct rec
	{
		int x,y,z;
	};

	int main()
	{
		int counter;
		FILE *ptr_myfile;
		struct rec my_record;

		ptr_myfile=fopen("test.bin","wb");
		if (!ptr_myfile)
		{
			printf("Unable to open file!");
			return 1;
		}
		for ( counter=1; counter &lt;= 10; counter++)
		{
			my_record.x= counter;
			fwrite(&amp;my_record, sizeof(struct rec), 1, ptr_myfile);
		}
		fclose(ptr_myfile);
		return 0;
	}

</code></pre>
</div>
<p>In this example we declare a structure rec with the members x,y and z of the type integer. In the main function we open (fopen) a file for writing (w). Then we check if the file is open, if not, an error message is displayed and we exit the program. In the &#8220;for loop&#8221; we fill the structure member <strong>x</strong> with a number. Then we write the record to the file. We do this ten times, thus creating ten records. After writing the ten records, we will close the file (don&#8217;t forget this).</p>
<p>So now we have written to a file, let&#8217;s read from the file we have just created. Take a look at the example:</p>
<div id="cl">
<pre><code>
	#include&lt;stdio.h&gt;

	/* Our structure */
	struct rec
	{
		int x,y,z;
	};

	int main()
	{
		int counter;
		FILE *ptr_myfile;
		struct rec my_record;

		ptr_myfile=fopen("test.bin","rb");
		if (!ptr_myfile)
		{
			printf("Unable to open file!");
			return 1;
		}
		for ( counter=1; counter &lt;= 10; counter++)
		{
			fread(&amp;my_record,sizeof(struct rec),1,ptr_myfile);
			printf("%d\n",my_record.x);
		}
		fclose(ptr_myfile);
		return 0;
	}

</code></pre>
</div>
<p>The only two lines that are changed are the two lines in the &#8220;for loop&#8221;. With the fread we read-in the records (one by one). After we have read the record we print the member x (of that record).</p>
<p>The only thing we need to explain is the fseek option. The function fseek must be declared like this:</p>
<div id="cl">
<pre><code>	
     int fseek(FILE * stream, long int offset, int whence);

</code></pre>
</div>
<p>The fseek function sets the file position indicator for the stream pointed to by the stream. The new position, measured in characters from the beginning of the file, is obtained by adding offset to the position specified by whence. Three macros are declared in stdio.h called: SEEK_SET, SEEK_CUR and SEEK_END.</p>
<p>If the position declared by whence is SEEK_SET, then the position is the beginning of the file.</p>
<p>The SEEK_END can be used if you want to go to the end of the file. (Using negative numbers it is possible to move from the end of the file.)</p>
<p>If whence is SEEK_CUR then the position is set, x bytes, from the current position.</p>
<p>Let&#8217;s take a look at an example:</p>
<div id="cl">
<pre><code>
	#include&lt;stdio.h&gt;

	/* Our structure */
	struct rec
	{
		int x,y,z;
	};

	int main()
	{
		int counter;
		FILE *ptr_myfile;
		struct rec my_record;

		ptr_myfile=fopen("test.bin","rb");
		if (!ptr_myfile)
		{
			printf("Unable to open file!");
			return 1;
		}
		for ( counter=9; counter &gt;= 0; counter--)
		{
			fseek(ptr_myfile,sizeof(struct rec)*counter,SEEK_SET);
			fread(&amp;my_record,sizeof(struct rec),1,ptr_myfile);
			printf("%d\n",my_record.x);
		}
		fclose(ptr_myfile);
		return 0;
	}

</code></pre>
</div>
<p>In this example we are using fseek to seek the last record in the file. This record we read with fread statement and with the printf statement we print member x of the structure my_record. As you can see the &#8220;for loop&#8221; also changed. The &#8220;for loop&#8221; will now countdown to zero. This counter is then used in the fseek statement to set the file pointer at the desired record. The result is that we read-in the records in the reverse order.</p>
<p>A last note: if you set the file position indicator to a position in a file and you want the first position in a file then you can use the function rewind to the first position in the file. The function rewind can be used like this:</p>
<div id="cl">
<pre><code>
	#include&lt;stdio.h&gt;

	/* Our structure */
	struct rec
	{
		int x,y,z;
	};

	int main()
	{
		int counter;
		FILE *ptr_myfile;
		struct rec my_record;

		ptr_myfile=fopen("test.bin","rb");
		if (!ptr_myfile)
		{
			printf("Unable to open file!");
			return 1;
		}

		fseek(ptr_myfile, sizeof(struct rec), SEEK_END);
		rewind(ptr_myfile);

		for ( counter=1; counter &lt;= 10; counter++)
		{
			fread(&amp;my_record,sizeof(struct rec),1,ptr_myfile);
			printf("%d\n",my_record.x);
		}
		fclose(ptr_myfile);
		return 0;
	}

</code></pre>
</div>
<p>With the fseek statement in this example we go to the end of the file. Then we rewind to first position in the file. Then read-in all records and print the value of member x. Without the rewind you will get garbage. (<strong>Try it!</strong>)</p>
<p>That is all for this tutorial.</p>

If I understand your question correctly <strong>you want to see only 1byte in the binary file</strong>.<br />
If this is the case you are using the wrong variable type (int in this case), because an unsigned int can hold all the values between 0 and UINT_MAX  inclusive. UINT_MAX must be at least 65535. The int types must contain at least <strong>16 bits</strong> to hold the required range of values. <strong>But it also can vary per compiler and the platform you are compiling for. </strong></p>
<p>On compilers for 8 and 16 bit processors (including Intel x86 processors executing in 16 bit mode, such as under MS-DOS), an int is usually 16 bits and has exactly the same representation as a short. On compilers for 32 bit and larger processors (including Intel x86 processors executing in 32 bit mode, such as Win32 or Linux) an int is <strong>usually 32 bits</strong> long and has exactly the same representation as a long. A int holds 32 bits (thus you see 01 00 00 00 in your hex-editor).</p>
<p>Try the following example to see the sizeof variable types for your target platform:<br />
#include&lt;stdio.h&gt;</p>
<p>int main()<br />
{<br />
	printf(&#8220;%d %d \n&#8221;, sizeof(int), sizeof(short int));<br />
	printf(&#8220;%d %d \n&#8221;, sizeof(signed int), sizeof(unsigned int));<br />
	printf(&#8220;%d %d \n&#8221;, sizeof(char), sizeof(unsigned char));<br />
	return 0;<br />
}</p>
<p>The output will be something like this (on a linux intel machine):<br />
4 2<br />
4 4<br />
1 1 </p>

#include&lt;stdio.h&gt;</p>

You are right, if you only look at the syntax. But if you open test.bin with text editor, you will see all kinds of ascii values. This is because we write/read records (structs.)<br />
So there will be now difference between the content of a file written with only &#8220;w&#8221; or &#8220;wb&#8221;. But to keep the syntax correct, we have changed the source code examples.<br />
The modes you can use are as follows:<br />
r  &#8211; open for reading<br />
w  &#8211; open for writing (file need not exist)<br />
a  &#8211; open for appending (file need not exist)<br />
r+ &#8211; open for reading and writing, start at beginning<br />
w+ &#8211; open for reading and writing (overwrite file)<br />
a+ &#8211; open for reading and writing (append if file exists)</p>
<p>To open a file in binary mode you must add the letter <strong>b</strong> to the end of the mode string; for example, &#8220;wb&#8221;.<br />
For the read and write modes, you can add the b, either after the plus sign &#8211; &#8220;r+b&#8221; &#8211; or before &#8211; &#8220;rb+&#8221;.</p>
