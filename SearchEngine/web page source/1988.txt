&#x2022; There are two exceptions to requiring a variable
name for a method call
&#x2013; Calling a method defined inside the current class definition
&#x2022; Use &#x201C;methodName(args)&#x201D; instead of &#x201C;varName.methodName(args)&#x201D;
&#x2013; Functions (methods) that are declared &#x201C;static&#x201D;
&#x2022; Use &#x201C;ClassName.methodName(args)&#x201D;
&#x2022; Calling a method of the current class
&#x2013; You don&#x2019;t need the variable name and the dot
&#x2013; For example, a Ship class might define a method called
degreeesToRadians, then, within another function in the same class
definition, do this:
double angle = degreesToRadians(direction);
&#x2022; No variable name and dot is required in front of
degreesToRadians since it is defined in the same class as the
method that is calling it

    Method Visibility
&#x2022; public/private distinction
&#x2013; A declaration of private means that &#x201C;outside&#x201D; methods
can&#x2019;t call it &#x2013; only methods within the same class can
&#x2022; Thus, for example, the main method of the Test2 class
could not have done

&#x2013; Attempting to do so would have resulted in an error at compile time
&#x2013; Only say public for methods that you want to guarantee
your class will make available to users
&#x2013; You are free to change or eliminate private methods
without telling users of your class

    Static Methods
&#x2022; Also called &#x201C;class methods&#x201D;
(vs. normal &#x201C;instance methods&#x201D;)
&#x2013; Static functions do not access any non-static methods or fields within their
class and are almost like global functions in other languages
&#x2022; Call a static method through the class name
&#x2013; ClassName.functionName(arguments);
&#x2022; Example: Math.cos
&#x2013; The Math class has a static method called cos that expects a double
precision number as an argument. So, you can call Math.cos(3.5) without
ever having any object (instance) of the Math class
&#x2022; double cosine = Math.cos(someAngle);
&#x2022; Note on the main method
&#x2013; Since the system calls main without first creating an object, static methods
are the only type of methods that main can call directly (i.e. without
building an object and calling the method of that object)

    Constructors
&#x2022; Constructors are special functions called when a
class is created with new
&#x2013; Constructors are especially useful for supplying values of fields
&#x2013; Constructors are declared through:
public ClassName(args) {
...
}
&#x2013; Notice that the constructor name must exactly match the class name
&#x2013; Constructors have no return type (not even void), unlike a regular
method
&#x2013; Java automatically provides a zero-argument constructor if and only
if the class doesn&#x2019;t define it&#x2019;s own constructor
&#x2022; That&#x2019;s why, in the first example, you could say
Ship1 s1 = new Ship1();
