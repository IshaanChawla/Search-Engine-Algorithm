						<div class="post-headline">		<h1>8.10 &#8212; Const class objects and member functions</h1>
		</div>		<div class="post-byline">By Alex, on September 11th, 2007</div>		<div class="post-bodycopy clearfix"><p>In the lesson on <a href="http://www.learncpp.com/cpp-tutorial/73-passing-arguments-by-reference/">passing arguments by reference</a>, we covered the merits of passing function parameters as const variables.  To recap, making variables const ensures their values are not accidentally changed.  This is particularly important when passing variables by reference, as callers generally will not expect the values they pass to a function to be changed.</p>
<p>Just like the built-in data types (int, double, char, etc&#8230;), class objects can be made const by using the const keyword.  All const variables must be initialized at time of creation.  In the case of built-in data types, initilization is done through explicit or implicit assignment:</p>

		<p>The leftmost const applies to the return value of the function.  GetValue() is returning a const reference to an int.  If the return value were not const, we could do something like this:</p>
<p>cSomething.GetValue() = 5;</p>
<p>This works because cSomething.GetValue() would return a reference to an int, which is set to m_nValue.  So this essentially becomes:</p>
<p>m_nValue = 5;</p>
<p>Obviously this defeats the spirit of the function, so we make the return value const to ensure people don&#8217;t modify the member variables via assignment this way.</p>
<p>The rightmost const applies to the function.  This means the GetValue() can be called on const objects because it&#8217;s guaranteed not to modify any of the member values.</p>
