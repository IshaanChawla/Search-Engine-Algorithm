<h2>ANSI (Windows-1252)</h2>
<p>ANSI was the default character set in Windows up to Windows 95.</p>

<p>ANSI is also called Windows-1252.</p>

<hr>

<h2>Important Note</h2>
<p>ANSI and ISO-8859-1 are very similar. They only differ in 32 characters.</p>
<p>In ANSI, the characters from 128 to 159 are used for some useful characters 
such as the Euro symbol.</p>
<p>In ISO-8859-1 these characters are mapped to control characters which are 
useless in HTML.</p>
<p>Many web developers declare ISO-8859-1 and use these 32 values as if they 
were using Windows-1252.</p>
<p>Because of this common misunderstanding, browsers will change to Windows-1252 
when ISO-8859-1 is declared. This is done for any DOCTYPE: HTML4, HTML5, and 
XHTML.</p>
<hr>

<h2>ANSI and ASCII</h2>
<p>The first part of ANSI (entity numbers from 0-127) is the original ASCII 
character-set. It contains numbers, upper and lowercase English letters, and 
some special characters.</p>
<p>For a closer look, please study our <a href="ref_html_ascii.asp">Complete 
ASCII Reference</a>.</p>
<hr>

<h2>ANSI Control Characters</h2>

<p>The ANSI control characters (range 00-31, plus 127) were designed to control 
hardware devices.</p>
<p>Control characters (except horizontal tab, line feed, and carriage return) 
have nothing to do inside an HTML document.</p>
