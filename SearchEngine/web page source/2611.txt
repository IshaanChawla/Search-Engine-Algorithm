                  <div>
                     <h2 class="title"><a name="hamcrestoverview"></a>1.&nbsp;Purpose of the Hamcrest matcher framework 
                     </h2>
                  </div>
               </div>
            </div><a class="indexterm" name="d396748e41"></a><p>
               <em class="wordasword">Hamcrest</em>
               is a framework for writing matcher objects allowing 'match'
               rules to be
               defined declaratively. Hamcrest can be
               integrated with other testing frameworks and simplifies the definition of writing tests.
               
            </p>
            <p>
               To use Hamcrest matchers in JUnit you use the
               <code class="code">assertThat</code>
               statement.
               
            </p>
            <p>
               Hamcrest is typically viewed as a third generation matcher framework. The first generation used
               <code class="code">assert(logical statement)</code>
               but such tests were not easily readable. The second generation introduced special methods for assertions, e.g.,
               <code class="code">assertEquals()</code>
               but that lead to lots of assert methods. Hamcrest uses
               <code class="code">assertThat</code>
               with a matcher which identifies if the assert was sucessful. See
               <a class="ulink" href="http://en.wikipedia.org/wiki/Hamcrest" target="_top">Wiki on Hamcrest</a>
               for more details.
               
            </p>
         </div>
         <div class="section">

               <div>
                  <div>
                     <h2 class="title"><a name="hamcrest_installation"></a>2.&nbsp;Using Hamcrest matchers
                     </h2>
                  </div>
               </div>
            </div><a class="indexterm" name="d396748e71"></a><div class="section">
               <div class="titlepage">
                  <div>
                     <div>
                        <h3 class="title"><a name="hamcrest_gradle"></a>2.1.&nbsp;Defining a Hamcrest dependency for Gradle
                        </h3>
                     </div>
                  </div>
               </div>
               <p>To use Hamcrest matchers for a project based on the Gradle build system, add the following dependencies to it.
                  
               </p>
               <p>
                  
               </p><pre class="programlisting">dependencies {
    <em class="hl-comment" style="color: #080">// Unit testing dependencies</em>
    testCompile <span class="hl-string" style="color:blue">'junit:junit:4.12'</span>
    <em class="hl-comment" style="color: #080">// Set this dependency if you want to use Hamcrest matching</em>
    androidTestCompile <span class="hl-string" style="color:blue">'org.hamcrest:hamcrest-library:1.1'</span>
} </pre><p>
                  
               </p>
            </div>
            <div class="section">
               <div class="titlepage">
                  <div>
                     <div>
                        <h3 class="title"><a name="hamcrest_eclipse"></a>2.2.&nbsp;Adding Hamcrest directly to the classpath in Eclipse
                        </h3>
                     </div>
                  </div>
               </div>
               <p>
                  The JUnit distribution included in Eclipse only contain the core
                  Hamcrest matcher. To use all available matchers,
                  download the latest
                  hamcrest-all-*.jar from
                  https://code.google.com/p/hamcrest/downloads/list and add it to your
                  projects classpath.
                  
               </p>
               <div class="warning" style="margin-left: 0.5in; margin-right: 0.5in;">
                  <h3 class="title">Warning</h3>
                  <p>
                     If you get the following exception "java.lang.SecurityException:
                     class "org.hamcrest.Matchers"'s signer information
                     does not match
                     signer information of other classes in the same package", ensure that
                     the hamcrest jar is before the
                     Junit library in the build path. You
                     can configure the order in the project properties under
                     <span class="guilabel">Java Build Path</span>
                     on the
                     <span class="guilabel">Order and Export</span>
                     tab.
                     
                  </p>
                  <p>
                     
                  </p>

         <div class="section">
            <div class="titlepage">
               <div>
                  <div>
                     <h2 class="title"><a name="hamcrest_usage"></a>3.&nbsp;Using Hamcrest
                     </h2>
                  </div>
               </div>
            </div>
            <p>The usage of Hamcrest matches is demonstrates by the following
               code snippet.
               
            </p>
            <p>
               
            </p><pre class="programlisting">assertThat(Long.valueOf(<span class="hl-number">1</span>), instanceOf(Integer.<span class="hl-keyword">class</span>)); </pre><p>
               
            </p>
            <p>
               The following list contains the most important Hamcrest matchers.
               
            </p>
            <div class="itemizedlist">
               <ul class="itemizedlist" type="disc">
                  <li class="listitem">
                     <p>is - decorator to improve readability</p>
                  </li>
                  <li class="listitem">
                     <p>allOf - matches if all matchers match (short circuits)</p>
                  </li>
                  <li class="listitem">
                     <p>anyOf - matches if any matchers match (short circuits)</p>
                  </li>
                  <li class="listitem">
                     <p>not - matches if the wrapped matcher doesn't match and vice
                        versa
                        
                     </p>
                  </li>
                  <li class="listitem">
                     <p>equalTo - test object equality using Object.equals</p>
                  </li>
                  <li class="listitem">
                     <p>hasToString - test Object.toString</p>
                  </li>
                  <li class="listitem">
                     <p>instanceOf, isCompatibleType - test type</p>
                  </li>
                  <li class="listitem">
                     <p>notNullValue, nullValue - test for null</p>
                  </li>
                  <li class="listitem">
                     <p>sameInstance - test object identity</p>
                  </li>
                  <li class="listitem">
                     <p>hasEntry, hasKey, hasValue - test a map contains an entry, key
                        or value
                        
                     </p>
                  </li>
                  <li class="listitem">
                     <p>hasItem, hasItems - test a collection contains elements</p>
                  </li>
                  <li class="listitem">
                     <p>hasItemInArray - test an array contains an element</p>
                  </li>
                  <li class="listitem">
                     <p>closeTo - test floating point values are close to a given
                        value
                        
                     </p>
                  </li>
                  <li class="listitem">
                     <p>greaterThan, greaterThanOrEqualTo, lessThan, lessThanOrEqualTo
                        - test ordering
                        
                     </p>
                  </li>
                  <li class="listitem">
                     <p>equalToIgnoringCase - test string equality ignoring case
                        
                     </p>
                  </li>
                  <li class="listitem">
                     <p>equalToIgnoringWhiteSpace - test string equality ignoring
                        differences in runs of whitespace
                        
                     </p>
                  </li>
                  <li class="listitem">
                     <p>containsString, endsWith, startsWith - test string matching
                        
                     </p>
                  </li>
               </ul>
            </div>
            <p>
               
            </p>
            <p>
               Hamcrest has the target to make tests as readable as possible. For
               example the
               <code class="code">is</code>
               method is a thin wrapper for
               <code class="code">equalTo(value)</code>.
               
            </p>
            <p>
               
            </p><pre class="programlisting"><em class="hl-comment" style="color: #080">// all statements test the same</em>
assertThat(a, equalTo(b));
assertThat(a, is(equalTo(b)));
assertThat(a, is(b)); </pre><p>
               
            </p>
         </div>
