		</div>		<div class="post-byline">By Alex, on December 16th, 2007</div>		<div class="post-bodycopy clearfix"><p>In this section, we&#8217;ll address some of the common issues that new programmers seem to run across with fairly high probability.  This is not meant to be a comprehensive list of compilation or execution problems, but rather a pragmatic list of solutions to very basic issues.  If you have any suggestions for other issues that might be added to this list, post them in the comments section below.</p>

<p><strong>Answer 1:</strong> Some compilers (eg. Bloodshed&#8217;s Dev C++) don&#8217;t automatically pause the console screen after the program has finished executing.  If this is the case with your compiler, the following two steps will fix your problem:</p>

<p><a name="Q4"></a><strong>Problem 4: When trying to use endl to end a printed line, the compiler says end1 is an &#8220;undeclared identifier&#8221;</strong></p>
<p><strong>Answer 4:</strong> Make sure you do not mistake the letter l (lower case L) in endl for the number 1.  endl is all letters.  I recommend using a font that makes it clear the differences between the letter lower case L, upper case i, and the number 1.  Also the letter capital o and the number zero can easily be confused in many non-programming fonts.</p>
</td>
</tr>

<tr>
<td>
<p><a name="Q5"></a><strong>Problem 5: My program compiles but it isn&#8217;t working correctly.  What do I do?</strong></p>
<p><strong>Answer 5:</strong> Debug it!  You can find information on how to debug programs in lesson 1, specifically sections <a href="http://www.learncpp.com/cpp-tutorial/111-debugging-your-program-stepping-and-breakpoints/">1.11</a> and <a href="http://www.learncpp.com/cpp-tutorial/111a-debugging-your-program-watching-variables-and-the-call-stack/">1.11a</a>..</p>
</td>
</tr>

<table border=1 cellspacing=0 cellpadding=3 width=100%>
<tr>
<td>
<p><a name="Q6"></a><strong>Problem 6: How do I turn on line numbering in Visual Studio?</strong></p>
<p><strong>Answer 6:</strong> Go to the Tools Menu, and choose Options.  Under the Text Editor submenu, choose All Languages (or C/C++), and you&#8217;ll see a checkbox on the right for Line numbers.</p>
</td>
</tr>
</table>
<table border=1 cellspacing=0 cellpadding=3 width=100%>
<tr>
<td>
<p><a name="Q7"></a><strong>Problem 7: When I compile my program in Visual Studio 2010, I get an error message about a COFF file being invalid.  How do I fix this?</strong></p>
<p><strong>Answer 7:</strong> If you see the following error when compiling with Visual Studio 2010:</p>
<pre>LINK : fatal error LNK1123: failure during conversion to COFF: file invalid or corrupt</pre>
<p>You&#8217;ve encountered a Microsoft OS/compiler incompatibility.  It has nothing to do with your code.</p>
<p>The best first option is to download and install <a href="http://www.microsoft.com/en-us/download/details.aspx?id=23691" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://www.microsoft.com/en-us/download/details.aspx?id=23691', 'Visual Studio 2010 Service Pack 1']);" >Visual Studio 2010 Service Pack 1</a>.</p>
<p>If that does not fix your issue, there are many other good suggestions on <a href="http://stackoverflow.com/questions/10888391/error-link-fatal-error-lnk1123-failure-during-conversion-to-coff-file-inval" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://stackoverflow.com/questions/10888391/error-link-fatal-error-lnk1123-failure-during-conversion-to-coff-file-inval', 'this Stack Overflow thread']);" >this Stack Overflow thread</a> about the various causes and solutions to this problem.</p>
</td>
</tr>
</table>
<table border=1 cellspacing=0 cellpadding=3 width=100%>
<tr>
<td>
<p><a name="Q8"></a><strong>Problem 8: When I compile my program, I get an error about unresolved external symbol _main or _WinMain@16</strong></p>
<p><strong>Answer 8:</strong> This means your compiler can&#8217;t find your main() function.  All programs must include a main() function.</p>
<p>There are a few things to check:<br />
a) Does your code include a function named main?<br />
b) Is main spelled correctly?<br />
c) Is the file containing main part of your project? (if not, either move the main function to one that is, or add the file to your project.  See lesson <a href="http://www.learncpp.com/cpp-tutorial/18-programs-with-multiple-files/">1.8 -- Programs with multiple files</a> for more information about how to do this)</a>.<br />
d) Is the file containing function main set to compile?  (Also see lesson <a href="http://www.learncpp.com/cpp-tutorial/18-programs-with-multiple-files/">1.8 -- Programs with multiple files</a> for more information about how to do this)</a>.</p>
</td>
</tr>
</table>
<table border=1 cellspacing=0 cellpadding=3 width=100%>
<tr>
<td>
<p><a name="Q9"></a><strong>Problem 9: When I compile my program, I get a warnings about &#8220;Cannot find or open the PDB file&#8221;</strong></p>
<p><strong>Answer 9:</strong> This is a warning, not an error, so it shouldn&#8217;t impact your program working.  However, it is annoying.  To fix it, go into the Debug menu -> Options and Settings -> Symbols, and check &#8220;Microsoft Symbol Server&#8221;.</p>
</td>
</tr>
</table>
<table border=1 cellspacing=0 cellpadding=3 width=100%>
<tr>
<td>
<p><a name="Q10"></a><strong>Problem 10: I&#8217;m using Code::Blocks or g++ on the command line, and none of the C++11 functionality works</strong></p>
<p><strong>Answer 10:</strong> For Code::Blocks, go to Project->Build options->Compiler settings->Compiler flags and check Have g++ follow C++11 ISO C++ language standard</p>
<p>For compiling with g++ on the command line, add the following to the command line: <code>-std=c++11</code></p>
</td>
</tr>

<td>
<p><strong>I have some other problem that I can&#8217;t figure out.  How can I get an answer quickly?</strong></p>
<p><strong>Answer:</strong> As you progress through the material, youll undoubtedly have questions or run into unexpected problems. The best and fastest place to get answers to your questions is on a site designed for programming questions and answers, like <a href="http://www.stackoverflow.com" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://www.stackoverflow.com', 'Stack Overflow']);" >Stack Overflow</a>.  Try posting your question there.  Remember to be thorough about what your problem is, and include all relevant information like what OS you&#8217;re on and what IDE you&#8217;re using.</p>

<p></p>
<p>First, <code>cin >> chIgnore</code> ignores the enter key if the user does not provide any other input.  You can verify this by hitting enter when the <code>cin >> chIgnore</code> line is being executed.  On the other hand, cin.get() accepts the enter key as valid input.</p>
<p>But perhaps more importantly is the second phenomena that occurs when more characters are placed into the input stream (cin) than are read out.  When asked for a letter in the above program, enter more than one letter and watch what happens.</p>
<p>Let&#8217;s say you enter &#8220;abcde&#8221;.  <code>cin >> buf[0]</code> reads the &#8216;a&#8217; character, but &#8220;bcde&#8221; are still left in the input stream.  Consequently, when the code gets to <code>cin >> chIgnore</code>, it reads the waiting &#8216;b&#8217; character out of the input stream and doesn&#8217;t pause at all!</p>
<p>The 3 step process is a little more foolproof.  First, <code>cin.clear()</code> clears any errors the user may have caused by providing invalid input in the past.  Second, <code>cin.ignore(255, '\n')</code> gets rid of up to 255 waiting characters in the input stream (or until a \n is encountered), which is almost always going to be enough to clear out the input stream.  Finally, we use <code>cin.get()</code> to wait for a character.  Since we just cleared the input stream, there are no characters waiting.  Consequently, the program will wait for the user to enter something.</p>
<p>In 99% of the cases you encounter, <code>cin >> myVariable</code> will probably work exactly as you expect.  But since the 3 line method is generally more foolproof and isn&#8217;t particularly burdensome to implement, I&#8217;d say it&#8217;s the superior solution.</p>

		</div>
		<p>For anyone else interested in this question:</p>
<p>clrscr() and getch() are part of the conio.h library (non-standard library that won&#8217;t come with most compilers and is not available on Linux). </p>
<p>clrscr() just clears the console screen.</p>
<p>getch() is a variant of the standard getchar() function. getch() waits for the user to press any character on the keyboard. The function returns the character, but doesn&#8217;t print the character to the screen like getchar() does. It also does not require the user to press the Enter key in order to submit the character (this is convenient because you no longer need to flush the standard input stream). </p>
<p>On Linux, you can get similar functionality by installing the curse.h library.</p>
