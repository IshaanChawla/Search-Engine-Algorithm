<h1 class="h1">Synchronization in Java</h1>
<p>Synchronization in java is the capability <em>to control the access of multiple threads to any shared resource</em>.</p>
<p>Java Synchronization is better option where we want to allow only one thread to access the shared resource.</p>
<hr/>
<h3 class="h2">Why use Synchronization</h3>
<p>The synchronization is mainly used to</p>
<ol>
<li>To prevent thread interference.</li>
<li>To prevent consistency problem.</li>
</ol>

<hr/>
<h3 class="h2">Types of Synchronization</h3>
<p>There are two types of synchronization</p>
<ol>
<li>Process Synchronization</li>
<li>Thread Synchronization</li>
</ol>
<p>
Here, we will discuss only thread synchronization.
</p>
<hr/>
<h3 class="h2">Thread Synchronization</h3>
<p>There are two types of thread synchronization mutual exclusive and inter-thread communication.
<ol class="points">
<li>Mutual Exclusive
<ol>
<li>Synchronized method.</li>
<li>Synchronized block.</li>
<li>static synchronization.</li>
</ol>
</li>
<li>Cooperation (Inter-thread communication in java)</li>
</ol>

<hr/>

<h3 class="h2">Mutual Exclusive</h3>
<p>Mutual Exclusive helps keep threads from interfering with one another while sharing data. This can be done by three ways in java:</p>
<ol class="points">
<li>by synchronized method</li>
<li>by synchronized block</li>
<li>by static synchronization</li>
</ol>

<hr/>
<h3 class="h2">Concept of Lock in Java</h3>
<p> Synchronization is built around an internal entity known as the lock or monitor. Every object has an lock associated with it. By convention, a thread that needs consistent access to an object's fields has to acquire the object's lock before accessing them, and then release the lock when it's done with them. 
</p>
<p>From Java 5 the package java.util.concurrent.locks contains several lock implementations.</p>
<hr/>
<h3 class="h2">Understanding the problem without Synchronization</h3>
<p>In this example, there is no synchronization, so output is inconsistent. Let's see the example:

<h3 class="h2">Java synchronized method</h3>
<p>If you declare any method as synchronized, it is known as synchronized method.</p>
<p>Synchronized method is used to lock an object for any shared resource.</p>
<p>When a thread invokes a synchronized method, it automatically acquires the lock for that object and releases it when the thread completes its task.</p>


<div class="codeblock"><textarea name="code" class="java" >
