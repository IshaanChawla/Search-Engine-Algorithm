<td  id="content" colspan="7"><h1 class="content_heading">SCWCD Training Courses</h1>
<h1 class="subtitle">For Sun Certified Web Component Developer Exam</h1><!-- #BeginLibraryItem "/Library/Quotes.lbi" -->
<div class="pull_quote" style="width: 230px; margin-right: 10px; margin-bottom: 10px;">

<p><a href="http://courses.coreservlets.com/student-reviews.html">more student reviews</a></p>
</div><!-- #EndLibraryItem --><p>
  Here are the topics covered in the SCWCD course taught by 
  <a href="about-instructor.html">Marty Hall</a>. If you have a group of 
  at least eight interested developers (10 for courses outside the US or Canada),
  <a href="mailto:hall@coreservlets.com">contact Marty</a> to arrange a course
  at your location. 
</p>
<p>
  This course can mostly be viewed as a combination of the 
  <a href="servlet+jsp-training.html">intermediate</a> and
  <a href="advanced-servlet+jsp-training.html">advanced</a>
  servlet and JSP training courses. However, this course:
</p>
<div style="padding-left: 215px;">
  <ul>
    <li>Adds two topics ("classic" old-style custom tags and
      six specific design patterns) that are not normally covered 
      in either the intermediate or advanced
      courses, but that are required for the SCWCD exam.</li>
    <li>In the interest of time, drops or speeds over a few of the
      best-practices topics that are not specifically part of the SCWCD exam.
      Omits all extra topics (JDBC, Ajax, Java 5 or Java 6 topics, Struts, JSF, etc.)
      that are not part of the exam objectives.</li>
    <li>Ties each topic to specific objectives on the SCWCD exam.</li>
    <li>Adds non-programming exercises relevant to the 
      SCWCD exam.</li>
  </ul>
</div>
<p>
  If your <i>only</i> goal is attaining real-world developer skills, the
  <a href="servlet+jsp-training.html">intermediate</a> 
  or <a href="advanced-servlet+jsp-training.html">advanced</a>
  courses (or a customized combination of topics!)
  are better options because there is more flexibility
  in the topics and the pace of coverage. Also, in the regular
  servlet and JSP courses, the exercises
  focus only on real-world skills, whereas some of the SCWCD
  exercises are about memorizing terminology, packages, and class names (because
  that is what the SCWCD exam asks).
  However, if you <i>also</i> want certification for your developers,
  this is the course for you.
</p>
<br clear="all"/>
<h2 class="book_part">Syllabus</h2>
  <h2 class="topics">Overview and Setup</h2>
  <ul>
    <li>Understanding the role of servlets</li>
    <li>Evaluating servlets vs. other technologies</li>
    <li>Understanding the role of JSP</li>
    <li>Configuring the server</li>
    <li>Configuring your development environment</li>
    <li>Testing the server setup</li>
  </ul>
  <h2 class="topics">Servlet Basics</h2>
  <ul>
    <li>The basic structure of servlets</li>
    <li>A simple servlet that generates plain text</li>
    <li>A servlet that generates HTML</li>
    <li>Servlets and packages</li>
    <li>Some utilities that help build HTML</li>
    <li>The servlet life cycle</li>
    <li>Servlet debugging strategies</li>
  </ul>
  <h2 class="topics">Handling the Client Request: Form Data</h2>
  <ul>
    <li>The role of form data</li>
    <li>Creating and submitting HTML forms</li>
    <li>Reading individual request parameters</li>
    <li>Reading the entire set of request parameters</li>
    <li>Handling missing and malformed data</li>
    <li>Dealing with incomplete form submissions</li>
    <li>Filtering special characters out of the request parameters</li>
  </ul>
  <h2 class="topics">Handling the Client Request: HTTP Request Headers</h2>
  <ul>
    <li>Reading HTTP request headers</li>
    <li>Building a table of all the request headers</li>
    <li>Understanding the various request headers</li>
    <li>Reducing download times by compressing pages</li>
    <li>Differentiating among types of browsers</li>
  </ul>
  <h2 class="topics">Generating the Server Response: HTTP Status Codes</h2>
  <ul>
    <li>Format of the HTTP response</li>
    <li>How to set status codes</li>
    <li>What the status codes are good for</li>
    <li>Shortcut methods for redirection and error pages</li>
    <li>A servlet that redirects users to browser-specific pages</li>
    <li>A front end to various search engines</li>
  </ul>
  <h2 class="topics">Generating the Server Response: HTTP Response Headers</h2>
  <ul>
