    Object Arrays:
Performance Problems
&#x2022; For very large arrays, undue paging can occur
&#x2013; Array of references (pointers) allocated first
&#x2013; Individual objects allocated next
&#x2013; Thus, for very large arrays of objects, reference and object can be on
different pages, resulting in swapping for each array reference
&#x2013; Example
String[] names = new String[10000000];
for(int i=0; i&lt;names.length; i++) {
names[i] = getNameFromSomewhere();
}
&#x2022; Problem does not occur with arrays of primitives
&#x2013; I.e., with arrays of int, double, and other types that start with
lowercase letter
&#x2013; Because system stores values directly in arrays, rather than storing
references (pointers) to the objects
