<div style="color:#000080;font-family:Arial;font-size:10pt">

<p>The <b>ArrayList
</b>class extends <b>AbstractList
</b>and implements the <b>List
</b>interface. <b>ArrayList
</b>
supports dynamic arrays that can grow as needed. In Java,
standard arrays are of a fixed length. After arrays are created, they cannot grow or shrink,
which means that you must know in advance how many elements an array will hold. But,
sometimes, you may not know until run time precisely how large of an array you need. To
handle this situation, the collections framework defines <b>ArrayList</b>.
In essence, an <b>ArrayList
</b>is a variable-length array of object references. That is, an <b>ArrayList
</b>can dynamically increase
or decrease in size. Array lists are created with an initial size. When this
size is exceeded, the collection is automatically enlarged. When objects are removed, the array
may be shrunk.</p>
<p>
<b>
ArrayList </b>has
the constructors shown here:
<code>
<br>ArrayList( )
<br>ArrayList(Collection <i>c</i>)
<br>ArrayList(int <i>capacity</i>)
</code></p>
<p>The first constructor builds an empty array list. The second
constructor builds an array list that is initialized with the elements of the collection <i>c</i>.
The third constructor builds an array list that has the specified initial <i>capacity</i>.
The capacity is the size of the underlying array that is used to store the elements. The capacity grows
automatically as elements are added to an array list.</p>
<p>The following program shows a simple use of <b>ArrayList</b>.
An array list is created, and then objects of type <b>String
</b>are added to it. (Recall
that a quoted string is translated into a
<b>
String </b>object.)
The list is then displayed. Some of the elements are removed and the list is displayed again.</p>
<code>
// Demonstrate ArrayList.
<br>import java.util.*;
<br>class ArrayListDemo {
<br>public static void main(String args[]) {
<br>// create an array list
<br>ArrayList al = new ArrayList();
<br>System.out.println(&quot;Initial size of al: &quot; +
<br>al.size());
<br>// add elements to the array list
<br>al.add(&quot;C&quot;);
<br>al.add(&quot;A&quot;);
<br>al.add(&quot;E&quot;);
<br>al.add(&quot;B&quot;);
<br>al.add(&quot;D&quot;);
<br>al.add(&quot;F&quot;);
<br>al.add(1, &quot;A2&quot;);
<br>System.out.println(&quot;Size of al after additions: &quot; +
<br>al.size());
<br>// display the array list
<br>System.out.println(&quot;Contents of al: &quot; + al);
<br>// Remove elements from the array list
<br>al.remove(&quot;F&quot;);
<br>al.remove(2);
<br>System.out.println(&quot;Size of al after deletions: &quot; +
<br>al.size());
<br>System.out.println(&quot;Contents of al: &quot; + al);
<br>}
<br>}</p>
</code>
<p>The output from this program is shown here:</p>
<code>Initial size of al: 0
<br>Size of al after additions: 7
<br>Contents of al: [C, A2, A, E, B, D, F]
<br>Size of al after deletions: 5
<br>Contents of al: [C, A2, E, B, D]</code>
<p>Notice that <b>a1
</b>starts out empty and
grows as elements are added to it. When elements are removed, its size is reduced.</p>
<p>Although the capacity of an <b>ArrayList
</b>object increases
automatically as objects are stored in it, you can increase the capacity of an <b>ArrayList
</b>object manually by
calling
<b>
ensureCapacity( )</b>.
You might want to do this if you know in advance that you will be storing many more items in the collection that it can currently
hold. By increasing its capacity once, at the start, you can prevent several
reallocations later. Because reallocations are costly in terms of time, preventing
unnecessary ones improves performance. The signature for <b>ensureCapacity(
) </b>is shown here:</p>
<p>void ensureCapacity(int <i>cap</i>)</p>
<p>Here, <i>cap
</i>is the new capacity. Conversely, if you want to reduce the size of the array that
underlies an <b>ArrayList
</b>object so that it is precisely as large as the number of items that it
is currently holding, call
<b>
<br>trimToSize( )</b>,
shown here:</p>
<code>void trimToSize( )</code>
<p>
<b>
Obtaining an Array from an ArrayList
</b>
</p>
<p>When working with <b>ArrayList</b>,
you will sometimes want to obtain an actual array that contains the contents of the list. As explained earlier, you can
do this by calling <b>toArray( )</b>.
Several reasons exist why you might want to convert a collection into an array
such as:</p>
<p>• To obtain faster processing times for certain operations.
<br>• To pass an array to a method that is not overloaded to
accept a collection.
<br>• To integrate your newer, collection-based code with legacy
code that does not understand collections.</p>
<p>Whatever the reason, converting an <b>ArrayList
</b>to an array is a trivial
matter, as the following program shows:</p>
<code>
// get array
<br>Object ia[] = al.toArray();
<br>int sum = 0;
<br>// sum the array
<br>for(int i=0; i&lt;ia.length; i++)
<br>sum += ((Integer) ia[i]).intValue();
<br>System.out.println(&quot;Sum is: &quot; + sum);
<br>}
<br>}
</code>
<p>The output from the program is shown here:
<code>
<br>Contents of al: [1, 2, 3, 4]
<br>Sum is: 10</code>
<p>The program begins by creating a collection of integers. As
explained, you cannot store primitive types in a collection, so objects of type <b>Integer
</b>are created and stored.
Next,
<b>
toArray( ) </b>is
called and it obtains an array of <b>Object</b>s.
The contents of this array are cast to <b>Integer</b>,
and then the values are summed.</p>
