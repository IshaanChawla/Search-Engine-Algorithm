<p class="MsoNormal" style="text-indent:.5in">Java supports recursion. Recursion
is the process of defining something in terms of itself. As it relates to java
programming, recursion is the attribute that allows a method to call itself. A
method that calls itself is said to be recursive.</p>
<p class="MsoNormal">&nbsp;<span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span>The classic example of recursion is the computation of the factorial of a
number. The factorial of a number N is the product of all the whole numbers
between 1 and N. for example, 3 factorial is 123, or 6. Here is how a
factorial can be computed by use of a recursive method.</p>
<p class="MsoNormal"><span style="font-family:&quot;Courier New&quot;">class
Factorial {<o:p>
</o:p>
</span></p>
<p class="MsoNormal"><span style="font-family:&quot;Courier New&quot;"><span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;
</span>int fact(int n) {<o:p>
</o:p>
</span></p>
<p class="MsoNormal"><span style="font-family:&quot;Courier New&quot;"><span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span>int result;<o:p>
</o:p>
</span></p>
<p class="MsoNormal"><span style="font-family:&quot;Courier New&quot;"><span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;
</span>if ( n ==1) return 1;<o:p>
</o:p>
</span></p>
<p class="MsoNormal"><span style="font-family:&quot;Courier New&quot;"><span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;
</span>result = fact (n-1) * n;<o:p>
</o:p>
</span></p>
<p class="MsoNormal"><span style="font-family:&quot;Courier New&quot;"><span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;
</span>return result;<o:p>
</o:p>
</span></p>
<p class="MsoNormal"><span style="font-family:&quot;Courier New&quot;"><span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;
</span>}<o:p>

</o:p>
</span></p>
<p class="MsoNormal"><span style="font-family:&quot;Courier New&quot;"><span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;
</span>public static void main (String args[]) {<o:p>
</o:p>
</span></p>
<p class="MsoNormal"><span style="font-family:&quot;Courier New&quot;"><span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;

</span><span style="font-family:&quot;Courier New&quot;">Factorial of 3 is 6<o:p>

<p class="MsoNormal" style="text-indent:.5in">If you are unfamiliar with
recursive methods, then the operation of <b style="mso-bidi-font-weight:normal">fact()</b>
may seem a bit confusing. Here is how it works. When <b style="mso-bidi-font-weight:
normal">fact() </b><span style="mso-spacerun: yes">&nbsp;</span>is called with
an argument of 1, the function returns 1; otherwise it returns the product of <b style="mso-bidi-font-weight:normal">fact(n-1)*n.</b>
to evaluate this expression, <b style="mso-bidi-font-weight:normal">fact() </b>is
called with <b style="mso-bidi-font-weight:normal">n-1</b>. this process repeats
until <b style="mso-bidi-font-weight:normal">n</b> equals 1 and the calls to the
method begin returning.</p>
<p class="MsoNormal">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
To better understand how the <b style="mso-bidi-font-weight:normal">fact() </b>method
works, lets go through a short example. When you compute the factorial of 3,
the first call to <b style="mso-bidi-font-weight:normal">fact()</b> will cause a
second call to be made with an argument of 2. this invocation will cause <b style="mso-bidi-font-weight:
normal">fact()</b> to be called a third time with an argument of 2. This call
will return 1, which is then be called a third time with an argument of 1. This
call will return1, which is then multiplied by 2 (the value of <b style="mso-bidi-font-weight:normal">n</b>
in the second invocation). This result (which is 2) is then returned to the
original invocation of <b style="mso-bidi-font-weight:normal">fact()</b> and
multiply by 3 ( the original value of <b style="mso-bidi-font-weight:normal">n).
</b>This yields the answer, 6. You might find it interesting to insert <b style="mso-bidi-font-weight:normal">println()
</b>statements into<b style="mso-bidi-font-weight:normal"> fact() </b>which will
show at what level each call is and what the intermediate answers are.</p>
<p class="MsoNormal">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
When a method calls itself, new local variables and parameters are allocated
storage on the stack, and the method code is executed with these new variables
from the start. A recursive call does not make a new copy of the method. Only
the arguments are new. As each recursive call returns, the old local variables
and parameters are removed from the stack, and execution resumes at the point of
the call inside the method. Recursive methods could be said to telescope
out and back.</p>
<p class="MsoNormal">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
Recursive versions of many routines may execute a bit more slowly than the
iterative equivalent because of the added overhead of the additional function
calls. Many recursive calls to a method could cause a stack overrun. Because
storage for parameters and local variables, it is possible that the stack could
be exhausted. If this occurs, the java run-time system will cause an exception.
However, you probably will not have to worry about this unless a recursive
routine runs wild.</p>
<p class="MsoNormal">&nbsp;<span style="mso-tab-count:1">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
</span>The main advantage to recursive methods is that they can be used to
create clearer and simpler versions of several algorithms than can their
iterative relatives. For example, the QuickSort sorting algorithm is quite
difficult to implement in an iterative way.<o:p>
