<p>Suppose I have the following snippet.</p>

<pre><code>int main()
{
    int num;
    int* cost;
    while(cin &gt;&gt; num)
    {
        int sum = 0;
        if (num == 0)
          break;

        // Dynamically allocate the array and set to all zeros
        cost = new int [num];
        memset(cost, 0, num);
        for (int i = 0; i &lt; num; i++)
        {
            cin &gt;&gt; cost[i];
            sum += cost[i];
        }
        cout &lt;&lt; sum/num;
    }
`  `delete[] cost;
    return 0;
}
</code></pre>

<p>Although I can move the <code>delete</code> statement inside the while loop

<td class="answercell">
    <div class="post-text" itemprop="text">
<blockquote>
  <p>Does C++ allocate different memory spaces each time I use operator <code>new</code>?</p>
</blockquote>

<p>Yes.</p>

<blockquote>
  <p>Does operator <code>delete</code> only delete the <em>last</em> allocated <code>cost</code> array?</p>
</blockquote>

<p>Yes. </p>

<p>You've lost the only pointers to the others, so they are irrevocably leaked. To avoid this problem, don't juggle pointers, but use <a href="http://stackoverflow.com/questions/395123">RAII</a> to manage dynamic resources automatically. <code>std::vector</code> would be perfect here (if you actually needed an array at all; your example could just keep reading and re-using a single <code>int</code>).</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>I <em>strongly</em> advise you not to use "C idioms" in a <em>C++</em> program.  Let the <code>std</code> library work <em>for</em> you:  that's why it's there.  If you want "an array (vector) of <em>n</em> integers," then that's what <code>std::vector</code> is all about, <em>and</em> it "comes with batteries included."  <em>You</em> don't have to monkey-around with things such as "setting a maximum size" or "setting it to zero."  You simply work with "this <em>thing,"</em> whose inner workings you do not [have to ...] care about, knowing that it has already been thoroughly designed and tested.</p>

<p>Furthermore, when you do this, you're working within C++'s existing framework for memory-management.  In particular, you're not doing anything "out-of-band" within your own application "that the standard library doesn't know about, and which might <em>(!!)</em> it up."</p>

<p>C++ gives you a very comprehensive library of fast, efficient, robust, well-tested functionality.  <em>Leverage</em> it.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>There is no <code>cost</code> array in your code. In your code <code>cost</code> is a <em>pointer</em>, not an array.</p>

<p>The actual <em>arrays</em> in your code are created by repetitive <code>new int [num]</code> calls. Each call to new creates a new, independent, <em>nameless</em> array object that lives somewhere in dynamic memory. The new array, once created by <code>new[]</code>, is accessible through <code>cost</code> pointer. Since the array is <em>nameless</em>, that <code>cost</code> pointer is the <em>only</em> link you have that leads to that nameless array created by <code>new[]</code>. You have no other means to access that nameless array.</p>

<p>And every time you do that <code>cost = new int [num]</code> in your cycle, you are creating a completely new, different array, breaking the link from <code>cost</code> to the previous array and making <code>cost</code> to point to the new one. </p>

<p>Since <code>cost</code> was your only link to the old array, that old array becomes inaccessible. Access to that old array is lost forever. It is becomes a memory leak.</p>

<p>As you correctly stated it yourself, your <code>delete[]</code> expression only deallocates the last array - the one <code>cost</code> ends up pointing to in the end. Of course, this is only true if your code ever executes the <code>cost = new int [num]</code> line. Note that your cycle might terminate without doing a single allocation, in which case you will apply <code>delete[]</code> to an uninitialized (garbage) pointer.</p>

<td class="answercell">
    <div class="post-text" itemprop="text">
<p>First off this line is wrong:</p>

<pre><code>memset(cost, 0, num);
</code></pre>

<p>It assumes an int is only one char long. More typically it's four. You should use something like this if you want to use memset to initialise the array:</p>

<pre><code>memset(cost, 0, num*sizeof(*cost));
</code></pre>

<p>Or better yet dump the memset and use this when you allocate the memory:</p>

<pre><code>cost = new int[num]();
</code></pre>

<p>As others have pointed out the delete is incorrectly placed and will leak all memory allocated by its corresponding new except for the last. Move it into the loop.</p>
