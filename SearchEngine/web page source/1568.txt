						<div class="post-headline">		<h1>4.3 &#8212; Static duration variables</h1>
		</div>		<div class="post-byline">By Alex, on June 19th, 2007</div>		<div class="post-bodycopy clearfix"><p>The static keyword is one of the more confusing keyword in the C++ language (maybe with the exception of the keyword &#8220;class&#8221;).  This is because it has different meanings depending on where it is used.</p>
<p>In section <a href="http://www.learncpp.com/cpp-tutorial/42-global-variables/">4.2 -- Global variables</a>, you learned that when applied to a variable declared outside of a block, it defines a global variable with internal linkage, meaning the variable could only be used in the file in which it was defined.</p>
<p>The static keyword can also be applied to variables declared inside a block, where it has a different meaning entirely.  In section <a href="http://www.learncpp.com/cpp-tutorial/4-1a-local-variables-and-local-scope/">4.1a -- Local variables and local scope</a>, you learned that local variables have automatic duration, which means they are created when the block is entered, and destroyed when the block is exited.  </p>
<p>Using the static keyword on local variables changes them from automatic duration to static duration (also called fixed duration).  A <strong>static duration</strong> variable (also called a &#8220;static variable&#8221;) is one that retains its value even after the scope in which it has been created has been exited!  Static duration variables are only created (and initialized) once, and then they are persisted throughout the life of the program.</p>
<p>The easiest way to show the difference between automatic and static duration variables is by example.</p>

<p>Static variables offer some of the benefit of global variables (they don&#8217;t get destroyed until the end of the program) while limiting their visibility to block scope.  This makes them much safer for use than global variables.</p>
<p><strong>Quiz</strong></p>
<p>1) What effect does using keyword &#8220;static&#8221; have on a global variable?  What effect does it have on a local variable?</p>
<p><strong>Quiz Answers</strong></p>
<p>1) <a class="solution_link_show" href="javascript:void(0)" onclick="wpSolutionToggle(document.getElementById('id1795919150'), this, 'Show Solution', 'Hide Solution')">Show Solution</a></p>
<div class="solution_div" id="id1795919150" style="display:none">
When applied to a global variable, the static keyword defines the global variable as having internal linkage, meaning the variable cannot be exported to other files.</p>
<p>When applied to a local variable, the static keyword defines the local variable as having static duration, meaning the variable will only be created once, and will not be destroyed until the end of the program.

		<p>s_itemID++ uses the postfix version of ++, which evaluates to a value and THEN increments.</p>
<p>So stepping through what happens:<br />
* s_itemID is initialized with value 0.<br />
* The value of s_itemID (0) is temporarily set aside.  The ++ operator increments s_itemID to 1.  Then the set aside value of 0 is returned to the caller.<br />
* The std::cout line prints the return value of 0.</p>
<p>If we&#8217;d changed line 4 to be &#8220;return ++s_itemID;&#8221; instead, it would have returned 1, because in that case, the value of s_itemID is incremented before it&#8217;s returned rather than after.</p>
