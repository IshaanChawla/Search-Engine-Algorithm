		<h1 class="firstheading"><a href="http://www.codingunit.com/c-tutorial-how-to-use-pointers" rel="bookmark" title="C Tutorial &#8211; How to use Pointers">
		C Tutorial &#8211; How to use Pointers</a></h1>

			<div class="entry">
				<div id="watermark">
					<p>To make full use of the C Programming language, you have to have a very good understanding of pointers. For most people it will take some time to fully understand pointers. So be patient. You have to learn pointers because they are used everywhere in the C language. Once you master the use of pointers, you will use them everywhere. OK, enough pep talk, let&#8217;s start.<span id="more-190"></span></p>
<p>Pointers are used (in the C language) in three different ways:</p>
<ul>
<li>To create dynamic data structures.</li>
<li>To pass and handle variable parameters passed to functions.</li>
<li>To access information stored in arrays. (Especially if you work with links).</li>
</ul>
<p>Pointers are also used by experienced programmers to make the code more efficient and thus faster.</p>
<p>So why use pointers? Why don&#8217;t we use arrays to create data structures?</p>
<p>The answer is simple. With an array you have to declare its maximum size (for every dimension) at the beginning. Let&#8217;s say you create an array that can hold a maximum of twenty megabytes. When the array is declared, the twenty megabytes is claimed. Now this time you have only data for ten megabytes.</p>
<p>(A next time it could be fifteen megabytes or five megabytes). So in this case ten megabytes of memory is wasted, because only ten megabytes from the twenty is used.</p>
<p>This is where pointers come in. With pointers, you can create dynamic data structures. Instead of claiming the memory up-front, the memory is allocated (from the heap) while the program is running. So the exact amount of memory is claimed and there is no waste. Even better, memory not used will be returned to the heap. (Freed memory can be used for other programs).</p>
<h2>Pointer basics</h2>
<p>To better understand pointers, it sometimes helps to compare a &#8220;normal variable&#8221; with a pointer.</p>
<p>When a &#8220;normal variable&#8221; is declared, memory is claimed for that variable. Let&#8217;s say you declare an integer variable MYVAR. Four bytes of memory is set aside for that variable. The location in memory is known by the name MYVAR. At the machine level that location has a memory address.</p>
<p>A pointer differs in the way that a pointer is a variable that points to another variable. A pointer holds the memory address of that variable. That variable contains a value. Pointers are also called <strong>address variables</strong> because they contain the addresses of other variables.</p>
<p><strong>Example:</strong> We have a piece of memory with a start address of 0x2000. That piece of memory contains a value and is named MYEXAMPLE.</p>
<p>(This is in fact a variable). We also have a pointer MYPOINT. In this case, our pointer MYPOINT contains the address 0x2000. This is the address of MYEXAMPLE so we say MYPOINT points to MYEXAMPLE. So there is the pointer and the value pointed to. (You can use a pointer in a certain way to get the value at the address to which the pointer points). Many novice programmers get pointers and their contents confused.<br />
So from now on every pointer starts with the extension ptr_ .  (for example: ptr_MYPOINT).</p>
<p>To declare a pointer you have to put an <strong>*</strong> in front of its name. A pointer can be <strong>typed</strong> or <strong>untyped</strong>. (A typed pointer points to a particular variable type such as an integer. An untyped pointer points to any data type). See the following example of a declaration of a typed pointer and an untyped pointer:</p>
<div id="cl">
<pre><code>
	#include&lt;stdio.h&gt;

	int main()
	{
		int *ptr_A;		/* A typed pointer */
		void *ptr_B;	/* A untyped pointer */
		return 0;
	}

</code></pre>
</div>
<p>Put the address of an integer into a &#8220;pointer to an integer&#8221; by using the &amp; operator (address operator)<br />
in front of the integer, to get the integer&#8217;s address.</p>
<p>Let&#8217;s take a look at an example:</p>
<div id="cl">
<pre><code>
	#include&lt;stdio.h&gt;

	int main()
	{
		int x;
		int *ptr_p;

		x = 5;
		ptr_p = &amp;x;

		return 0;
	}

</code></pre>
</div>
<p><strong>Note:</strong> The integer x contains the value 5 on a specific address. The address of the integer x is copied in the pointer ptr_p. So ptr_p points to the address of x. In short: ptr_p = &amp;x; means, &#8220;Assign to ptr_p the address of x.&#8221;</p>
<p>To access the value of the integer that is being pointed to, you have to dereference the pointer. The * is used to dereference a pointer. Take a look at the following example:</p>
<div id="cl">
<pre><code>
	#include&lt;stdio.h&gt;

	int main()
	{
		int x,y;
		int *ptr_p;

		x = 5;
		ptr_p = &amp;x;
		y = *ptr_p;
		printf("%d\n", y);

		return 0;
	}

</code></pre>
</div>
<p><strong>Note:</strong> The integer x has a value of five. The pointer ptr_p gets the address of integer x.<br />
The value pointed to is *ptr_p. (in this case five). So the integer y now contains the value of five.</p>
<p>Take a look at the following example:</p>
<div id="cl">
<pre><code>
	#include&lt;stdio.h&gt;

	int main()
	{
		int x;
		int *ptr_p;

		x = 5;
		ptr_p = &amp;x;

		*ptr_p = 10;		

		printf("%d\n", x);

		return 0;
	}

</code></pre>
</div>
<p><strong>Note:</strong> First the ptr_p contains the address of the integer x (ptr_p = &amp;x). Then we change the value of the integer where ptr_p is pointing to (*ptr_p = 10;). The integer x now equals ten.</p>
<p>That&#8217;s all for this tutorial. You should now understand the fundamentals of pointers. If not, read it again until you&#8217;ll get it, before reading the <a href="http://www.codingunit.com/c-tutorial-more-on-pointers" title="C Tutorial – More on Pointers">next C tutorial on pointers</a>.</p>
