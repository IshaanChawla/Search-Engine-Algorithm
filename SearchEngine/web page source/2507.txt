<h1 class="h1">Spring Interview Questions</h1>
<p>Spring interview questions and answers are frequently asked because it is now widely used framework to develop enterprise application in java. There are given a list of top 40 frequently asked spring interview questions.</p>
<hr/>
<h3 class="h3">1) What is Spring?</h3>
<p>It is a lightweight, loosely coupled and integrated framework for developing enterprise applications in java.</p>

<hr/>
<h3 class="h3">2) What are the advantages of spring framework?</h3>
<ol>
<li>Predefined Templates</li>
<li>Loose Coupling</li>
<li>Easy to test</li>
<li>Lightweight</li>
<li>Fast Development</li>
<li>Powerful Abstraction</li>
<li>Declarative support</li>
</ol>
<a href="spring-tutorial" target="_blank">More details...</a>
<hr/>
<h3 class="h3">3) What are the modules of spring framework?</h3>
<ol>
<li>Test</li>
<li>Spring Core Container</li>
<li>AOP, Aspects and Instrumentation</li>
<li>Data Access/Integration</li>
<li>Web</li>
</ol>
<a href="spring-modules" target="_blank">More details...</a>

<hr/>
<h3 class="h3">4) What is IOC and DI?</h3>
<p>IOC (Inversion of Control) and DI (Dependency Injection) is a design pattern to provide loose coupling. It removes the dependency from the program.</p>
<p>Let's write a code without following IOC and DI.</p>
<div class="codeblock"><textarea name="code" class="java" >
public class Employee{
Address address;
Employee(){
address=new Address();//creating instance
}
}
</textarea></div>
<p>Now, there is dependency between Employee and Address because Employee is forced to use the same address instance.</p>
<p>Let's write the IOC or DI code.</p>
<div class="codeblock"><textarea name="code" class="java" >
public class Employee{
Address address;
Employee(Address address){
this.address=address;//not creating instance
}
}
</textarea></div>
<p>Now, there is no dependency between Employee and Address because Employee is not forced to use the same address instance. It can use any address instance.</p>

<hr/>
<h3 class="h3">5) What is the role of IOC container in spring?</h3>
<p>IOC container is responsible to:</p>
<ul>
<li>create the instance</li>
<li>configure the instance, and</li>
<li>assemble the dependencies</li>
</ul>
<a href="ioc-container" target="_blank">More details...</a>

<hr/>
<h3 class="h3">6) What are the types of IOC container in spring?</h3>
<p>There are two types of IOC containers in spring framework.</p>
<ol>
<li>BeanFactory</li>
<li>ApplicationContext</li>
</ol>
<a href="ioc-container" target="_blank">More details...</a>

<hr/>
<h3 class="h3">7) What is the difference between BeanFactory and ApplicationContext?</h3>
<p>BeanFactory is the <b>basic container</b> whereas ApplicationContext is the <b>advanced container</b>. ApplicationContext extends the BeanFactory interface. ApplicationContext provides more facilities than BeanFactory such as integration with spring AOP, message resource handling for i18n etc.</p>

<hr/>
<h3 class="h3">8) What is the difference between constructor injection and setter injection?</h3>
<table class="alt">
<tr><th>No.</th><th>Constructor Injection</th><th>Setter Injection</th></tr>
<tr><td>1)</td><td>No Partial Injection</td><td>Partial Injection</td></tr>
<tr><td>2)</td><td>Desn't override the setter property</td><td>Overrides the constructor property if both are defined.</td></tr>
<tr><td>3)</td><td>Creates new instance if any modification occurs</td><td>Doesn't create new instance if you change the property value</td></tr>
<tr><td>4)</td><td>Better for too many properties</td><td>Better for few properties.</td></tr>
</table>
<a href="difference-between-constructor-and-setter-injection" target="_blank">More details...</a>

<hr/>
<h3 class="h3">9) What is autowiring in spring? What are the autowiring modes?</h3>
<p>Autowiring enables the programmer to inject the bean automatically. We don't need to write explicit injection logic.</p>
<p>Let's see the code to inject bean using dependency injection.</p>
<div class="codeblock"><textarea name="code" class="java" >
&lt;bean id="emp" class="com.javatpoint.Employee" autowire="byName" /&gt;
</textarea></div>

<p>The autowiring modes are given below:</p>
<table class="alt">
<tr><th>No.</th><th>Mode</th><th>Description</th></tr>
<tr><td>1)</td><td>no</td><td>this is the default mode, it means autowiring is not enabled.</td></tr>
<tr><td>2)</td><td>byName</td><td>injects the bean based on the property name. It uses setter method.</td></tr>
<tr><td>3)</td><td>byType</td><td>injects the bean based on the property type. It uses setter method.</td></tr>
<tr><td>4)</td><td>constructor</td><td>It injects the bean using constructor</td></tr>
</table>

<p class="n">The "autodetect" mode is deprecated since spring 3.</p>

<hr/>
<h3 class="h3">10) What are the different bean scopes in spring?</h3>
<p>There are 5 bean scopes in spring framework.</p>
<table class="alt">
<tr><th>No.</th><th>Scope</th><th>Description</th></tr>
<tr><td>1)</td><td>singleton</td><td>The bean instance will be only once and same instance will be returned by the IOC container. It is the default scope.</td></tr>
<tr><td>2)</td><td>prototype</td><td>The bean instance will be created each time when requested.</td></tr>
<tr><td>3)</td><td>request</td><td>The bean instance will be created per HTTP request.</td></tr>
<tr><td>4)</td><td>session</td><td>The bean instance will be created per HTTP session.</td></tr>
<tr><td>5)</td><td>globalsession</td><td>The bean instance will be created per HTTP global session. It can be used in portlet context only.</td></tr>
</table>

<hr/>
<h3 class="h3">11) In which scenario, you will use singleton and prototype scope?</h3>
<p>Singleton scope should be used with EJB <b>stateless session bean</b> and prototype scope with EJB <b>stateful session bean</b>.</p>

<hr/>
<h3 class="h3">12) What are the transaction management supports provided by spring?</h3>
<p>Spring framework provides two type of transaction management supports:</p>
<ol>
<li><b>Programmatic Transaction Management</b>: should be used for few transaction operations.</li>
<li><b>Declarative Transaction Management</b>: should be used for many transaction operations.</li>
</ol>
<hr/>
<h2 class="h2">&raquo; Spring JDBC Interview Questions</h2>

<hr/>
<h3 class="h3">13) What are the advantages of JdbcTemplate in spring?</h3>
<p><b>Less code</b>: By using the JdbcTemplate class, you don't need to create connection,statement,start transaction,commit transaction and close connection to execute different queries. You can execute the query directly.</p>
<a href="spring-JdbcTemplate-tutorial" target="_blank">More details...</a>

<hr/>
<h3 class="h3">14) What are classes for spring JDBC API?</h3>
<ol>
<li>JdbcTemplate</li>
<li>SimpleJdbcTemplate</li>
<li>NamedParameterJdbcTemplate</li>
<li>SimpleJdbcInsert</li>
<li>SimpleJdbcCall</li>
</ol>
<a href="spring-JdbcTemplate-tutorial" target="_blank">More details...</a>

<hr/>
<h3 class="h3">15) How can you fetch records by spring JdbcTemplate?</h3>
<p>You can fetch records from the database by the <b>query method of JdbcTemplate</b>. There are two interfaces to do this:</p>
<ol>
<li><a href="ResultSetExtractor-example" target="_blank">ResultSetExtractor</a></li>
<li><a href="RowMapper-example" target="_blank">RowMapper</a></li>
</ol>

<hr/>
<h3 class="h3">16) What is the advantage of NamedParameterJdbcTemplate?</h3>
<p>NamedParameterJdbcTemplate class is used to pass value to the named parameter. A named parameter is better than ? (question mark of PreparedStatement).</p>
<p>It is <b>better to remember</b>.</p>
<a href="spring-NamedParameterJdbcTemplate-example" target="_blank">More details...</a>
<hr/>
<h3 class="h3">17) What is the advantage of SimpleJdbcTemplate?</h3>
<p>The <b>SimpleJdbcTemplate</b> supports the feature of var-args and autoboxing.</p>
<a href="spring-SimpleJdbcTemplate-example" target="_blank">More details...</a>

<hr/>
<h2 class="h2">&raquo; Spring AOP Interview Questions</h2>

<hr/>
<h3 class="h3">18) What is AOP?</h3>
<p>AOP is an acronym for Aspect Oriented Programming. It is a methodology that divides the program logic into pieces or parts or concerns.</p>
<p>It increases the modularity and the key unit is Aspect.</p>
<a href="spring-aop-tutorial" target="_blank">More details...</a>

<hr/>
<h3 class="h3">19) What are the advantages of spring AOP?</h3>
<p>AOP enables you to dynamically add or remove concern before or after the business logic. It is <b>pluggable</b> and <b>easy to maintain</b>.</p>
<a href="spring-aop-tutorial" target="_blank">More details...</a>

<hr/>
<h3 class="h3">20) What are the AOP terminology?</h3>
<p>AOP terminologies or concepts are as follows:</p>
<ul>
<li>JoinPoint</li>
<li>Advice</li>
<li>Pointcut</li>
<li>Aspect</li>
<li>Introduction</li>
<li>Target Object</li>
<li>Interceptor</li>
<li>AOP Proxy</li>
<li>Weaving</li>
</ul>
<a href="spring-aop-tutorial" target="_blank">More details...</a>
<hr/>
<h3 class="h3">21) What is JoinPoint?</h3>
<p>JoinPoint is any point in your program such as field access, method execution, exception handling etc.</p>
<hr/>
<h3 class="h3">22) Does spring framework support all JoinPoints?</h3>
<p>No, spring framework supports method execution joinpoint only.</p>
<hr/>
<h3 class="h3">23) What is Advice?</h3>
<p>Advice represents action taken by aspect.</p>
<hr/>
<h3 class="h3">24) What are the types of advice in AOP?</h3>
<p>There are 5 types of advices in spring AOP.</p>
<ol>
<li>Before Advice</li>
<li>After Advice</li>
<li>After Returning Advice</li>
<li>Throws Advice</li>
<li>Around Advice</li>
</ol>
<hr/>
<h3 class="h3">25) What is Pointcut?</h3>
<p>Pointcut is expression language of Spring AOP.</p>
<hr/>
<h3 class="h3">26) What is Aspect?</h3>
<p>Aspect is a class in spring AOP that contains advices and joinpoints.</p>
<hr/>
<h3 class="h3">27) What is Introduction?</h3>
<p>Introduction represents introduction of new fields and methods for a type.</p>
<hr/>
<h3 class="h3">28) What is target object?</h3>
<p>Target Object is a proxy object that is advised by one or more aspects.</p>
<hr/>
<h3 class="h3">29) What is interceptor?</h3>
<p>Interceptor is a class like aspect that contains one advice only.</p>
<hr/>
<h3 class="h3">30) What is weaving?</h3>
<p>Weaving is a process of linking aspect with other application.</p>
<hr/>
<h3 class="h3">31) Does spring perform weaving at compile time?</h3>
<p>No, spring framework performs weaving at runtime.</p>
<hr/>
<h3 class="h3">32) What are the AOP implementation?</h3>
<p>There are 3 AOP implementation.</p>
<ol>
<li>Spring AOP</li>
<li>Apache AspectJ</li>
<li>JBoss AOP</li>
</ol>

<hr/>
<h2 class="h2">&raquo; Spring MVC Interview Questions</h2>

<hr/>
<h3 class="h3">33) What is the front controller class of Spring MVC?</h3>
<p>The <b>DispatcherServlet</b> class works as the front controller in Spring MVC.</p>
<a href="spring-3-mvc-tutorial" target="_blank">More details...</a>
<hr/>
<h3 class="h3">34) What does @Controller annotation?</h3>
<p>The <b>@Controller</b> annotation marks the class as controller class. It is applied on the class.</p>
<hr/>
<h3 class="h3">35) What does @RequestMapping annotation?</h3>
<p>The <b>@RequestMapping</b> annotation maps the request with the method. It is applied on the method.</p>
<hr/>
<h3 class="h3">36) What does the ViewResolver class?</h3>
<p>The <b>View Resolver</b> class resolves the view component to be invoked for the request. It defines prefix and suffix properties to resolve the view component.</p>
<hr/>
<h3 class="h3">37) Which ViewResolver class is widely used?</h3>
<p>The <b>org.springframework.web.servlet.view.InternalResourceViewResolver</b> class is widely used.</p>
<hr/>
<h3 class="h3">38) Does spring MVC provide validation support?</h3>
<p>Yes.</p>
