<td  id="content" colspan="7"><h1 class="content_heading">Java Programming Tutorial<br/>
  A Fast-Moving Guide to Java Programming for Experienced Developers</h1>

<h2 class="content_subheading" align="center"><font color="#0000ff">
  Interested in training from the author of these tutorials?
  See the upcoming <a href="../public-courses/java7/index.html">Java 7</a> and  
  <a href="../public-courses/java8/index.html">Java 8</a>
  programming courses in Maryland (co-sponsored by Johns Hopkins <i>Engineering for Professionals</i>),
  or contact <a href="mailto:info@coreservlets.com">info@coreservlets.com</a>
  for info on customized courses onsite at <i>your</i> location. See 
  <a href="http://courses.coreservlets.com/java-training.html">the Java training page</a> for
  various options for the onsite courses.
</font></h2>
<br/>
<p>Following is an extensive series of tutorials on Java programming from coreservlets' lead instructor
  <a href="http://courses.coreservlets.com/about-instructor.html">Marty Hall</a>. 
  Since each section includes
  exercises and exercise solutions, this can also be viewed as a self-paced Java training course.
  All the slides, source code, exercises, and exercise solutions are free for unrestricted use.
  Click on a section below to expand its content. The relatively few parts on IDE development and deployment use
  Eclipse, but of course none of the actual code is Eclipse-specific. These tutorials assume that you already know 
  <i>some</i> high-level programming language; they move too fast
  for first-time programmers. To arrange a Java-related course at <i>your</i> organization based on these materials, contact <a href="mailto:info@coreservlets.com">info@coreservlets.com</a> or see 
  <a href="http://courses.coreservlets.com/java-training.html">the Java training page</a>.</p>

<div class="accordion-div">
  <h1 class="book_part"><a class="accordionLink" href="#Tutorial-Intro" name="Tutorial-Intro">
  <b>Overview of the Java Programming Tutorial Series</b></a></h1>
<div> <!-- Accordion content -->
  <p>These fast-moving tutorials are derived from 
  <a href="http://courses.coreservlets.com/about-instructor.html">Marty Hall's</a> world-renowned
  <a href="http://courses.coreservlets.com/">live JSF 2, PrimeFaces, Hadoop, Ajax, jQuery, Android, and Java EE training courses</a>, and have been tested by Marty in live training courses
    in the US, Canada, Australia, Cambodia, Puerto Rico, Japan, Mexico, India, Norway, and the Philippines. 
    Click on a topic below to get the detailed tutorial,
    download the source code, or try out exercises on the topic.
  <a href="index.html">The training materials home page</a> has tutorials on many  
    more advanced topics related to Java and Web programming,
    but the core Java language is the foundation for all of the topics.
  </p>
  <p>
    The key to learning is practicing, so each section contains exercises for you to try on your own.
    The exercise solutions are also included, but you will almost certainly learn better if you
    try some or all of the exercises on your own before peeking at our solutions. <b>Update:</b> based on student
    feedback, we have added more simple/warmup exercises to each section below (and added the corresponding code
    to the downloadable solution sets).
  </p>
  <p>
    Because Java 8 introduces such major changes, there is also a separate
  <a href="http://www.coreservlets.com/java-8-tutorial/">tutorial on lambdas and streams in Java 8</a>. 
    That tutorial, unlike this one, is aimed at people that <i>already</i> know Java, and it <i>only</i>
    covers the new Java 8 features. For general Java programming, the tutorial here is the one you want,
    and <i>all</i> the Java 8 topics are also included <i>in identical form</i> in this tutorial.
  </p>
  <p>
    To arrange a Java-related course at <i>your</i> organization based on these materials, contact
  <a href="mailto:info@coreservlets.com">info@coreservlets.com</a> or see 
  <a href="http://courses.coreservlets.com/java-training.html">the Java training page</a>. 
    These courses can be customized to
    use any combination of the materials on the <a href="index.html">J2EE tutorials site</a> 
    (Java 7, Java 8, JSF 2, PrimeFaces, Android development, Hadoop, GWT,
    Ajax, jQuery, Ext/JS, Spring, Hibernate, RESTful Web Services, servlets, JSP, etc.),
    and new materials can be added for specific client needs. To learn more details about the instructor,
    the curriculum for the live training courses, or the public course schedule, please see
  <a href="http://courses.coreservlets.com">http://courses.coreservlets.com</a>.
  </p>

<h1 class="book_part"><a name="Java-8-Overview"><b>Topics Specific to Java 8</b></a></h1>
<div> <!-- Accordion content -->
  <p>
    Originally, the Java 8 topics were available only separately as part of 
  <a href="http://www.coreservlets.com/java-8-tutorial/">the Java 8 lambdas and streams tutorial</a>. 
    However, the final version of Java SE 8 was released in March 2014, Java 8 is now mainstream, 
    and the formerly separate topics are now integrated into this general Java tutorial. The 
  <a href="http://www.coreservlets.com/java-8-tutorial/">Java 8 tutorial</a> is still being maintained
    for those already familiar with earlier Java versions, but <i>the exact same topics are reproduced
      in identical form below</i>.
  </p>
  <p>
    Although Java 8 is now a standard part of Java,  it is by <i>far</i> the most significant upgrade to the Java programming language since at least 2004, probably since Java's inception, and the features are particularly useful for those doing enterprise and server-side Java, or those dealing with large data sets. In general, Java 8's high-level goals were to make more flexible code, to better use multiple cores, and to more easily deal with large data sets. In specific, there are four main reasons that it is important to know Java 8:
  </p>
  <ol>
    <li><b>More flexible and reusable code (thanks to lambdas).</b> Lambda expressions in Java 8 are a way of 
      representing "functions", and their judicious use can make your code significantly more adapatable and 
      reusable.</li>
    <li><b>Convenience (thanks to high-level Stream methods).</b> Streams are wrappers around collections (or 
      other data sources) that use lambda expressions pervasively. They support many convenient and high-performance 
      operations that use lambdas, including "map", "reduce", "filter", and "forEach". These methods make many types 
      of code much simpler to write, as compared to the comparatively clunky and low-level Collection methods.</li>
    <li><b>Faster and more memory efficient code (thanks to lazy evaluation and automatic parallelization).</b> 
      Streams support lazy evaluation, so if you map firstName over Employees, filter ones that start with "P", 
      then choose the first, it really only maps and filters until the first match. Streams can also be parallel, 
      so that the operations are automatically done in parallel, without any explicit multithreading code.</li>
    <li><b>Adapt to the times.</b> Others will be using lambdas and streams, since they are standard parts of Java SE 
      as of 3/2014. So, you have to learn the Java 8 features simply to be able to use and modify others code. 
      Besides, once you get a taste of their power, you will want to use the new features frequently.</li>
  </ol>
</div> <!-- Accordion content -->
