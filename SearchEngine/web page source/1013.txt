<div style="color:#000080;font-family:Arial;font-size:10pt">
Java defines four integer types: <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">byte</font></b>,
<b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">short</font></b>, <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int</font></b>,
and <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">long</font></b>.
All of these are signed, positive and negative values. Java does not support unsigned,
positive-only integers. Many other computer languages, including C/C++, support both
signed and unsigned integers. However, Java's designers felt that unsigned integers
were unnecessary.
<p>Specifically, they felt that the concept of <i><font FACE="Arial,Italic" SIZE="2" COLOR="#010100">unsigned
</font></i>was used mostly to
specify the behavior of the <i><font FACE="Arial,Italic" SIZE="2" COLOR="#010100">high-order
bit, </font></i>which defined the <i><font FACE="Arial,Italic" SIZE="2" COLOR="#010100">sign
</font></i>of an <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int
</font></b>when expressed as a number. Java manages the
meaning of the high-order bit differently, by adding a special &quot;unsigned right
shift&quot; operator. Thus, the need for an unsigned integer type was eliminated.</p>
<p>The <i><font FACE="Arial,Italic" SIZE="2" COLOR="#010100">width
</font></i>of an integer type should
not be thought of as the amount of storage it consumes, but rather as the <i><font FACE="Arial,Italic" SIZE="2" COLOR="#010100">behavior
</font></i>it defines for variables
and expressions of that type. The Java run-time environment is free to use whatever size it
wants, as long as the types behave as you declared them. In fact, at least one
implementation stores <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">byte</font></b>s
and <font FACE="Arial,Bold" SIZE="2" COLOR="#010100">
<b>short</b></font>s as
32-bit (rather than 8- and 16-bit) values to improve performance, because that is the word size of most computers currently in use.</p>
<p>The width and ranges of these integer types vary widely, as
shown in this table:</p>

<p>Let's look at each type of integer.</p>
<p><b><font FACE="Arial,Bold" SIZE="4" COLOR="#810000">
byte
</font></b>
</p>
<p>The smallest integer type is <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">byte</font></b>.
This is a signed 8-bit type that has a range from –128 to 127. Variables of type
<b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">byte
</font></b>are especially useful
when you're working with a stream of data from a network or file. They are also useful when you're
working with raw binary data that may not be directly compatible with Java's other
built-in types. Byte variables are declared by use of the <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">byte
</font></b>keyword. For example, the
following declares two <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">byte
</font></b>variables called <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">b
</font></b>and <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">c</font></b>:</p>
<p><font FACE="CourierNew" SIZE="2" COLOR="#010100">
byte b, c;
</font></p>
<p><b><font FACE="Arial,Bold" SIZE="4" COLOR="#810000">
short
</font></b></p>
<p><b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">
short </font></b>is
a signed 16-bit type. It has a range from –32,768 to 32,767. It is probably
the least-used Java type, since it is defined as having its high
byte first (called <i><font FACE="Arial,Italic" SIZE="2" COLOR="#010100">big-endian
</font></i>
format). This type is mostly applicable to 16-bit computers,
which are becoming increasingly scarce. Here are some examples of <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">short
</font></b>variable declarations:</p>
<p><font FACE="CourierNew" SIZE="2" COLOR="#010100">
short s;
<br>short t;
</font></p>
<p><b><font COLOR="#000081">
Note
</font></b>
<br>&quot;Endianness&quot; describes how multibyte data types, such
as <b>short</b>, <b>int</b>,
and <b>long</b>, are stored in
memory. If it takes 2 bytes to represent a <b>short</b>,
then which one comes first, the most significant or the least
significant? To say that a machine is big-endian, means that the most significant byte is
first, followed by the least significant one. Machines such as the SPARC and
PowerPC are big-endian, while the Intel x86 series is little-endian.</p>
<p><b><font FACE="Arial,Bold" SIZE="4" COLOR="#810000">
int
</font></b>
</p>
<p>The most commonly used integer type is <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int</font></b>.
It is a signed 32-bit type that has a range from –2,147,483,648 to 2,147,483,647. In addition to other
uses, variables of type <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int
</font></b>are commonly employed to control loops and to index arrays. Any time
you have an integer expression involving <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">byte</font></b>s,
<b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">short</font></b>s, <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int</font></b>s,
and literal numbers, the entire expression is <font FACE="Arial,Italic" SIZE="2" COLOR="#010100">
<i>promoted </i> </font>to <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int </font></b>before
the calculation is done.</p>
<p>The <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int
</font></b>type is the most
versatile and efficient type, and it should be used most of the time when you want to create a number for counting or indexing
arrays or doing integer math. It may seem that using <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">short
</font></b>or <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">byte
</font></b>will save space, but
there is no guarantee that Java won't promote those types to <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int
</font></b>internally anyway.
Remember, type determines behavior, not size. (The only exception is arrays,
where <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">byte </font></b>is
guaranteed to use only one byte per array element, <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">short
</font></b>will use two bytes, and <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int
</font></b>will use four.)</p>
<p><b><font FACE="Arial,Bold" SIZE="4" COLOR="#810000">
long
</font></b></p>
<p><b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">
long </font></b>is a
signed 64-bit type and is useful for those occasions where an <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int
</font></b>type is not large enough to hold the desired value. The range of a <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">long
</font></b>is quite large. This
makes it useful when big, whole numbers are needed. For example, here is
a program that computes the number of miles that light will travel in a
specified number of days.&nbsp;</p>
<blockquote>
  <p><font FACE="CourierNew" SIZE="2" COLOR="#010100">
// Compute distance light travels using long variables.
<br>class Light {
<br>public static void main(String args[]) {
<br>int lightspeed;
<br>long days;
<br>long seconds;
<br>long distance;
<br>// approximate speed of light in miles per second
<br>lightspeed = 186000;
<br>days = 1000; // specify number of days here
<br>seconds = days * 24 * 60 * 60; // convert to seconds
<br>distance = lightspeed * seconds; // compute distance
<br>System.out.print(&quot;In &quot; + days);
<br>System.out.print(&quot; days light will travel about &quot;);
<br>System.out.println(distance + &quot; miles.&quot;);
<br>}
<br>}
</font>
  </p>
</blockquote>
<p>This program generates the following output:</p>
<p><font FACE="CourierNew" SIZE="2" COLOR="#010100">
In 1000 days light will travel about 16070400000000 miles.
</font>
</p>
<p>Clearly, the result could not have been held in an <b><font FACE="Arial,Bold" SIZE="2" COLOR="#010100">int
</font></b>variable.</p>
